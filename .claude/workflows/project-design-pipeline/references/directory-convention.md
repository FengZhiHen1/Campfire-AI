# 全局目录结构约定

> 本文件为工作流级共享规范。所有 `project-design-pipeline` 的 Skill 均需遵守。
> 定义 `docs/` 目录下各产物的路径、命名规则和用途。
>
> **⚠️ 红线警告：本文件中所有路径格式均为硬性约束。任何 Skill 在写入 `docs/` 前必须校验目标路径符合本规范。
> 违反格式的路径（如含中文数字、非标准序号等）视为产物不合格，必须拒绝写入并修正。**

---

## 1. docs/ 直接产物

| 文件 | 说明 |
|------|------|
| `docs/项目名称-技术栈设计.md` | 技术栈设计文档，项目架构与技术选型权威记录 |
| `docs/项目名称-项目结构.md` | 项目结构设计文档，分层架构、目录骨架、层间数据流、包组织约定 |

---

## 2. docs/功能设计/ 子目录

### 2.1 项目级分析文档

| 文件 | 说明 |
|------|------|
| `docs/功能设计/功能模块全拆解.md` | 所有功能模块的登记与分类 |
| `docs/功能设计/模块依赖关系分析.md` | 模块间依赖关系、Mermaid 图、依赖矩阵 |
| `docs/功能设计/[序号]-[分组]/[编号]-[名称]/_sync-issues.md` | 模块级同步矛盾记录（子工作流 s12 写入、父工作流 s09 扫描） |

### 2.2 单模块文档目录

每个功能模块在独立子目录下产生三份文档：

```
docs/功能设计/[序号]-[分组]/[编号]-[名称]/
  ├── [编号]-[名称]-意图文档.md
  ├── [编号]-[名称]-设计文档.md
  └── [编号]-[名称]-落地规范.md
```

**命名约定**：

| 占位符 | 格式要求 | 来源 | 示例 |
|--------|---------|------|------|
| `[序号]` | **两位阿拉伯数字**（`01`–`99`），决定分组目录排列顺序 | `功能模块全拆解.md` 的章节顺序 | `01`、`02`、`12` |
| `[分组]` | 纯中文类别名，反映模块业务归属，**不得含序号或编号** | `功能模块全拆解.md` 的章节标题（去除序号前缀后） | `用户域`、`商品域` |
| `[编号]` | **必须严格使用** `功能模块全拆解.md` 中分配给该模块的编号，不得自行编造或修改 | `功能模块全拆解.md` 的"模块编号"列 | `USR-01`、`M01` |
| `[名称]` | 中文，与 `功能模块全拆解.md` 中登记的模块名称一致 | `功能模块全拆解.md` 的"模块名称"列 | `订单系统`、`支付网关` |

**示例**：
```
docs/功能设计/01-用户域/USR-01-订单系统/
docs/功能设计/01-用户域/USR-02-支付网关/
docs/功能设计/02-商品域/PRD-01-库存管理/
```

## 3. 硬性禁止格式

> 以下路径格式**一律禁止**。任何 Skill 不得生成、写入或接受以下格式的路径。

| 禁止格式 | 错误示例 | 正确写法 |
|---------|---------|---------|
| 中文数字作为序号 | `一-用户域`、`五-管线容错与恢复` | `01-用户域`、`05-管线容错与恢复` |
| 罗马数字作为序号 | `I-用户域`、`V-管线` | `01-用户域`、`05-管线` |
| 序号与分组缺少分隔符 | `01用户域`、`05管线` | `01-用户域`、`05-管线` |
| 序号不足两位 | `1-用户域`、`5-管线` | `01-用户域`、`05-管线` |
| 分组名包含序号 | `01-一-用户域`、`05-五-管线` | `01-用户域`、`05-管线` |
| 模块名称含连词 | `用户注册与登录`、`处理订单并通知` | 拆分为 `用户注册` 和 `用户登录` |
| 目录层级缺失 | `docs/功能设计/USR-01-订单系统-意图文档.md` | `docs/功能设计/01-用户域/USR-01-订单系统/USR-01-订单系统-意图文档.md` |

## 4. 序号标准化规则

> 当源文档（如 `功能模块全拆解.md`）使用非阿拉伯数字序号时，必须按以下映射标准化。

| 源序号格式 | 标准化为 | 示例 |
|-----------|---------|------|
| 中文数字 `一`、`二`…`九`、`十`… | 对应两位阿拉伯数字 | `一`→`01`、`五`→`05`、`十二`→`12` |
| 阿拉伯数字 `1`、`2`… | 补齐两位 | `1`→`01`、`5`→`05`、`12`→`12` |
| 罗马数字 `I`、`II`… | 对应两位阿拉伯数字 | `I`→`01`、`V`→`05` |

> **优先策略**：`module-breakdown-designer` 应在生成 `功能模块全拆解.md` 时直接使用 `## 01-[分组名]` 格式，从源头消除转换需求。
> Skill 在读取源文档推断分组路径时，若必须执行转换，应遵循本表规则。

---

## 5. 路径构建流程（所有 Skill 通用）

任何需要写入 `docs/功能设计/` 的 Skill，必须按以下流程构建目标路径：

1. **读取权威源**：以 `docs/功能设计/功能模块全拆解.md` 为唯一权威数据源
2. **提取字段**：从全拆解表中提取目标模块的 `[编号]`、`[名称]`、所属功能域章节
3. **标准化序号**：将功能域章节序号标准化为两位阿拉伯数字（参照 §4 映射表）。若章节标题已为 `[两位数字]-[分组名]` 格式，直接取用
4. **组装路径**：`docs/功能设计/[序号]-[分组]/[编号]-[名称]/[编号]-[名称]-[文档类型].md`
5. **校验**：对照 §3 禁止格式清单逐项检查，确认无违规项后写入

---

## 6. 路径自检清单

> 所有 Skill 在写入 `docs/功能设计/` 前必须逐项通过。任一项未通过 = 禁止写入，需修正后重新校验。

- [ ] `[序号]` 为两位阿拉伯数字（`01`–`99`），非中文数字、罗马数字或单位数
- [ ] `[分组]` 为纯中文类别名，不含序号、编号或数字前缀
- [ ] `[编号]` 与 `功能模块全拆解.md` 中登记完全一致，无自行编造或修改
- [ ] `[名称]` 与 `功能模块全拆解.md` 中登记完全一致
- [ ] 完整路径格式为 `docs/功能设计/[序号]-[分组]/[编号]-[名称]/[编号]-[名称]-[文档类型].md`
- [ ] 目录层级完整（三级：`功能设计` → `序号-分组` → `编号-名称`），无扁平化或层级缺失
- [ ] 路径中无中文数字、罗马数字、连词（"与"/"及"/"和"）

## 7. 临时文件

Agent 运行时的中间产物写入 `.tmp/<workflow_instance_id>/`，不在 `docs/` 下留痕迹。

临时文件命名使用 kebab-case（如 `requirements-context.md`、`architecture-decisions.md`）。
