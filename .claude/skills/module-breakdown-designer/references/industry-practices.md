# 行业常见实践 — 模块检查清单

使用本参考文件，将提取出的模块与已识别项目类型的标准功能域进行对比。如果设计文档遗漏了某项，评估是否应将其作为 `🔄 行业补充` 模块添加。

---

## SaaS / B2B 平台

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **身份与访问** | 用户注册、登录（多种 SSO）、多因素认证、基于角色的访问控制、团队/组织管理、API 密钥管理 | 被认为"认证库会处理" |
| **计费与支付** | 订阅计划、用量计量、发票、支付网关集成、税金计算、催款管理 | 被推迟到"上线后" |
| **审计与合规** | 活动日志、管理审计追踪、数据保留策略、GDPR 删除、导出用户数据 | 被视为"合规开销" |
| **通知** | 邮件模板、应用内通知、推送通知、通知偏好、摘要调度 | 被混入其他模块 |
| **导入/导出** | 批量数据导入、批量导出、格式转换（CSV/Excel/JSON）、进度追踪 | 被视为"高级用户功能" |
| **集成** | Webhook 管理、第三方 API 连接器、OAuth 应用市场、事件流 | 被推迟到"平台阶段" |
| **分析与报表** | 用量仪表盘、管理报表、自定义报表构建器、定时报表、数据导出 | 与产品分析混淆 |
| **支持与反馈** | 应用内帮助中心、工单升级、功能请求投票、NPS 调研 | 被分配给外部工具 |
| **速率限制与配额** | 每用户速率限制、每组织配额、突发处理、限制覆盖管理 | 被埋入 API 网关配置 |

## AI Agent / 大模型应用

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **提示工程** | 提示版本管理、A/B 测试、提示模板库、动态提示组装 | 被视为"实现细节" |
| **模型路由** | 基于成本的模型选择、降级链路、基于延迟的路由、模型能力注册表 | 在客户端中硬编码 |
| **上下文管理** | Token 预算分配、上下文窗口压缩、相关性评分、记忆检索 | 被混入"记忆系统" |
| **输出验证** | 模式验证、语义评分、幻觉检测、输出过滤、重试逻辑 | 被认为"框架会处理" |
| **成本追踪** | 单次请求 Token 计数、每用户成本分摊、预算告警、用量预测 | 被推迟到"计费阶段" |
| **人工介入** | 审批队列、基于置信度的路由、反馈收集、偏好学习 | 被视为"UI 层面的问题" |
| **智能体编排** | 多智能体调度、状态持久化、检查点/恢复、事件路由 | 框架特定，似乎"已经解决" |
| **嵌入与检索** | 文档分块、嵌入生成、向量搜索、混合检索（关键词+语义）、重排序 | 被混入 RAG 流水线 |
| **安全护栏** | 毒性检测、PII 脱敏、主题拦截、对抗性输入过滤 | 被视为"企业级功能" |
| **评估** | 基准数据集管理、自动评估流水线、人工评估界面、回归测试 | 被推迟到"质量问题出现时" |

## 电商

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **目录管理** | 商品分类、SKU 变体、属性模式、批量导入、目录版本控制 | 一开始很简单，后来变复杂 |
| **库存** | 库存水平、预留库存、多仓库、安全库存告警、缺货订单 | 最初假设"永远有货" |
| **定价** | 基础价格、动态定价、优惠码、阶梯定价、货币转换 | 一开始硬编码 |
| **履约** | 订单拆分、物流商选择、物流追踪集成、配送预估 | 被推迟给"物流合作伙伴" |
| **退货与退款** | 退货授权、退款计算、重新入库、换货处理 | "一开始我们手动处理" |
| **评价与评分** | 客户评价、卖家评分、审核队列、 helpfulness 投票、照片上传 | 被视为"锦上添花" |
| **搜索与发现** | 分面搜索、自动补全、推荐引擎、个性化、热门趋势 | 一开始用简单的 LIKE 查询 |
| **购物车与结账** | 访客结账、保存支付方式、地址簿、税金计算、放弃购物车召回 | 早期设计过度简化 |
| **促销** | 限时抢购、买 X 送 Y、积分、推荐奖励、联盟追踪 | 后来临时添加 |
| **多商户** | 卖家入驻、佣金计算、卖家仪表盘、纠纷仲裁 | 直到 marketplace 阶段才需要 |

## 内部工具 / 管理后台

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **数据管理** | CRUD 脚手架、批量操作、数据导入/导出、软删除、回收站恢复 | 由 admin 框架生成 |
| **权限** | 资源级权限、字段级可见性、操作级授权 | 假设"管理员能看到一切" |
| **工作流引擎** | 审批链、条件分支、升级计时器、工作流模板 | 用硬编码的 if 语句构建 |
| **报表** | SQL 查询构建器、定时报表、图表组件、CSV/PDF 导出 | "我们可以直接查数据库" |
| **审计日志** | 变更历史、差异查看器、谁在何时做了什么、日志保留 | 对终端用户不可见 |
| **系统健康** | 任务队列监控、错误追踪、性能指标、依赖状态 | 是运维问题，不是产品问题 |
| **配置** | 功能开关、系统设置、环境配置、维护模式 | 硬编码或环境变量 |
| **用户模拟** | 管理员以用户身份登录、以用户视角查看、调试会话 | 有安全风险，一开始回避 |

## 游戏 / 交互应用

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **成长体系** | 经验值、等级、技能树、成就系统、排行榜 | 核心功能，很少遗漏 |
| **经济系统** | 虚拟货币、道具市场、交易、通胀控制、回收机制 | 常被过度简化 |
| **社交** | 好友列表、公会/战队、聊天、在线状态、匹配 | 被推迟到"社区阶段" |
| **内容分发** | 资源包、补丁系统、DLC 管理、区域内容控制 | 假设"商店会处理" |
| **分析** | 事件追踪、漏斗分析、A/B 测试、玩家分群 | 第三方 SDK 似乎足够 |
| ** moderation** | 举报系统、自动过滤、玩家处罚、申诉流程 | "我们会手动 moderation" |
| **本地化** | 文本翻译、配音管理、文化适配、RTL 支持 | 上线时只做英文 |
| **存档与同步** | 云存档、跨设备同步、冲突解决、备份/恢复 | 假设本地存档足够 |
| **锦标赛** | 对阵表生成、赛程安排、排名算法、奖品分发 | 直到竞技模式才需要 |

## 移动应用

| 功能域 | 典型模块 | 常被遗漏的原因 |
|:---|:---|:---|
| **离线支持** | 本地数据缓存、同步队列、冲突解决、离线指示器 | "假设永远在线" |
| **推送通知** | Token 管理、富媒体通知、通知分类、深度链接 | 被推迟到" engagement 阶段" |
| **应用生命周期** | 引导流程、应用更新提示、强制更新、弃用下线 | 第一版中极简处理 |
| **设备集成** | 相机、GPS、通讯录、日历、生物识别、触觉反馈 | 功能特定，未通用化 |
| **应用商店** | 应用内购买、订阅管理、收据验证、促销优惠 | 复杂，被推迟 |
| **性能** | 崩溃报告、ANR 检测、内存分析、网络拦截 | 纯技术问题，非功能问题 |
| **无障碍** | 屏幕阅读器支持、动态字体、色彩对比度、语音控制 | "以后再加" |

## 跨领域通用项（适用于所有类型）

无论项目类型如何，都必须检查以下领域：

| 功能域 | 需检查的模块 |
|:---|:---|
| **可观测性** | 日志、指标、分布式追踪、告警、健康检查、SLI/SLO 仪表盘 |
| **安全** | 认证、授权、密钥管理、漏洞扫描、依赖审计 |
| **部署** | 环境配置、数据库迁移、功能开关、蓝绿部署、回滚 |
| **测试** | 单元测试、集成测试、端到端测试、负载测试、混沌工程、测试数据管理 |
| **文档** | API 文档、用户指南、运维手册、架构决策记录（ADR）、入职文档 |
| **数据治理** | 备份/恢复、数据归档、PII 处理、保留策略、匿名化 |

## 使用本参考的方法

1. 从用户回答或设计文档内容中识别项目类型。
2. 阅读上面对应的检查清单。
3. 对每一行，自问："设计文档是否明确覆盖了这项？"
4. 如果是，但未作为独立模块呈现，确保将其表示为一个独立模块。
5. 如果否，但项目明显需要它，添加一个 `🔄 行业补充` 模块。
6. 如果否，且项目明确排除它，在附录 A 中添加一项并说明理由。
7. 最后务必过一遍"跨领域通用项"清单——这些是最常被遗漏的。
