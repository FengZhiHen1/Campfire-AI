# Confirmation 问题设计示例

本文件存放 module-intent-writer 中各步骤的 `confirm_questions` 设计参考，供 Skill 执行时按需使用。

所有示例均使用 `PENDING_CONFIRM` 的 `confirm_questions` 字符串数组格式。

---

## 示例 1：模块选择 — 项目状态汇报与模块确认

```
项目当前状态：
- 已完成意图文档：3 个（M01 用户输入解析、M03 章节规划、M05 数据持久化），其中 M01/M03 已冻结
- 未处理：2 个（M02 世界观构建引擎、M04 人物设定生成）

confirm_questions: [
  "请从以下未处理模块中选择本次要处理的目标模块：\n- M02 世界观构建引擎（🔴 核心）\n- M04 人物设定生成（🟢 一般）\n\n或选择结束工作流。",
  "若需查看更多模块详情，请回复模块编号。"
]
```

## 示例 2：步骤 1 意图澄清 — 核心模块首轮澄清

```
调研背景：模块 M02（世界观构建引擎）对应总设计 3.2 节，上游为 M01，下游为 M03/M05。已审查契约索引，无冲突。以下为首轮业务决策点。

confirm_questions: [
  "【业务完整性】世界观构建的"完整性"如何定义？\n选项：\nA. 全维度强制生成 (Recommended) — 每次必须生成基础设定+文化细节+地理环境三个维度，不允许用户选择部分维度\nB. 允许用户选择维度 — 用户可勾选需要哪些维度，更灵活但实现更复杂",
  "【超时降级】当生成超时但已产出部分内容时，业务上应如何处理？\n选项：\nA. 展示草稿并让用户决定 (Recommended) — 用户可预览部分结果，选择继续等待或接受草稿\nB. 直接提示失败 — 用户看不到任何结果，体验较差但逻辑简单\nC. 自动后台重试 — 自动重新生成，用户无感知，但可能产生额外成本",
  "【用户编辑】世界观生成结果是否允许用户编辑后再保存？\n选项：\nA. 允许编辑后保存 (Recommended) — 用户可修改 AI 生成的内容再保存，更实用\nB. 仅允许接受或重新生成 — 不允许编辑，用户只能选择用或不用"
]
```

## 示例 3：步骤 1 意图澄清 — 核心模块后续轮次澄清

```
已确认：全维度强制生成、允许编辑后保存、超时展示草稿。\n\n本轮聚焦剩余未澄清维度：

confirm_questions: [
  "【验收标准量化】世界观生成的一致性率指标应设定为多少？\n选项：\nA. >= 95% (Recommended) — 与题材和风格标签高度一致\nB. >= 90% — 适度放宽，允许一定创意偏离\nC. 不设定具体数值，以人工审核为准",
  "【状态机】世界观构建是否需要业务状态流转？\n选项：\nA. 需要 — 空闲→构建中→已完成/失败，含草稿降级状态 (Recommended)\nB. 不需要 — 简单的同步请求-响应模式即可"
]
```

## 示例 4：步骤 1 意图澄清 — 一般模块快速确认

```
经过自检，模块 M08（用户登录）的以下维度已确认清晰：\n- 业务边界：标准账号密码登录 + 第三方 OAuth 登录\n- 输入/输出：用户名/密码、OAuth 授权码 → 会话令牌（业务含义明确）\n- 业务规则：使用行业默认安全规则（密码强度、Token 过期）\n- 状态机：无复杂异步流程\n- 兼容性：与已有用户模块无冲突

confirm_questions: [
  "本模块标记为一般功能模块（🟢），业务需求已足够清晰。是否直接生成意图文档 `M08-用户登录-意图文档.md`？\n选项：\nA. 直接生成 (Recommended) — 基于现有设计文档直接输出意图文档，不再追加问答\nB. 切换到全量澄清 — 认为该模块需要更严格的需求对齐，进入核心模块的多轮问答流程"
]
```

## 示例 5：步骤 2 书写授权

```
经过 3 轮问答，本模块的所有关键业务决策点已明确：\n\n- 业务边界：接收上游 M01 的题材/风格标签，产出世界观设定给下游 M03/M05\n- 输入/输出：题材类型（必填，枚举值）、风格标签（可选）、世界观文本（三维度）\n- 状态机：空闲→构建中→已完成/失败，含草稿降级\n- 业务规则：全维度强制生成、允许用户编辑后保存\n- 验收标准：一致性率 >= 95%、完整性三维度覆盖率 >= 95%\n- 留给规范阶段：具体超时秒数、重试次数、Pydantic 类型定义

confirm_questions: [
  "所有业务需求已澄清完毕。是否授权开始生成意图文档 `M02-世界观构建引擎-意图文档.md`？\n选项：\nA. 开始生成 — 立即进入文档生成步骤 (Recommended)\nB. 还需要补充 — 还有未覆盖的决策点，回到意图澄清步骤继续提问"
]
```

## 示例 6：步骤 4 冻结授权

```
意图文档已生成。请确认以下核心内容是否已锁定：\n\n- 业务边界：接收上游 M01 的题材/风格标签，产出世界观设定给下游 M03/M05\n- 验收标准：一致性率 >= 95%、完整性三维度覆盖率 >= 95%、用户可编辑后保存\n- 关键业务规则：全维度强制生成、超时展示草稿让用户决定\n\n留给规范阶段的技术决策已明确列出：超时秒数、重试策略、Pydantic 类型、状态机技术实现。

confirm_questions: [
  "冻结后，本文档将作为后续技术规格编写的强制输入，不可擅自修改。如需修改，必须通过回退机制重新解冻。是否确认冻结？\n选项：\nA. 冻结确认 (Recommended) — 意图文档已锁定，标记为冻结状态，允许进入规范编写阶段\nB. 还需要修改 — 意图文档还有需要调整的地方，返回修改"
]
```

---

## 示例 7：步骤 4 冻结授权 — 修改后重新冻结

```
已根据您的意见修改文档：\n- 调整了验收标准中一致性率的具体计算方式\n- 补充了异常场景 3：用户取消构建的业务策略

confirm_questions: [
  "文档已根据您的意见修改。请确认修改后的内容，并重新授权冻结。\n选项：\nA. 冻结确认 (Recommended) — 修改后的意图文档已锁定，重新标记为冻结状态\nB. 还需要修改 — 继续调整"
]
```
