### 1.1 项目定位

反向代理路由是篝火智答系统的网关层组件，位于微信小程序客户端与 FastAPI 应用层之间。它承担三项核心职责：作为 HTTPS 终端对外提供加密通信、将 API 请求路由转发至后端应用服务、以及分发静态资源。在整体架构中，它是外部流量进入系统的唯一入口——所有客户端请求必须经过反向代理才能到达业务服务，因此其可用性直接影响系统整体的可访问性。

### 1.4 业务目标

1. 所有客户端（微信小程序）发起的 HTTPS 请求能够被正确接收并代理转发至后端 API 服务，代理成功率 >= 99.9%
2. API 路由 `/api/v1/*` 的请求在 100ms 内完成从 Nginx 到 FastAPI 的代理转发（不含上游处理时间）
3. 生产环境强制启用 TLS 1.3 加密，不允许降级至更低版本的 TLS 协议或明文 HTTP
4. SSL 证书通过 Let's Encrypt 自动续期，确保证书过期前自动完成更新，全年因证书过期导致的服务中断时间为零
5. 静态资源（如有）启用 7 天缓存策略并通过 gzip 压缩传输，减少带宽消耗

### 1.5 用户旅程 / 业务流程

1. **用户发起请求**：用户在微信小程序中触发操作（如提交咨询、查询档案），小程序向服务器域名发起 HTTPS 请求
2. **Nginx 接收请求**：Nginx 在 443 端口接收加密的 HTTPS 请求，完成 TLS 握手后解密为明文 HTTP 请求
3. **路由匹配与代理转发**：Nginx 根据请求路径匹配路由规则，将 `/api/v1/*` 请求代理转发至 FastAPI 容器，同时透传客户端 IP、请求来源等必要信息
4. **后端处理与响应**：FastAPI 处理业务逻辑后返回响应，Nginx 将响应透传回客户端
5. **SSE 流式推送**：对于应急咨询等需要流式响应的场景，Nginx 关闭缓冲功能，将 SSE 事件流实时透传给小程序客户端
6. **健康检查**：Nginx 将 `/health` 路径直接转发至 FastAPI 的健康检查端点，供 Docker 容器编排和服务发现使用

### 1.6 输入/输出业务定义

#### 1.6.1 输入

| 字段名 | 业务含义 | 是否必填 | 业务约束 | 示例值 |
|--------|---------|---------|---------|--------|
| 客户端请求 | 用户通过微信小程序发起的 HTTPS API 请求或静态资源请求 | 是 | 必须遵循 HTTPS 协议（生产环境）或 HTTP 协议（开发环境） | `GET https://campfire-ai.example.com/api/v1/consult/stream` |
| 域名 | 客户端请求访问的服务域名 | 是 | 必须指向部署服务器的公网 IP | `campfire-ai.example.com` |
| 请求路径 | 客户端请求的资源路径 | 是 | 系统定义的所有 API 路由及静态资源路径 | `/api/v1/consult/stream` |
| 请求方法 | HTTP 请求方法 | 是 | 常见方法：GET、POST、PUT、DELETE | `POST` |
| 请求头 | 客户端随请求携带的头部信息 | 否 | 包含 Host、Content-Type、Authorization 等标准头部 | `Authorization: Bearer <token>` |
| SSE 流式标记 | 标记是否为流式响应请求 | 否 | 仅当需要 SSE 流式推送时启用 | `Accept: text/event-stream` |

#### 1.6.2 输出

| 字段名 | 业务含义 | 业务约束 | 示例值 |
|--------|---------|---------|--------|
| 代理响应 | Nginx 从后端服务获取后返回给客户端的响应内容 | 透传 FastAPI 响应，不修改响应体内容 | HTTP 200 JSON 响应体或 SSE 事件流 |
| 响应状态码 | 表示请求处理结果的状态码 | 200（成功）、4xx（客户端错误）、5xx（服务端错误）等标准 HTTP 状态码 | `200 OK` |
| 响应头 | 服务器返回给客户端的头部信息 | 包含 Content-Type、Cache-Control 等标准头部 | `Content-Type: application/json` |

### 1.7 状态的业务定义（如适用）

本功能点不涉及状态流转，故无需状态机。反向代理路由为纯请求-响应模式，每次请求独立处理，不维护请求之间的状态关系。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：后端服务不可用
- **业务触发条件**：FastAPI 或 Worker 服务因故障、重启或维护而无法响应 Nginx 的代理请求
- **业务处理策略**：
  1. Nginx 向上游客户端返回 502 Bad Gateway 错误页面
  2. 运维人员通过健康检查和容器编排的自动重启策略恢复服务
  3. 记录异常发生时间和频率，供后续运维分析
- **业务影响**：用户无法使用系统核心功能，直至后端服务恢复

#### 1.8.2 异常 2：SSL 证书过期或无效
- **业务触发条件**：Let's Encrypt 自动续期失败导致 SSL 证书过期，或证书域名与实际域名不匹配
- **业务处理策略**：
  1. 客户端在尝试建立 HTTPS 连接时收到浏览器/小程序的安全警告
  2. 自动续期失败时应通过监控告警通道（钉钉/企业微信）即时通知运维人员
  3. 运维人员手动执行证书续期或替换为有效证书后重启 Nginx 服务
- **业务影响**：所有外部 HTTPS 请求被客户端安全策略拦截，系统完全不可通过公网访问

#### 1.8.3 异常 3：流量过载
- **业务触发条件**：瞬时并发请求量超过 Nginx 配置的 worker_connections 或后端服务承载能力
- **业务处理策略**：
  1. 超出连接上限的请求被 Nginx 拒绝，返回 503 Service Unavailable
  2. 系统应配合 DEPLOY-01（容器编排）的自动扩缩容机制增加后端副本
  3. 压测阶段确定合理的 worker_connections 和 keepalive 参数，避免资源耗尽
- **业务影响**：部分用户的请求被拒绝，但已建立的连接不受影响

### 1.9 验收标准

| 编号 | 验收条件 | 验收方式 | 通过标准 |
|------|---------|---------|---------|
| AC-01 | HTTPS 加密通信 | 安全扫描 | 生产环境强制启用 TLS 1.3，不支持 SSLv3/TLSv1.0/TLSv1.1/TLSv1.2 降级 |
| AC-02 | API 代理转发 | 集成测试 | `/api/v1/*` 路径的请求成功转发至 FastAPI 容器且响应内容完整无修改 |
| AC-03 | 静态资源缓存 | 性能测试 | 静态资源响应头包含 `Cache-Control: public, immutable` 且 `max-age` 为 7 天 |
| AC-04 | Gzip 压缩 | 性能测试 | 文本类响应（JSON、CSS、JS）启用 gzip 压缩，压缩率 >= 60% |
| AC-05 | SSL 证书自动续期 | 长期监控 | 全年因证书过期导致的服务中断次数为零，续期过程无需人工介入 |
| AC-06 | 健康检查转发 | 集成测试 | `/health` 路径请求成功转发至 FastAPI 健康检查端点并返回正确状态 |
| AC-07 | SSE 流式支持 | 集成测试 | SSE 流式请求的响应在 Nginx 层不被缓冲，客户端逐 Token 接收无延迟堆积 |
| AC-08 | 后端不可用处理 | 故障注入测试 | 后端服务停止后，Nginx 向客户端返回 502 错误而非超时挂起 |

### 1.10 业务依赖关系

| 依赖方 | 关系类型 | 业务交互说明 |
|--------|---------|-------------|
| 微信小程序客户端 | 上游数据来源 | 所有用户操作产生的 HTTPS 请求通过反向代理进入系统 |
| FastAPI 应用层（api-server） | 下游代理目标 | Nginx 将 `/api/v1/*` 请求代理转发至 FastAPI 容器处理 |
| DEPLOY-01（容器编排） | 部署协同 | Nginx 被 Docker Compose 编排在同一个 Docker 网络中，通过服务名访问后端容器 |
| DEPLOY-05（环境配置管理） | 配置供给 | SSL 证书路径、服务域名、代理目标地址等配置由环境配置管理模块提供 |
| Let's Encrypt（外部服务） | 证书供给 | 提供免费 SSL 证书的自动签发与续期能力 |

### 1.11 业务约束清单

1. **[传输安全]** 生产环境必须启用 HTTPS 且仅支持 TLS 1.3 协议，禁止明文 HTTP 传输业务数据
2. **[合规要求]** SSL 证书必须使用权威 CA 签发（Let's Encrypt 或类似受信任的 CA），禁止使用自签名证书
3. **[服务可用性]** Nginx 容器重启策略为 unless-stopped，确保异常退出后自动恢复
4. **[缓存策略]** 静态资源缓存时间至少 7 天且须启用 gzip 压缩，最小压缩体积为 256 字节
5. **[设计边界]** 本模块不负责请求级别的身份认证和授权校验（该职责归属 AUTH-04 五级 RBAC 鉴权）、不负责 API 级别的限流控制（该职责归属 SEC-04 防刷限流）
6. **[设计边界]** 本模块不负责 SSL 证书的签发和申请（该职责由 Let's Encrypt 客户端或运维人员手动执行），只负责加载和使用已签发的证书
