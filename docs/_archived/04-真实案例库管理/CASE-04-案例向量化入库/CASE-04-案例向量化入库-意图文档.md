### 1.1 项目定位

案例向量化入库是真实案例库管理分组的核心数据流水线模块。它负责将审核通过的结构化干预案例卡片转化为可供智能检索的向量化知识单元，写入项目统一检索索引。在整体业务流程中，它是连接案例审核产出与 RAG 检索消费的关键中间环节——上游接收专家审核确认的专业干预方案，下游向智能应急咨询提供可溯源的语义匹配素材。向量化质量直接决定检索的精准度，进而影响应急方案生成的可执行性。

### 1.4 业务目标

1. 所有经审核通过的案例卡片，在审核通过后自动触发向量化入库流程，无需人工干预
2. 每条案例卡片的向量化切片保持四要素（场景、行为、干预、结果）语义完整性，不可跨切片拆分
3. 向量化入库的可用性目标为 99.5%，即 1000 条审核通过的案例中不超过 5 条因系统原因索引失败
4. 索引失败的案例在管理端明确可见且异常标记清晰，运营人员可通过手动操作重新触发索引

### 1.5 用户旅程 / 业务流程

1. **审核通过触发**：专家在审核界面点击"通过"审核后，案例审核模块自动向向量化入库模块发送索引入库任务
2. **异步队列等待**：索引入库任务进入后台队列，等待系统处理。此时案例仍保持"已审核"身份，管理端显示"索引中"状态标记
3. **向量化处理**：系统后台自动执行文本切片组装、向量嵌入生成和索引写入。处理过程对前端用户透明，不阻塞审核操作
4. **入库完成**：向量写入成功后，案例管理端状态更新为"已索引入库"，意味着该案例可供智能应急咨询的 RAG 检索消费
5. **异常处理**：若向量化连续失败，案例保持"已审核"但标记为"索引异常"，不参与 RAG 检索。运营人员在管理端收到异常提示后，可手动重新触发索引流程

### 1.7 状态的业务定义

案例向量化入库涉及 4 个业务状态：

- **队列等待**：审核通过后，索引入库任务已成功投递至后台队列，等待系统分配处理资源。此时案例管理端显示"索引中"。
- **处理中**：系统正在对该案例卡片执行文本组装、向量嵌入生成和索引写入操作。处理过程对最终用户透明。
- **已入库**：向量化全部完成且已成功写入检索索引，该案例可供 RAG 语义检索消费。这是正常流程的终态。
- **索引异常**：向量化入库连续失败（系统自动重试未恢复），标记为异常。案例保持已审核身份，但不参与 RAG 检索。运营人员可在管理端手动触发重新索引。

进入"队列等待"的条件：案例审核模块判定审核通过并成功投递索引任务。
退出"已入库"的条件：向量写入检索索引成功并确认可被检索。
进入"索引异常"的条件：系统自动重试次数用尽后仍未成功。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：向量嵌入生成失败

- **业务触发条件**：系统调用嵌入服务生成文本向量时，因外部服务不可用或请求超时等原因无法在合理时间内获得向量结果
- **业务处理策略**：
  1. 系统自动重试，最多再尝试 2 次
  2. 若 3 次尝试均失败，标记该案例卡片为"索引异常"
  3. 案例保持已审核身份，已有审核结果不受影响，但暂不参与 RAG 检索
  4. 运营人员在管理端看到异常标记后，可手动重新触发整个向量化流程
- **业务影响**：该案例暂时无法被智能应急咨询检索到，但不影响其他已入库案例的检索和生成

#### 1.8.2 异常 2：检索索引写入失败

- **业务触发条件**：向量已生成，但在写入检索索引时因索引服务异常或存储空间不足等原因无法完成持久化
- **业务处理策略**：
  1. 系统自动重试写入操作，最多再尝试 2 次
  2. 若 3 次尝试均失败，标记为"索引异常"
  3. 案例保持已审核身份，管理端显示异常标记
  4. 运营人员可根据异常详情决定直接重试或联系运维排查底层服务
- **业务影响**：已生成的向量丢失需重新生成，案例暂不参与 RAG 检索

#### 1.8.3 异常 3：文本切片内容异常

- **业务触发条件**：案例卡片的四段式字段中存在空值或内容长度异常（如某字段为空字符串、所有字段合计内容不足最低业务长度要求）
- **业务处理策略**：
  1. 在向量化处理前进行文本完整性校验
  2. 若任何四要素字段为空，拒绝进入向量化流程，标记为"索引异常"
  3. 在管理端异常详情中明确指出缺失的具体字段，引导运营人员联系专家补充后重新提交审核
- **业务影响**：不完整的案例卡片不会产生低质量的向量切片，保证检索索引质量的底线

### 1.11 业务约束清单

1. **四要素强制绑定**：每条案例卡片的四要素（场景、行为、干预、结果）必须在同一切片中完整保留，不可因文本长度或技术原因将其拆分到不同切片中。这是保证 RAG 检索结果语义完整性的核心约束。
2. **审核通过为唯一触发源**：只有经专家终审通过的案例卡片才能进入向量化入库流程。草稿状态、审核中状态、已驳回状态的案例一律不触发索引。
3. **索引失败不影响审核结果**：案例向量化入库失败不应导致审核结果被撤销或回退。专家的专业审核工作成果必须被保留，索引异常仅影响检索可用性。
4. **个人隐私数据不入索引**：所有进入向量化流程的案例文本必须在案例提交阶段已完成 PII 脱敏处理。本模块不额外执行脱敏操作，但必须在处理前校验文本中是否仍包含可识别的个人信息。
5. **免责声明保持完整**：案例卡片中的"AI 建议仅供参考，不构成医疗诊断或治疗建议"声明在向量化过程中必须完整保留，不可因文本处理而截断或丢失。
6. **设计边界——本模块不负责的业务**：本模块不负责案例审核决策、案例录入编辑、RAG 检索策略调优、案例淘汰标记管理。这些职责分别归属 CASE-03、CASE-01、CSLT-02、CASE-06 模块。
