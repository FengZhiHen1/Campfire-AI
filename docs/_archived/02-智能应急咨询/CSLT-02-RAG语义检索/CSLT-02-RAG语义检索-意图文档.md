### 1.1 项目定位

RAG语义检索是篝火智答应急咨询核心链路中的语义匹配引擎。它接收用户的紧急行为描述与患者个性化档案标签，从真实案例库中检索出语义最相关且符合档案标签条件的干预案例切片。在整体流程中，它是一个检索增强节点——上游是档案过滤条件的输入与用户问题的提交，下游是应急方案生成，它为下游提供真实有效的参考案例上下文，直接决定了AI应急方案的可执行性与精准度。

### 1.4 业务目标

1. 家属提交行为描述与患者档案后，系统在500毫秒内检索出语义最匹配的干预案例切片，返回给应急方案生成模块
2. 检索结果必须100%符合患者档案标签的条件过滤（在降级放宽之前）
3. 检索出的案例切片必须是已审核通过且未被淘汰的真实案例，不得包含来源不可靠的内容
4. 当精确过滤结果不足默认10条时，允许在可控范围内逐层放宽条件以扩大召回量
5. 检索结果的排序优先为最近的有效案例和循证等级最高的案例，确保下游生成参考最新的专业意见

### 1.5 用户旅程 / 业务流程

本模块为后台服务模块，无面向终端用户的直接交互界面。以下从系统内部业务流程描述其工作场景：

1. **检索触发**：家属在应急咨询界面提交行为描述后，系统上游模块（CSLT-01危机分级判定和PROF-02档案驱动检索过滤）分别产出危机等级和标签过滤条件，二者与用户原始行为描述文本一并传入本模块
2. **执行检索**：本模块将用户行为描述转化为语义查询，携带档案标签条件在案例向量索引中执行混合检索——先用标签条件精确筛选候选集，再对候选集按语义相似度排序，同时排除已标记为失效的案例
3. **降级处理（如需要）**：若精确过滤后匹配的案例切片数量不足默认返回量（10条），系统按预定顺序逐层放宽标签条件并重新检索，在保障质量的前提下扩大召回范围
4. **结果输出**：将最终排序的案例切片列表（含切片文本、来源案例标识、相似度分数、循证等级、录入时间）输出给下游CSLT-03，供其组装为Prompt中的参考案例上下文

### 1.7 状态的业务定义

本功能点不涉及状态流转，故无需状态机。每次检索为独立的同步请求-响应操作。检索过程中的内部状态（检索中、超时、完成/部分完成）通过输出字段隐式传递——"是否完整检索"字段为"部分"时表示触发了超时截断，下游可据此做出相应的业务提示。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：检索结果不足期望数量

- **业务触发条件**：应用标签精确过滤后，匹配的案例切片数量少于期望返回数量（默认10条）
- **业务处理策略**：
  1. 按预设顺序逐层放宽标签过滤条件：先放宽情绪等级（如从"重度"放宽为"重度+中度+轻度"，扩大匹配范围），仍不足则放宽行为类型（如从"情绪崩溃"放宽为包含"情绪崩溃+刻板+攻击"），仍不足则移除所有标签条件使用纯语义检索
  2. 每层放宽后重新检索，一旦达到期望数量即停止继续放宽
  3. 若三层放宽后仍不足期望数量，返回所有实际检索到的结果（可能为0条）
  4. 在结果中标注"实际检索到N条"，以便下游模块感知并调整生成策略
- **业务影响**：检索结果少于期望数量时，下游生成的应急方案可参考的案例背景减少，方案质量可能降低。结果为空时，下游需依赖通用知识生成方案（不含具体案例参考）

#### 1.8.2 异常 2：检索耗时超过500毫秒

- **业务触发条件**：从检索开始到指定时间点（500毫秒）时，检索尚未返回期望数量的结果
- **业务处理策略**：
  1. 立即将已检索到的案例切片（可能少于期望数量）返回给下游，不等待剩余结果
  2. 将"是否完整检索"标记为"部分"，"检索耗时"字段记录实际耗时
  3. 如果有部分结果（数量 > 0），下游正常使用；如果超时且结果为0条，标记为"检索未完成"，下游模块可提示用户"系统正在查找匹配案例，请稍候"或使用无案例的降级生成策略
- **业务影响**：超时但返回部分结果时，下游仍可基于已有案例生成方案，只是可参考的案例数量减少。超时且零结果时，下游无法参考任何案例，必须采用纯通用策略生成方案

#### 1.8.3 异常 3：案例库中无可检索的案例

- **业务触发条件**：案例向量索引中不存在任何处于"已审核通过"状态且未被淘汰标记的案例切片（可能发生在系统刚上线、案例库完全为空时）
- **业务处理策略**：
  1. 返回空的案例切片列表，附带明确标记"案例库暂无可用案例"
  2. 不尝试放宽任何过滤条件（因为问题在于案例库本身为空，而非过滤条件过严）
  3. 下游模块感知此状态后，使用不包含案例参考的通用应急方案生成策略
- **业务影响**：应急方案将完全依赖于LLM通用知识，缺少个性化案例支持，生成的方案是一般性建议而非基于真实案例的专业指导。系统应向家属额外提示"当前为通用建议，完善案例库后可获得更精准的指导"

### 1.9 验收标准

| 编号 | 验收条件 | 验收方式 | 通过标准 |
|------|---------|---------|---------|
| AC-01 | 标签过滤有效性 | 测试用例 | 传入明确的标签条件（如行为类型=情绪崩溃，情绪等级=重度），在未触发降级放宽前的检索结果中，100%的案例切片符合该标签条件 |
| AC-02 | 语义检索相关性 | 人工抽检 | 随机抽取20次检索，Top-3结果中至少2条与用户行为描述存在明确的语义关联（由抽检人员判断） |
| AC-03 | 500毫秒时延保障 | 性能测试 | 在标准数据规模下（案例库不超过10万切片），95分位检索耗时 ≤ 500毫秒 |
| AC-04 | 逐层降级机制 | 测试用例 | 构造精确过滤结果为0的测试场景，验证系统按情绪等级→行为类型→移除标签的顺序逐层放宽并返回结果 |
| AC-05 | 失效案例排除 | 测试用例 | 案例库中存在被标记为过时/有误/争议/强制下架的案例切片时，所有检索结果中均不包含这些切片 |
| AC-06 | 时效排序正确性 | 测试用例 | 当两条切片语义相似度基本相同时（分数差不超过0.05），录入时间更新（时效更高）的切片排在更前面 |
| AC-07 | 超时部分结果返回 | 场景测试 | 模拟检索缓慢场景（耗时超过500毫秒），验证系统在500毫秒时立即返回已有结果，且返回结果数量大于0 |
| AC-08 | 空案例库处理 | 测试用例 | 案例库完全为空时，检索返回空列表且附带明确的状态标记，不抛出业务异常 |

### 1.11 业务约束清单

1. **检索质量一致性**：在相同的用户行为描述和档案标签条件下，每次检索返回的排名结果必须基本一致（前3条不应有显著随机波动，允许因新案例入库导致的新增变动）
2. **时效衰减不可逆**：案例因时效降低权重的规则为硬性约束——0至1年权重系数1.0，1至3年权重系数0.7，3年以上权重系数0.5。该衰减规则全文统一生效，不可因特定案例或场景豁免
3. **隐私数据不进入检索**：本模块的检索查询文本中不得包含患者真实姓名、身份证号、手机号码、家庭住址、学校名称等PII信息——该约束由上游模块（SEC-03 PII检测脱敏）在查询传入前保证，本模块在执行检索时仅关注查询内容的语义特征
4. **案例淘汰无条件优先**：任何被CASE-06标记为失效状态（过时/有误/争议/强制下架）的案例切片，在本模块检索结果中绝对不可出现。即使该切片是所有候选项中语义相似度最高的，也必须排除。这是一条不可逾越的安全红线
5. **不当医疗建议零容忍**：检索结果中不得出现来源标注为"AI自动生成且未经专家审核"的案例切片（该类切片在CASE-04入库阶段已被拦截）。所有可检索的案例必须经过专业审核流程。这是确保平台不负医疗责任的核心防线
