### 1.1 项目定位

危机分级判定模块是「篝火智答」应急咨询业务流程的第一道安全防线。家属在应急时刻打开平台输入患者当前的行为表现后，系统通过三层递进判定机制——前置行为类型选择、规则引擎关键词匹配、大模型精调复审——对当前场景的危机严重程度做出三级评定（轻度/中度/重度）。判定结果直接决定 AI 是否深度回答、页面是否进入应急模式、以及是否自动触发人工工单及其优先级。

在整体业务流程中，危机分级判定处于应急咨询的最前端：位于用户输入行为描述之后、RAG 应急建议生成之前。它是区分「AI 可自主应对」与「必须人工介入」的关键决策点，也是平台安全承诺的核心体现。

### 1.4 业务目标

1. 每次应急咨询启动时，系统在可接受时间内完成危机等级判定并输出给下游模块消费
2. 重度场景（涉及自伤、攻击、逃跑、用药）的判定召回率必须达到 100%，不允许漏判
3. 否定词过滤功能不得误判含高危关键词但语义安全的输入（如"没有自伤行为"），误判率目标为零
4. 三层递进判定机制的最终输出必须无条件遵守"宁升勿降"合并策略，任何一层判重度即为重度
5. 重度判定后必须在 200 毫秒内完成前端应急模式切换，AI 深度回答必须完全阻断
6. LLM 精调复审的超时降级机制必须保证在 LLM 不可用时系统仍可依靠规则引擎独立运行

### 1.5 用户旅程 / 业务流程

1. **家属输入行为描述**：家属在应急咨询界面输入患者当前的行为表现文字描述（如"孩子在商场突然捂住耳朵蹲在地上，怎么拉都不走"）。

2. **前置行为类型选择**：系统弹出行为类型选择窗口（含 7 类选项），家属勾选最接近的场景。若勾选的是自伤/攻击/逃跑/用药四种高危类型中的任意一项，系统立即判定为重度，页面切换为红色应急模式，仅展示安全提示。

3. **规则引擎关键词匹配**（前置选择未命中高危时执行）：系统以零延迟匹配输入文本中的关键词。若命中重度关键词（如撞墙、撞头、割腕等）且未通过否定词过滤，则直接判定为重度；若命中档案叠加规则，则判定为中度并附加人工复核标记。

4. **RAG 应急建议生成**（规则引擎未命中重度时执行）：系统正常检索案例库并生成四段式应急建议。页面顶部显示"系统正在复核安全风险"提示条。

5. **LLM 精调复审**（与步骤 4 并行执行）：大模型在 3 秒内完成复审，若复审发现应升级等级，则页面平滑切换为应急模式并覆盖步骤 4 的展示；若复审维持或超时，则提示条消失，保持原展示。

6. **结果消费**：最终危机等级传递给下游模块——决定工单是否创建及其优先级，同时作为应急方案生成的策略参数。

### 1.7 状态的业务定义

危机分级判定本身是无状态判定——每次应急咨询独立执行三层判定，不维护跨咨询会话的状态流转。单次判定的内部流程可描述为以下阶段，但不构成持久化状态机：

- **等待输入**：家属尚未完成行为类型选择和文字描述输入
- **前置判定**：家属已完成前置行为类型勾选，正在执行高危类型直接判定
- **规则匹配**：规则引擎正在对输入文本执行关键词匹配（0 毫秒同步操作）
- **并行复审**：规则引擎未命中重度，RAG 建议生成与 LLM 复审并行执行
- **合并且输出**：综合各层判定结果按判定矩阵输出最终等级

> 本功能点不涉及持久化状态流转，故无需状态机。前端页面状态流转（正常模式→应急模式）属于 CSLT-07（应急咨询界面）的职责。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：LLM 精调复审超时

- **业务触发条件**：LLM 精调复审在约定时限内未返回结果
- **业务处理策略**：
  1. 默认采纳规则引擎的初审结果作为最终等级
  2. 页面提示条正常消失，RAG 生成的应急建议正常展示
  3. 超时后 LLM 返回的结果仅记入日志供后续分析，不改变当前展示
- **业务影响**：用户体验不受影响，RAG 建议照常阅读；超时日志数据用于后续词库和策略迭代

#### 1.8.2 异常 2：关键词词库加载失败

- **业务触发条件**：规则引擎启动时或运行时关键词词库配置不可用
- **业务处理策略**：
  1. 降级为仅依赖前置行为类型选择判定（第零层仍完整可用）
  2. 前端正常展示行为类型选择窗口，规则引擎层跳过
  3. 在判定结果中标注"规则引擎降级"
  4. 所有输入自动进入 LLM 复审层（即使第零层未命中高危）
- **业务影响**：安全保障降低一层，但前置行为类型选择的第零层防护仍完整可用；需运维人员尽快修复词库

#### 1.8.3 异常 3：患者档案上下文缺失

- **业务触发条件**：家属尚未创建任何患者档案即进入应急咨询
- **业务处理策略**：
  1. 规则引擎和 LLM 复审仍正常执行（不受档案缺失影响）
  2. 档案叠加规则在无档案时自动跳过，不触发复核标记
  3. 判定结果标注"档案缺失"，提示家属后续补充档案以获得更精准的个性化判定
- **业务影响**：判定功能核心链路不受影响，但个性化精度下降

| AC-07 | 最终判定矩阵 | 测试用例 | 全部 7 种规则初审×LLM 复审组合均输出正确的最终等级，宁升勿降原则无违反，LLM 不可降级规则引擎 |

### 1.11 业务约束清单

1. **安全底线不可突破**：涉及身体伤害、走失、用药等场景，系统必须强制阻断 AI 深度回答并自动转人工。不允许任何配置或降级路径绕过此约束。
2. **宁过度拦截，不漏风险**：规则引擎的关键词收录宁可扩大范围，也不允许真实重度场景漏判为轻度。判定矩阵严格执行"任何一方判重度即为重度"。
3. **家属参与关键判定**：将"是否涉及身体安全"的初始判断通过前置行为类型选择窗口交给家属，降低模型黑盒风险。高危类型的前置判定不可被规则引擎或 LLM 覆盖为更低等级。
4. **合规要求**：所有 AI 输出必须附带"不构成医疗诊断或治疗建议"免责声明；规则引擎重度阻断后输出的安全提示必须包含"建议立即联系专业医疗机构"的引导。
5. **响应时效要求**：前置选择判定 200 毫秒内反馈；关键词匹配 0 毫秒级别完成；LLM 复审 3 秒超时降级。危机时刻每一毫秒的延迟都可能影响安全——响应时效不得因非安全相关的功能需求而被牺牲。
6. **降级路径保证**：关键词词库加载失败时，前置选择层必须保持完整可用。LLM 服务不可用时，规则引擎必须独立完成判定，系统整体判定功能不可完全失效。
7. **设计边界**：本模块不负责应急方案文本的生成（归属 CSLT-03）、不负责前端页面的 UI 渲染与状态切换（归属 CSLT-07）、不负责工单的创建与派发逻辑（归属 TICK-01/TICK-02）、不负责 AI 输出内容的安全审核（归属 SEC-02）。
