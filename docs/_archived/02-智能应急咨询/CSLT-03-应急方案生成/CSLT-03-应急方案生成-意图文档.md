### 1.1 项目定位

应急方案生成是篝火智答应急咨询核心链路中的内容生成节点。它接收上游 RAG 检索返回的真实干预案例切片、患者个性化档案摘要和危机等级判定结果，将其组装为 Prompt 后调用大模型，生成面向家属的四段式结构化应急建议。在整体业务流程中，它位于危机分级判定与案例检索之后、流式推送与置信度后校验之前——是将检索到的专业知识转化为家属可执行的行动指南的关键环节。

### 1.4 业务目标

1. 家属提交行为描述后，系统在可接受时间内组装 Prompt 并生成完整的四段式应急方案，首字延迟不超过 3 秒
2. 每次生成的应急方案必须覆盖即时安全干预动作、情绪安抚话术、后续观察指标和就医判断标准四个段落，缺一不可
3. 每条参考案例建议必须标注来源序号，全文底部统一展示来源清单，引用准确率不低于 90%
4. 免责声明必须在每次生成结果中完整呈现，不得出现缺失或被生成内容覆盖的情况
5. 危机等级为重度且阻断标记激活时，必须严格阻断深度回答，仅输出预设安全提示文本
6. 上游无参考案例时，必须明确告知用户当前无可参考案例，引导其咨询专业人士

### 1.5 用户旅程 / 业务流程

本模块为后台服务模块，无面向终端用户的直接交互界面。以下从系统内部业务流程描述其工作场景：

1. **接收生成请求**：家属在应急咨询界面提交行为描述后，上游链路（CSLT-01 危机分级判定 + CSLT-02 RAG语义检索）完成判定与检索，将案例切片列表、危机等级与阻断标记、患者档案摘要连同用户原始行为描述文本，一并传入本模块
2. **Prompt 组装**：本模块将输入材料按预设顺序编排为 Prompt——系统指令包含四段式输出要求、免责声明模板、来源引用规则和专业术语解释规则；上下文包含用户行为描述、患者档案摘要和检索到的案例切片列表（含切片文本与来源标识）
3. **生成与流式输出**：调用大模型生成应急方案，将生成的文本逐段传递至下游 CSLT-04 进行流式推送。若 CSLT-01 传递的阻断深度回答标记为真，则跳过生成环节，直接输出预设的安全提示文本
4. **结果交付**：生成完成后，将完整应急方案（四段式文本 + 来源引用清单 + 免责声明）传递给 CSLT-05 进行置信度后校验
5. **重新生成（如用户触发）**：生成完成后，用户可在前端点击「重新生成」按钮，系统在 30 秒冷却期后接受请求，基于同一输入重新执行生成流程

### 1.7 状态的业务定义

本模块为无状态生成服务——每次应急咨询独立执行 Prompt 组装与 LLM 调用，不维护跨咨询会话的持久状态。单次生成流程的内部阶段如下：

- **等待输入**：尚未接收到上游的完整输入数据，处于空闲状态
- **组装中**：正在将输入材料编排为 Prompt，尚未发起 LLM 调用
- **生成中**：已向大模型发起请求，正在等待文本产出
- **已完成**：四段式应急方案已完整生成并输出给下游
- **阻断输出**：危机阻断标记激活，跳过深度生成，直接输出预设安全提示

本功能点不涉及持久化状态流转，故无需状态机。前端交互状态（生成中/已完成/失败/冷却中）属于 CSLT-08（咨询编排逻辑）的职责范围。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：上游案例切片列表为空

- **业务触发条件**：上游 CSLT-02 降级放宽后仍无法检索到任何匹配的案例切片（如罕见行为类型无对应案例记录）
- **业务处理策略**：
  1. 在 Prompt 中注入「当前无可参考的干预案例」提示，要求 LLM 基于通用专业知识生成建议
  2. 在生成的应急方案首段明确告知用户：「当前暂无与您描述情况相匹配的真实干预案例参考，以下建议基于通用专业知识生成」
  3. 在全文中不标注任何案例来源序号（因为无案例可引用）
  4. 来源引用清单输出为「无」并说明原因
  5. 在方案末尾追加「建议您咨询专业医生或特教老师获取更具针对性的指导」
- **业务影响**：应急方案缺乏真实案例支撑，可信度降低，用户可能对该方案的参考价值存疑。建议在呈现时通过前端视觉区分（如灰色边框提示条）以管理用户预期

#### 1.8.2 异常 2：大模型生成超时或失败

- **业务触发条件**：大模型在约定时限（建议 10 秒内完成全流程生成）内未返回完整结果，或返回结果无法解析为四段式结构
- **业务处理策略**：
  1. 若已生成部分文本（至少一个完整段落），将已生成的段落输出并标注「部分生成」，告知用户其余段落因技术原因未能完成
  2. 若未生成任何文本，明确告知用户「应急方案生成失败，请稍后重试」，不输出任何不完整的片段
  3. 无论部分生成或完全失败，均允许用户等待 30 秒冷却期后重新请求生成
  4. 记录该次失败信息（生成场景、输入摘要、失败原因）供后续质量分析与改进
- **业务影响**：用户体验受损，但系统可通过重新生成机制让用户再次获取建议。部分生成时已产出的段落仍可供家属参考

#### 1.8.3 异常 3：危机阻断标记激活

- **业务触发条件**：上游 CSLT-01 传递的阻断深度回答标记为真（即危机等级为重度，涉及自伤、攻击、逃跑或用药等高风险场景）
- **业务处理策略**：
  1. 完全跳过 Prompt 组装和 LLM 调用环节
  2. 直接输出预设的安全提示文本，内容面向家属告知当前场景已超出 AI 可安全应对范围
  3. 安全提示文本中强制包含紧急联系指引（如建议立即拨打急救电话或联系最近的医疗机构）
  4. 不生成任何深度建议，不输出案例引用，不包含四段式结构
- **业务影响**：本模块在该场景下仅扮演安全提示传递角色。后续的人工工单生成与专家对接由 CSLT-05 和 TICK-01 负责

### 1.11 业务约束清单

1. **【合规约束】** 每次生成输出必须包含免责声明：「以上建议由 AI 生成，仅供参考，不构成医疗诊断或治疗建议。如情况紧急，请立即联系专业医疗机构。」该声明不可被 LLM 生成内容覆盖、省略或弱化。
2. **【安全约束】** 当危机判定为重度时，本模块必须阻断深度回答——不得生成任何包含具体干预步骤、安抚话术或就医判断的四段式建议，仅输出预设的安全提示文本。
3. **【质量约束】** 大模型调用温度参数不得超过 0.3，以确保生成结果的高确定性和低幻觉风险。
4. **【性能约束】** 从接收完整输入到首个文字内容产出的时间不得超过 3 秒；全流程生成完成建议在 10 秒内。
5. **【用户体验约束】** 用户触发重新生成后，系统施加 30 秒冷却期限制，冷却期内不允许再次触发生成。冷却期的存在和剩余时间应由前端明确展示给用户。
6. **【设计边界】** 本模块不负责案例检索（归属 CSLT-02）、不负责危机等级判定（归属 CSLT-01）、不负责置信度后校验与工单创建（归属 CSLT-05）、不负责流式推送的传输协议实现（归属 CSLT-04）。本模块聚焦于 Prompt 组装与 LLM 生成这一个环节。

### 1.12 留给规范阶段的技术决策

| 序号 | 技术决策项 | 意图文档中的约束 | 留给规范阶段的理由 |
|------|-----------|-----------------|-------------------|
| 1 | Prompt 模板具体设计 | 必须包含四段式输出结构、免责声明模板、来源引用规则、专业术语解释规则；阻断场景仅输出安全提示文本 | 具体模板文本措辞、系统指令与 Few-shot 示例的编排方式、各输入模块在 Prompt 中的排列顺序和权重，由 spec-writer 根据 LLM 行为特征确定 |
| 2 | LLM 调用参数精确值 | 温度参数不超过 0.3 | Top-P、max_tokens、frequency_penalty 等其余参数的具体数值由 spec-writer 根据生成质量基准测试确定 |
| 3 | 冷却期实现机制 | 用户两次生成之间间隔不少于 30 秒 | 冷却期的校验位置（前端拦截或后端拒绝）、剩余时间的展示方式、冷却期内用户交互的 UI 处理由 spec-writer 确定 |
| 4 | 数据模型技术类型 | 输入/输出字段的业务含义已明确定义 | 具体数据结构定义（字段类型、枚举值、嵌套结构、校验规则）由 spec-writer 在落地规范中确定 |
| 5 | 异常处理技术实现 | 三种业务异常场景及处理策略已定义 | 具体的异常类、日志格式、错误码、降级链路的代码实现由 spec-writer 确定 |
| 6 | 档案上下文格式化方式 | 注入范围已明确（5条事件 + 诊断标签，约 500-800 字） | 事件记录摘要的提取逻辑、各字段在 Prompt 中的排列顺序和文本格式模板由 spec-writer 确定 |
| 7 | 生成超时的精确阈值 | 建议全流程完成在 10 秒内 | 精确的全流程超时上限以及超时后的截断策略由 spec-writer 根据 LLM API 性能基准和用户体验测试确定 |
| 8 | 安全提示文本的预设内容 | 阻断场景必须输出预设安全提示 | 具体的提示文本措辞、紧急联系指引的内容（如是否包含固定急救电话号码）、是否需要根据具体危机类型展示不同的安全提示变体，由 spec-writer 确定 |
