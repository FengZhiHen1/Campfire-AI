# 应急咨询 — 咨询历史 内容骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#02-智能应急咨询`（CSLT-06 咨询历史管理）
> 目标平台：miniprogram（原生导航栏 + 触控）
> 数据来源：`consultApi.fetchHistoryList(page, pageSize)` → `HistoryListResponse`

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 咨询历史（consult-history） |
| 所属页面 | 应急咨询（consult） |
| 页面职责 | 展示用户所有历史咨询记录列表，支持搜索、分页浏览、跳转详情 |
| 入口来源 | Tab "应急咨询" → 导航栏入口；或主咨询页完成后的操作入口 |
| 出口目标 | 点击列表条目 → 咨询详情页（`detail.tsx`），携带 `id` 参数 |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `list` | `ConsultationHistoryListItem[]` | `consultApi.fetchHistoryList()` | 已加载的全部列表条目 |
| `loading` | `boolean` | 组件内部状态 | 初始加载与翻页加载中标识 |
| `error` | `string \| null` | 组件内部状态 | 加载失败的错误信息 |
| `searchQuery` | `string` | 用户输入 | 搜索关键词（客户端过滤） |
| `currentPage` | `number` | 组件内部状态 | 当前已加载的页码 |
| `hasMore` | `boolean` | 派生自 `total` vs `list.length` | 是否还有更多数据可加载 |
| `total` | `number` | `HistoryListResponse.total` | 后端总条目数 |
| `crisisLevel` | `CrisisLevel` | `item.crisis_level` | 单条记录的危机等级（mild/moderate/severe） |
| `consultationTime` | `string` | `item.consultation_time` | 咨询发生时间 |
| `behaviorDescription` | `string` | `item.behavior_description` | 用户行为描述原文 |
| `hasFeedback` | `boolean` | `item.has_feedback` | 用户是否已提交反馈 |

**状态路由**：页面自身管理 loading / error / empty / normal 四态。无外部状态机依赖——历史列表是独立只读数据源。

## 导航关系

```
Tab "应急咨询"
    │
    ├── 主咨询页（index.tsx）
    │       │ 导航栏历史入口 → 咨询历史
    │       │ 已完成态 "查看历史" → 咨询历史
    │       ▼
    ├── 咨询历史（本页，history.tsx）
    │       │ 点击条目 → goDetail(id)
    │       ▼
    │   咨询详情（detail.tsx）
    │       │ 原生返回 → 咨询历史
    │       ▼
    │   ...
```

咨询历史是应急咨询 Tab 下的二级页面，进入方式：
- 从主咨询页导航栏进入
- 从主咨询页已完成态的操作入口进入

返回行为：
- 原生导航栏返回按钮 → 回到上一页（主咨询页或 Tab 首页）
- 从详情页返回 → 回到本页（列表状态保持，不重新加载）

## 内容区块

### 区块A — 导航栏

- **Logic-ID**: 无（原生渲染）
- **职责**: 页面标识 + 返回入口
- **内容**: 标题"咨询历史"
- **交互**: 原生导航栏自带返回按钮
- **三态**: 不适用（原生组件，静态）
- **data-testid**: 无（原生组件）

### 区块B — 搜索栏

- **Logic-ID**: `searchQuery`
- **职责**: 让用户在已加载列表中按关键词快速定位某条历史记录
- **内容**:
  - 一个文本输入区域
  - 占位提示文字："搜索历史咨询记录…"
- **数据来源**: 用户输入 → `searchQuery` 状态
- **过滤方式**: 客户端本地匹配——对 `behavior_description` 字段做大小写不敏感的关键词包含匹配
- **交互**:
  - 用户输入时实时过滤列表（无需点击搜索按钮）
  - 清空搜索框时恢复完整列表
  - 过滤结果为空时：列表区域显示"未找到匹配记录"提示，同时保留搜索框内容供用户修改
- **三态**:
  - Loading: 搜索框在加载中仍然可交互（不影响搜索输入）
  - Empty: 搜索框始终可见
  - Error: 搜索框始终可见
- **data-testid**: `history-search`, `history-search-input`

### 区块C — 历史列表（核心内容区）

- **Logic-ID**: `list`（已过滤）, `currentPage`, `hasMore`, `crisisLevel`, `consultationTime`, `behaviorDescription`, `hasFeedback`
- **职责**: 按时间倒序展示用户的所有历史咨询记录，每条可见核心信息，点击可查看详情

#### 列表条目内容

每条历史记录包含以下信息（按优先级排列）：

**第一优先级（用户一瞥即知）**：
- 危机等级标识："轻度"/"中度"/"重度" 三档，语义与主咨询页一致
- 行为描述摘要：`behavior_description` 的前约 50 个字符，超出截断并追加省略符号

**第二优先级（辅助判断）**：
- 咨询时间：`consultation_time`，格式为可读的时间表述
- 可信标签：标识 AI 生成方案的可信程度——"高可信"/"中可信"/"需复核"

#### 可选辅助信息

- 行为类型标签：如果后端提供 `behavior_types` 字段，在卡片上展示为标签列表（如"自伤行为""情绪崩溃"）；当前后端暂未返回此字段，预留展示位置
- 反馈状态：如果 `has_feedback === true`，条目上显示"已反馈"标记

#### 条目交互

- 点击整条记录 → 跳转咨询详情页（`detail.tsx`），传递 `id` 参数
- 点击反馈：无交互——反馈在详情页或咨询完成后的弹窗中进行

#### 排序

- 按 `consultation_time` 倒序排列（最新记录在最上方）

#### 三态

| 态 | 行为 |
|----|------|
| Loading（首次加载） | 显示占位内容——条目区域有形状提示但无具体文字（见区块D） |
| Empty（无记录） | 显示空状态引导（见区块E） |
| Error（加载失败） | 显示错误状态（见区块F） |
| Normal（有数据） | 正常渲染列表条目 |

- **data-testid**: `history-list`, `history-card-{n}`, `history-card-level-{n}`, `history-card-summary-{n}`, `history-card-time-{n}`, `history-card-trust-{n}`, `history-card-feedback-mark-{n}`

### 区块D — 加载骨架

- **Logic-ID**: `loading`（首次加载，`list.length === 0`）
- **职责**: 告知用户正在加载历史记录，缓解等待焦虑
- **内容**:
  - 条目区域有形状提示（同形内容区占位），让用户预知此处将有列表条目
  - 文字提示："正在整理您的咨询记录…"
- **数据来源**: 无——纯过渡态展示
- **交互**: 无用户可操作项。数据到达后自动切换到正常态或空态
- **区别于翻页加载**：首次加载使用骨架占位（`loading && list.length === 0`）；翻页加载时不显示骨架，而是在列表底部显示"加载中…"指示（见区块G）
- **三态**: 本区块即是 Loading 态的展现
- **data-testid**: `history-loading`, `history-loading-skeleton-row`, `history-loading-text`

### 区块E — 空状态

- **Logic-ID**: `list.length === 0` 且 `loading === false` 且 `error === null`
- **职责**: 告知用户尚无咨询记录，并引导前往主咨询页发起第一次咨询
- **内容**:
  - 一个与品牌相关的意象元素（纯装饰，帮助缓和"空"的感受）
  - 标题："暂无咨询记录"
  - 说明文字："当需要应急建议时，前往应急咨询页面发起对话"
  - 一个操作入口：触发导航到主咨询页
- **数据来源**: 无外部数据依赖
- **交互**: 操作入口触发跳转到主咨询页（`/views/consult/pages/index`）
- **搜索导致的空态**：当 `searchQuery` 非空且过滤结果为空时，不与全局空态混淆——显示过滤专用提示"未找到匹配记录"，并提供清空搜索的入口
- **三态**: 本区块即是 Empty 态的展现
- **data-testid**: `history-empty`, `history-empty-graphic`, `history-empty-title`, `history-empty-subtitle`, `history-empty-cta-btn`

### 区块F — 错误状态

- **Logic-ID**: `error`（非 null）
- **职责**: 告知用户加载失败，提供重试入口
- **内容**:
  - 错误示意元素
  - 标题："无法加载记录"
  - 辅助说明："请检查网络连接后重试"
  - 一个操作入口："重新加载" → 重新调用 `load()`（重置为第 1 页）
- **数据来源**: `error`（错误信息字符串）
- **交互**:
  - 主操作："重新加载" → 触发 `load()`
  - 如果部分数据已加载（翻页加载失败场景）：保留已加载列表 + 底部显示重试入口（见区块G），不替换为全屏错误态
- **不使用** toast/alert 阻断式弹窗
- **三态**: 本区块即是 Error 态的展现
- **data-testid**: `history-error`, `history-error-icon`, `history-error-title`, `history-error-subtitle`, `history-error-retry-btn`

### 区块G — 翻页加载区（列表底部）

- **Logic-ID**: `hasMore`, `currentPage`
- **职责**: 用户滚动到列表底部时自动加载下一页数据
- **触发条件**: 列表滚动到底部、`hasMore === true`、非 Loading 中
- **交互**: 无用户手动操作——滚动到接近底部时自动触发 `loadNextPage()`
- **加载中指示**：底部显示"加载中…"提示，同时阻止重复触发加载
- **加载完毕**：`hasMore === false` 时显示"已加载全部记录"
- **加载失败**：底部显示"加载失败，点击重试"，保留已加载列表条目
- **三态**:
  - Loading: 底部显示"加载中…"
  - Empty（无更多）: 底部显示"已加载全部记录"
  - Error: 底部显示"加载失败，点击重试"
- **data-testid**: `history-load-more`, `history-load-more-loading`, `history-load-more-end`, `history-load-more-error`

## 交互行为汇总

| 行为 | 触发方式 | 结果 |
|------|---------|------|
| 搜索 | 用户输入关键词 | 客户端实时过滤列表，大小写不敏感 |
| 查看详情 | 点击列表条目 | 跳转 `detail.tsx?id={id}` |
| 初始加载 | 页面进入 | 请求第 1 页数据（20 条） |
| 加载更多 | 滚动到列表底部 | 自动请求下一页（20 条），追加到列表 |
| 下拉刷新 | 列表顶部下拉 | 重置为第 1 页并重新加载 |
| 重试加载 | 点击"重新加载" | 初始加载失败时重置为第 1 页重试 |
| 翻页重试 | 点击"点击重试" | 仅重试当前页码的加载，保留已加载数据 |
| 前往咨询 | 点击空状态操作入口 | 跳转主咨询页 |
| 返回 | 点击原生返回按钮 | 回到上一页，列表状态保持 |

## 下拉刷新行为

- 用户在列表顶部下拉触发
- 下拉时显示刷新指示
- 请求第 1 页数据（全新加载，不追加）
- 刷新完成后恢复到列表顶部
- 刷新期间不阻止列表滚动（非模态）
- **data-testid**: `history-pull-refresh`

## 平台行为差异

| 行为 | miniprogram |
|------|------------|
| 导航 | 原生导航栏，标题居中，自带返回按钮 |
| 滚动 | 页面内容区可滚动（scroll-view），启用 `scrollY` |
| 下拉刷新 | `scroll-view` 的 `refresher-enabled` + `onRefresherRefresh` |
| 触底加载 | `scroll-view` 的 `onScrollToLower`，设置 `lower-threshold` |
| 安全区 | 底部预留 safe-area-inset-bottom。列表底部翻页加载区不被安全区遮挡 |
| 触控 | 列表条目整体可点击（不小范围热区），条目间有明确分隔 |

## 文件拆分

```
consult/pages/
├── history.tsx           ← 接线容器
│   → 管理 loading / error / empty / normal 四态
│   → 管理 searchQuery 本地过滤
│   → 管理分页状态（currentPage, hasMore）
│   → 调用 consultApi.fetchHistoryList(page, pageSize)
│   → 通过 Props 注入数据到子组件
├── history.config.ts     ← 页面配置
├── history.scss          ← 样式
└── components/
    ├── HistorySearchBar.tsx     ← 搜索栏（区块B）
    ├── HistoryCard.tsx          ← 列表条目卡片（区块C）
    │   → 接收 ConsultationHistoryListItem + trustLabel + behaviorTypes
    │   → 纯展示，通过 onClick prop 触发导航
    │   → 解析 crisis_level → 等级文字 + 可信标签文字
    ├── HistoryEmpty.tsx         ← 空状态（区块E）
    ├── HistoryError.tsx         ← 错误状态（区块F）
    └── HistoryLoadMore.tsx      ← 翻页加载区（区块G）
        → 接收 hasMore / isLoadingMore / loadError
        → 展示对应状态的文字提示
```

**原则**:
- `history.tsx` 是唯一接线点 — 管理数据获取、分页、搜索过滤、状态路由
- 所有子组件为纯展示型 — 数据和回调通过 Props 接收
- 危机等级文字映射（mild→"轻度"/moderate→"中度"/severe→"重度"）与可信标签映射（mild→"高可信"/moderate→"中可信"/severe→"需复核"）提取为工具函数，在 HistoryCard 内或共享 utils 中实现

## 与主咨询页的内容关联

以下内容模式与主咨询页语义一致，在 OD 设计时应保持统一的信息传达方式：

| 内容模式 | 主咨询页 | 咨询历史 | 说明 |
|---------|---------|---------|------|
| 危机等级标识 | 导航栏内 / 置信度区域 | 列表条目首行 | 同样的三档等级体系，语义一致 |
| 可信标签 | 置信度标识（区块F） | 列表条目首行 | 语义一致——"高可信"/"中可信"/"需复核" |
| 免责声明 | 页面底部常驻条 | 不在本页展示 | 本页是索引列表，不直接展示方案内容 |
