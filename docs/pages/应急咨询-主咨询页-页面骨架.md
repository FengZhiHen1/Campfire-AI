## 导航关系

```
Tab "应急咨询"
    │
    ▼
idle (本页)
    │ 用户操作"开始咨询" → startConsult()
    ▼
selecting_behavior (select.tsx)  ← 独立页面
    │ 提交 → submitting (本页)
    │ 取消 → idle (本页)
    ▼
submitting → streaming → completed (本页)
    │
    ├── startNewConsult() → selecting_behavior (select.tsx)
    ├── goToTicket() → 工单详情
    └── goBackToSelecting() → selecting_behavior (select.tsx)

异常路径：
    submitting ──[提交失败]──→ submit_failed
    streaming  ──[SSE 中断]──→ stream_failed
    streaming  ──[置信度不足]──→ ticket_guide
```

## 内容区块

### 区块A — 导航栏

- **Logic-ID**: `sessionState`, `crisisLevel`
- **职责**: 标识当前页面，在危机场景中展示危机等级
- **内容**:
  - 始终展示页面标题"应急咨询"
  - `crisisLevel` 有值时：额外展示危机等级文字（"等级：轻度"/"等级：中度"/"等级：重度"）
- **数据来源**: `sessionState` 决定是否展示危机标识（仅 streaming/completed 状态）；`crisisLevel` 决定等级文字
- **交互**: 无（纯信息展示）。idle/submitting/error/ticket_guide 状态不展示危机标识
- **三态**:
  - Loading: 不适用（导航栏无异步加载）
  - Empty: 无 crisisLevel 时不展示危机标识
  - Error: 不适用
- **data-testid**: `consult-navbar`, `consult-navbar-crisis-badge`

### 区块B — idle 入口区

- **Logic-ID**: `sessionState`, `startConsult`
- **职责**: 让用户理解页面用途，并提供开始咨询的入口
- **内容**:
  - 一个抽象的暖光/篝火意象图形（纯装饰，帮助建立"篝火智答"品牌认知）
  - 页面标题"应急咨询"
  - 副标题说明：告知用户此页面的用途——描述孩子的行为表现，获取基于真实案例的应急建议
  - 一个操作入口：触发"开始咨询"流程
- **数据来源**: 无外部数据依赖，纯引导页
- **交互**: 操作入口触发 `startConsult()`，跳转到行为选择页（`select.tsx`）
- **三态**:
  - Loading: 不适用（idle 是稳态）
  - Empty: 不适用
  - Error: 不适用
- **data-testid**: `consult-idle`, `consult-idle-start-btn`, `consult-idle-graphic`

### 区块C — submitting 加载区

- **Logic-ID**: `sessionState`, `isConsultActive`
- **职责**: 告知用户系统正在分析案例库，缓解等待焦虑
- **内容**:
  - 一个暖光/光晕意象的图形元素（可带动画）
  - 主文案："正在分析案例库…"
  - 副文案："匹配最相似的历史案例"
  - 进度指示：传达"系统正在工作中"的信号（如进度点序列）
- **数据来源**: 无——纯过渡态展示
- **交互**: 无用户可操作项。SSE 首段结构化数据到达后自动切换到 streaming 状态
- **过渡**: 不展示灰色矩形骨架屏——submitting 是活跃的等待状态
- **三态**:
  - Loading: 暖光图形 + 进度指示同时播放（本区块唯一态）
  - Empty: 不适用
  - Error: 不适用（提交失败走 submit_failed 状态）
- **data-testid**: `consult-submitting`, `consult-submitting-glow`, `consult-submitting-dots`, `consult-submitting-text`

### 区块D — 用户查询摘要

- **Logic-ID**: `behaviorDescription`, `behaviorTypeSelection`, `emotionLevel`
- **职责**: 紧凑展示用户提交的内容，作为下方方案的上下文锚点——让用户随时知道"我问了什么"
- **内容**:
  - 第一行：已选行为类型名称 + 情绪等级
  - 第二行：用户输入的行为描述原文（较长时截断，不占用过多空间）
- **数据来源**: `behaviorDescription`（文本）、`behaviorTypeSelection`（类型标签列表）、`emotionLevel`（等级）
- **交互**: 无（纯信息展示）
- **三态**:
  - Loading: 不适用（streaming 首帧即显示完整数据）
  - Empty: 不适用（必有用户输入才进入 streaming）
  - Error: 不适用
- **data-testid**: `consult-user-summary`, `consult-user-summary-type`, `consult-user-summary-desc`

### 区块E — 干预方案卡片（核心内容区）

- **Logic-ID**: `planSections`, `accumulatedText`, `sessionState`
- **职责**: 承载 AI 生成的四段式应急方案——用户来此页面的唯一目的
- **内容**（四段固定结构）:
  1. **即时安全干预**：在危机发生的当下应采取的具体安全措施
  2. **情绪安抚话术**：对孩子的安抚语言模板（首句为可直接使用的引用话术）
  3. **后续观察指标**：需要持续关注的指标列表
  4. **就医判断标准**：判断是否需要立即送医的标准
- **数据来源**: `planSections`（结构化段落数组），`accumulatedText`（流式原始文本）
- **四段标题始终可见**：即使在流式接收中，四个段落的标题也从第一帧就全部显示，避免后续内容填充导致布局跳动
- **交互**: 无（内容为只读展示）

#### streaming 态行为

- 卡片结构（标题 + 四段标题）从第一帧就完整出现
- 当前正在接收的段落：文本逐句追加，有"正在输入"的视觉反馈
- 尚未开始的段落：显示"（待生成）"占位提示
- 已接收完成的段落：标记为"已完成"
- 段落完成后，下一段自动开始接收

#### completed 态行为

- 全部四段内容完整展示
- 每段内容独立展示，段落间有明确分隔
- 安抚话术段中，首句以引用形式突出（区别于后续说明性文字）
- 底部信息栏：案例引用数量（如"基于 N 个相似案例"）、置信度等级标识

#### 四段式内容规范

| 段名 | 内容特征 |
|------|---------|
| 即时安全干预 | 操作步骤列表，每步一条 |
| 情绪安抚话术 | 首句为可直接使用的安抚引语，后续为辅助说明 |
| 后续观察指标 | 以标记列表呈现的观察项 |
| 就医判断标准 | 关键就医触发条件（仅需就医时呈现，可能为空） |

#### 三态

| 态 | 行为 |
|----|------|
| Loading (streaming) | 标题 + 四段结构可见，内容逐句填充 |
| Empty | 不适用 — streaming 必有内容 |
| Error | 不适用 — streaming 失败走 stream_failed 状态 |

- **data-testid**: `consult-plan-card`, `consult-plan-card-header`, `consult-plan-section-{n}`, `consult-plan-footer`

### 区块F — 置信度标识

- **Logic-ID**: `crisisLevel`, `ticketGuide`（低可信时联动）
- **职责**: 告知用户本次方案的 AI 置信程度
- **内容**: 三档文字标识
  - 高可信（crisisLevel: mild 或置信度 ≥ 0.85）："高可信"
  - 中可信（crisisLevel: moderate 或置信度 0.70–0.84）："中可信"
  - 低可信/需复核（crisisLevel: severe 或置信度 < 0.70）："需人工复核"
- **数据来源**: `crisisLevel` 和置信度评分（来自 SSE done 事件）
- **交互**: 纯信息展示，无点击行为。低可信时自动触发工单引导区（区块I）的出现
- **三态**:
  - Loading: 不展示（streaming 期间置信度未知）
  - Empty: 不适用
  - Error: 不适用
- **data-testid**: `consult-confidence-badge`

### 区块G — 参考案例

- **Logic-ID**: `referencedCases`
- **职责**: 展示方案所引用的真实案例来源，提供可溯源性
- **内容**: 每条案例包含
  - 案例编号/标题
  - 案例摘要文本
- **数据来源**: `referencedCases`（数组）
- **条件渲染**: `referencedCases.length > 0` 时才展示
- **交互**:
  - 默认折叠（仅显示案例总数和展开入口）
  - 用户可点击展开查看案例列表
  - 点击单条案例 → 跳转案例卡片详情页
- **三态**:
  - Loading: 不适用（数据随 completed 状态一起到达）
  - Empty: 无参考案例时整个区块不渲染
  - Error: 不适用
- **data-testid**: `consult-ref-cases`, `consult-ref-cases-toggle`, `consult-ref-case-{n}`

### 区块H — 操作区

- **Logic-ID**: `startNewConsult`, `goToTicket`, `isConsultActive`
- **职责**: 提供方案生成完成后的后续操作入口
- **内容**:
  - completed 态：
    - 主操作："开始新咨询" → `startNewConsult()`
    - 辅助操作："联系人工专家" → `goToTicket()`
  - streaming 态：操作入口显示"生成中…"且不可点击
- **交互**:
  - 主操作：触发新一轮咨询流程
  - 辅助操作：跳转工单模块
- **三态**:
  - Loading (streaming): 操作入口不可用，显示"生成中…"
  - Empty: 不适用
  - Error: 不适用
- **data-testid**: `consult-actions`, `consult-actions-new-btn`, `consult-actions-expert-link`

### 区块I — 人工兜底条

- **Logic-ID**: `ticketGuide`, `goToTicket`
- **职责**: 当 AI 置信度不足时，引导用户转人工专家
- **条件渲染**: `ticketGuide.show === true` 时才出现
- **内容**: 醒目的引导文字 "立即联系人工专家"
- **数据来源**: `ticketGuide.show`（控制显隐）、`ticketGuide.riskLevel`（风险等级）
- **交互**: 点击直接调用 `goToTicket()`，无二次确认
- **出现方式**: 从页面底部滑入（不是突兀出现）
- **三态**:
  - Loading: 不适用
  - Empty: `ticketGuide.show === false` 时整个区块不渲染
  - Error: 不适用
- **data-testid**: `consult-escalation-bar`

### 区块J — 免责声明

- **Logic-ID**: 无（纯合规展示）
- **职责**: 合规提示——告知用户 AI 建议的性质和限制
- **内容**: 固定文案 "基于归档案例的 AI 生成建议，不构成医疗诊断。严重情况请咨询专业医生。"
- **全状态常驻**：idle / submitting / streaming / completed / error / ticket_guide 所有状态均显示
- **三态**: 不适用（纯静态常驻，无数据依赖）
- **data-testid**: `consult-disclaimer`

## 错误状态

### submit_failed 状态

- **触发**: 提交请求失败（网络异常或服务端错误）
- **内容**:
  - 错误图形元素
  - 标题："生成失败"
  - 错误详情：由 `getErrorMessage(errorCode)` 动态提供，不硬编码
  - 两个操作入口：
    - "重试提交" → `retrySubmit()`
    - "返回修改" → `goBackToSelecting()`
- **不使用** toast/alert 阻断式弹窗——错误信息直接展示在页面内容区
- **data-testid**: `consult-error`, `consult-error-retry-btn`, `consult-error-back-btn`

### stream_failed 状态

- **触发**: SSE 连接中断且重连耗尽
- **内容**: 与 submit_failed 类似，但主操作为"重新生成"（`retryStream()`）
- **操作入口**:
  - "重新生成" → `retryStream()`
  - "返回修改" → `goBackToSelecting()`
- **data-testid**: `consult-error`, `consult-error-retry-btn`, `consult-error-back-btn`

### ticket_guide 状态

- **触发**: 方案生成完成但置信度不足，不适合直接展示建议
- **内容**:
  - 咨询/引导图形元素
  - 标题："建议联系专家"
  - 说明文字：告知用户 AI 对当前情况的置信度较低，建议通过人工咨询获取更准确建议
  - 两个操作入口：
    - "联系专家" → `goToTicket()`
    - "开始新咨询" → `startNewConsult()`
- **data-testid**: `consult-ticket-guide`, `consult-ticket-guide-expert-btn`

## 文件拆分

```
consult/pages/
├── index.tsx           ← 接线容器
│   → 导入 useConsult Hook
│   → 按 sessionState 路由到对应子组件
│   → 通过 Props 注入数据和回调
├── select.tsx          ← 行为选择页（selecting_behavior，独立页面）
└── components/
    ├── IdleEntry.tsx        ← idle 入口区（区块B）
    ├── SubmittingGlow.tsx   ← submitting 加载区（区块C）
    ├── UserQuerySummary.tsx ← 用户查询摘要（区块D）
    ├── PlanCard.tsx         ← 干预方案卡片（区块E）
    │   → streaming/completed 两态由 isStreaming prop 驱动
    ├── ConfidenceBadge.tsx  ← 置信度标识（区块F）
    ├── ReferenceCases.tsx   ← 参考案例（区块G）
    ├── EscalationBar.tsx    ← 人工兜底条（区块I）
    └── DisclaimerBar.tsx    ← 免责声明（区块J）
```

**原则**:
- `index.tsx` 是唯一接线点 — import useConsult，将 Hook 返回值拆解后通过 Props 注入子组件
- 所有子组件为纯展示型 — 不 import 任何 Hook，所有数据和回调通过 Props 接收
- Props 类型内联定义于组件文件（如需跨文件共享，抽取到 `contracts/` 目录）
