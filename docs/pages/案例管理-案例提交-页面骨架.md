## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 案例提交（cases-submit） |
| 所属页面 | 案例管理（cases） |
| 页面职责 | 提交已提取的干预案例卡片——填写基础信息 + 四段式内容 |
| 入口来源 | 案例列表 FAB / 案例详情"提取卡片" → `submit?narrativeId={id}` |

### Logic-ID 清单

| ID | 类型 | 用途 |
|----|------|------|
| `title` | `string` | 案例标题 |
| `behaviorTypeIdx` | `number` | 行为类型选择器索引 |
| `severityIdx` | `number` | 严重程度选择器索引 |
| `sceneIdx` | `number` | 场景选择器索引 |
| `evidenceLevelIdx` | `number` | 循证等级选择器索引 |
| `quartetValues` | `Record<string, string>` | 四段式内容文本 |
| `isSubmitting` | `boolean` | 提交中 |

## 内容区块

### 区块A — 封面图占位
- **职责**: 上传现场环境图（辅助理解情境）
- **内容**: 相机图标 + "点击上传封面图" + 辅助提示
- **交互**: 点击触发图片选择（MVP 可仅展示占位）
- **data-testid**: `submit-cover`

### 区块B — 基本信息
- **职责**: 填写案例元数据
- **必填字段**: 案例标题（文本输入）、行为类型（选择器）、严重程度（选择器）、场景（选择器）、循证等级（选择器）
- **data-testid**: `submit-field`

### 区块C — 四段式内容
- **职责**: 填写四段结构化干预内容
- **四段**（每段独立卡片）:
  1. 即时安全干预动作 — "请描述应立即采取的安全措施"
  2. 情绪安抚话术 — "请提供可直接使用的安抚话术"
  3. 后续观察指标 — "列出需要持续关注的指标"
  4. 就医判断标准 — "明确需要就医的触发条件"
- **每段**: 必填标记 + 输入提示 + 多行文本区（≤2000 字）
- **data-testid**: `submit-card`, `submit-card__textarea`

### 区块D — 底部操作栏
- **职责**: 提交案例
- **内容**: "提交案例"按钮
- **提交中**: 按钮显示"提交中..."且不可点击
- **data-testid**: `submit-actions__btn`

### data-testid 节点清单

```
submit-cover, submit-cover__icon
submit-field, submit-field__picker, submit-field__input
submit-card--{accent}, submit-card__textarea
submit-actions, submit-actions__btn
```
