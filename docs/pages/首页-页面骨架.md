# 首页 内容骨架

> 生成日期：2026-06-28
> 从现有代码反推：`views/shared/pages/home.tsx` + `useHomePage` Hook
> 目标平台：miniprogram（触控）

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 首页（home） |
| 页面职责 | Tab 首页——品牌问候 + 应急咨询快捷入口 + 最近咨询预览 + 档案概要 |
| 入口来源 | Tab "首页"（`views/shared/pages/home`） |
| 出口目标 | 应急咨询（switchTab）、咨询历史、档案列表、档案创建 |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `loading` | `boolean` | `useHomePage().loading` | 咨询历史加载中 |
| `hasError` | `boolean` | `useHomePage().hasError` | 数据加载失败 |
| `consultHistory` | `ConsultationHistoryListItem[]` | `consultApi.fetchHistoryList(1, 5)` | 最近 5 条咨询记录 |
| `profiles` | `ProfileListItem[]` | `useProfileStore().list` | 用户档案列表 |
| `profilesLoading` | `boolean` | `useHomePage().profilesLoading` | 档案列表加载中 |
| `greeting` | `string` | 客户端时间派生 | 时段问候语（早上好/下午好/晚上好） |

**Tab 导航结构**（4 个 Tab）：
```
首页（本页） | 咨询 | 案例 | 档案
```

## 内容区块

### 区块A — 顶部问候区

- **Logic-ID**: `greeting`, `profiles`
- **职责**: 品牌标识 + 个性化问候——让用户感受到"这是我的空间"
- **内容**:
  - 品牌名称"篝火智答" + 品牌意象元素（篝火）
  - 时段问候语：根据当前时间自动切换——5:00-11:59 "早上好"、12:00-17:59 "下午好"、其余"晚上好"
  - 副标题：有档案时"今天孩子状态怎么样？"，无档案时"欢迎开始使用篝火智答"
- **数据来源**: `greeting`（时间派生）、`profiles`（判断是否有档案）
- **交互**: 无（纯展示）
- **三态**: 不适用（纯静态，无异步加载）
- **data-testid**: `home-greeting`, `home-greeting-brand`, `home-greeting-title`, `home-greeting-subtitle`

### 区块B — 错误通知条

- **Logic-ID**: `hasError`, `load`
- **职责**: 数据加载失败时告知用户并提供重试入口
- **内容**: 错误提示文字"数据加载失败"+ 重试按钮
- **条件渲染**: `hasError === true` 时显示
- **交互**: 点击"重试"→ 重新调用 `load()`
- **data-testid**: `home-error-banner`, `home-error-banner-retry`

### 区块C — 应急咨询入口卡片（核心）

- **Logic-ID**: 无（纯导航——这是首页最重要的操作入口）
- **职责**: 向用户传达"这里可以解决紧急问题"——首页的首要行动召唤
- **内容**:
  - 紧急标识元素
  - 标题："应急咨询"
  - 说明文字："遇到紧急情况？描述当前状况，AI 将在几秒内生成个性化建议"
  - 主操作入口："立即咨询"
- **数据来源**: 无外部数据依赖，纯导航组件
- **交互**: 点击"立即咨询"→ `switchTab` 跳转到咨询 Tab
- **位置**: 始终可见——不论数据加载状态如何
- **三态**: 不适用（纯静态导航，无数据依赖）
- **data-testid**: `home-emergency`, `home-emergency-btn`

### 区块D — 最近咨询记录区

- **Logic-ID**: `consultHistory`, `loading`
- **职责**: 展示最近的咨询记录摘要——让用户快速了解上次咨询的情况，或引导首次咨询

#### 区域标题
- "最近咨询" + "查看全部 →"（跳转咨询历史页）

#### 内容（按状态）

**有记录时**：
- 展示最新一条咨询记录的摘要卡片：
  - 第一行：咨询记录标识 + 相对时间（如"3 小时前""昨天"）
  - 第二行：行为描述摘要（`behavior_description`，长文本截断）
  - 引导操作："→ 开始咨询"——点击跳转应急咨询 Tab
- **点击卡片** → `switchTab` 到咨询 Tab

**无记录时**：
- 空态提示元素
- "暂无咨询记录"
- 引导文字："点击上方'立即咨询'开始第一次对话"

**加载中**：
- 骨架占位

#### 三态

| 态 | 行为 |
|----|------|
| Loading | 骨架占位 |
| Empty | 空态引导——引导首次咨询 |
| Error | 不单独处理——错误由区块B统一承载 |

- **data-testid**: `home-section-header`, `home-consult-card`, `home-consult-empty`, `home-skeleton`

### 区块E — 个人档案快捷卡片

- **Logic-ID**: `profiles`, `profilesLoading`
- **职责**: 展示当前档案概览——让用户知道"在看哪个孩子的数据"，或引导创建首个档案

#### 区域标题
- "个人档案" + "查看全部 →"（跳转档案列表）

#### 内容（按状态）

**有档案时**：
- 档案概要卡片（显示第一个档案）：
  - 头像占位
  - 昵称 + 年龄段标签 + 诊断类型标签
  - 主要行为类型标签（如有）
  - 引导操作："→ 查看档案"——点击跳转档案列表
- **点击卡片** → 跳转档案列表页

**无档案时**：
- 创建档案引导卡片：
  - 图标："创建孩子的第一份档案"
  - 说明："完善的档案能帮助 AI 更精准地匹配案例"
  - 引导操作："创建档案 →"——点击跳转档案创建页

**加载中**：
- 骨架占位

#### 三态

| 态 | 行为 |
|----|------|
| Loading | 骨架占位 |
| Empty | 创建引导卡片 |
| Error | 不单独处理——档案加载失败时保留引导卡片 |

- **data-testid**: `home-profile-card`, `home-profile-create`, `home-skeleton--profile`

## 交互行为汇总

| 行为 | 触发方式 | 结果 |
|------|---------|------|
| 应急咨询 | 点击"立即咨询" | `switchTab` 到咨询 Tab |
| 查看咨询历史 | 点击"查看全部 →" | `navigateTo` 咨询历史页 |
| 从咨询记录开始咨询 | 点击咨询摘要卡片 | `switchTab` 到咨询 Tab |
| 查看档案列表 | 点击档案卡片 / "查看全部 →" | `navigateTo` 档案列表页 |
| 创建档案 | 点击"创建档案 →" | `navigateTo` 档案创建页 |
| 重试加载 | 点击错误通知条"重试" | 重新调用 `load()` |

## 文件拆分

```
shared/pages/
├── home.tsx           ← 接线容器
│   → 导入 useHomePage Hook
│   → 管理时段问候逻辑
│   → 四态路由（normal / loading / empty / error）
│   → 通过 Props 注入数据到子组件
└── components/
    ├── HomeGreeting.tsx        ← 顶部问候区（区块A）
    ├── HomeErrorBanner.tsx     ← 错误通知条（区块B）
    ├── HomeEmergencyCard.tsx   ← 应急咨询入口卡片（区块C）
    ├── HomeConsultSection.tsx  ← 最近咨询区（区块D）
    │   → 管理有记录/空态/加载三态
    └── HomeProfileSection.tsx  ← 档案快捷区（区块E）
        → 管理有档案/无档案/加载三态
```
