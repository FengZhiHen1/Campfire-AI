# 案例管理 — 案例详情 内容骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#04-真实案例库管理`（CASE-01/08/09）
> 目标平台：miniprogram（触控）
> 数据来源：`useCaseDetailPage()` Hook

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 案例详情（cases-detail） |
| 所属页面 | 案例管理（cases） |
| 页面职责 | 展示一条案例叙事原文 + 关联卡片摘要列表 + 操作入口 |
| 入口来源 | 案例列表页 → `detail?narrativeId={id}` |

### Logic-ID 清单

| ID | 类型 | 用途 |
|----|------|------|
| `data` | `NarrativeDetail \| null` | 叙事详情（标题/原文/来源/状态/关联卡片） |
| `loading` | `boolean` | 加载中 |
| `error` | `string \| null` | 加载错误 |

---

## 内容区块

### 区块A — 概览信息
- **职责**: 一瞥即知案例基本情况
- **内容**: 标题、来源类型标签、状态圆点 + 状态文字、关联卡片数量
- **数据来源**: `data.title`, `data.source_type`, `data.status`, `data.cards.length`
- **data-testid**: `detail-overview`

### 区块B — 叙事原文
- **职责**: 展示完整叙事文本（Markdown 渲染）
- **内容**: `data.narrative` 经 MarkdownRenderer 渲染
- **data-testid**: `detail-section`

### 区块C — 关联卡片列表
- **职责**: 展示该叙事下提取的案例干预卡片
- **条件渲染**: `data.cards.length > 0`
- **每卡片内容**: 标题、审核状态标签、行为类型/严重程度/场景标签
- **交互**: 点击卡片 → 跳转卡片详情页
- **data-testid**: `detail-card-item-{n}`

### 区块D — 操作区
- **草稿态**: 显示"提取卡片"+"编辑原文"两个操作入口
- **已通过态**: 显示"该案例已通过审核"提示
- **交互**: 提取卡片 → 跳转提取结果页；编辑原文 → 跳转编辑页

### 三态
- Loading: 骨架占位 + "加载中…"
- Error: 错误信息 + "重新加载"按钮
- 空数据: "未找到案例"

### data-testid 节点清单

```
detail-overview, detail-overview__title, detail-overview__status
detail-section, detail-section__title
detail-card-item-{n}, detail-card-item__status
detail-actions, detail-actions__btn--primary, detail-actions__btn--secondary
```
