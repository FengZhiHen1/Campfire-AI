# 档案管理 — 档案列表 内容骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#03-个性化档案`（PROF-01/03/06/07）
> 目标平台：miniprogram（原生导航栏 + 触控）
> 数据来源：`useProfile()` Hook → `UseProfileReturn`；`listEvents()` API

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 档案列表（profiles-index） |
| 所属页面 | 档案管理（profiles） |
| 页面职责 | 展示用户所有档案的概览信息 + 所选档案的事件记录时间线。是档案模块的主入口页。 |
| 入口来源 | Tab "档案管理" 或全局导航 |
| 出口目标 | 点击编辑 → 档案编辑页（`edit.tsx?mode=edit&profileId={id}`）；点击创建 → 档案编辑页（`edit.tsx?mode=create`） |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `profiles` | `ProfileListItem[]` | `useProfile().profiles` | 档案列表（最多 5 个） |
| `isLoading` | `boolean` | `useProfile().isLoading` | 档案列表加载中 |
| `error` | `Error \| null` | `useProfile().error` | 档案列表加载错误 |
| `selectedProfile` | `ProfileListItem \| null` | 组件内部 `selectedIdx` 派生 | 当前选中的档案 |
| `selectedDetail` | `ProfileResponse \| null` | `getProfile(id)` | 当前选中档案的详细信息 |
| `events` | `EventListItem[]` | `listEvents(profileId)` | 当前档案的事件记录列表 |
| `eventsLoading` | `boolean` | 组件内部状态 | 事件列表加载中 |
| `eventsPage` | `number` | 组件内部状态 | 事件列表已加载页码 |
| `eventsHasMore` | `boolean` | 组件内部状态 | 事件列表是否有更多 |
| `showQuickRecord` | `boolean` | 组件内部状态 | 快速记录弹窗是否可见 |
| `isSubmitting` | `boolean` | `useQuickRecord().isSubmitting` | 事件提交中 |

### 相关 Logic-ID（来自关联模块）

| ID | 模块 | 关联方式 |
|----|------|---------|
| `nickname` | `ProfileListItem` | 档案昵称（可为空，展示"未命名"） |
| `age_range` | `ProfileListItem` | 年龄段（服务端计算） |
| `diagnosis_type` | `ProfileListItem` | 诊断类型 |
| `primary_behavior` | `ProfileListItem` | 主要行为类型 |
| `is_default` | `ProfileListItem` | 是否默认档案 |
| `event_time` | `EventListItem` | 事件发生时间 |
| `behavior_type` | `EventListItem` | 事件行为类型 |
| `severity_level` | `EventListItem` | 事件严重程度 |
| `has_professional_note` | `EventListItem` | 是否有专业评估 |

**状态路由**：本页管理 cold-start / loading / error / normal 四态。normal 态内部包含事件时间线空态和快速记录弹窗显隐态。

## 内容区块

### 区块A — 导航栏

- **Logic-ID**: 无（自定义导航栏，非原生）
- **职责**: 页面品牌标识 + 编辑入口
- **内容**:
  - 左侧：品牌标识（篝火意象元素 + 标题"个人档案"）
  - 右侧：当 `selectedProfile` 存在时，显示"编辑"入口
- **交互**:
  - 点击"编辑" → 跳转编辑页（`edit.tsx?mode=edit&profileId={selectedProfile.profile_id}`）
  - 冷启动 / 加载态 / 错误态下不显示编辑按钮
- **三态**:
  - Loading: 不显示编辑按钮
  - Empty（冷启动）: 仅显示品牌标识，无编辑按钮
  - Error: 不显示编辑按钮
- **data-testid**: `profile-index-navbar`, `profile-index-navbar-title`, `profile-index-navbar-edit-btn`

### 区块B — 档案横向切换器

- **Logic-ID**: `profiles`, `selectedProfile`
- **职责**: 在多个档案之间快速切换——用户一瞥即知有几个档案、当前在看哪个
- **内容**: 每个档案条目包含
  - 头像占位（当前以固定图形表示，未来可扩展为自定义头像）
  - 档案昵称（`nickname` 或 "未命名"）
  - 当前选中项有选中标识

- **条目数量**: 最多 5 个档案条目 + 1 个"添加"按钮（当 `profiles.length < 5` 时显示）

- **数据来源**: `profiles` 数组（来自 `useProfile()`）

- **交互**:
  - 点击档案条目 → 切换 `selectedIdx`，触发加载该档案的详情和事件列表
  - 点击"添加" → 跳转 `edit.tsx?mode=create`
  - 横向可滚动（当档案数量较多时）

- **条件渲染**: 仅在 normal 态且有档案时显示。冷启动 / 加载态 / 错误态不显示

- **三态**:
  - Loading: 不显示（整页 loading 覆盖）
  - Empty（冷启动）: 不显示（走冷启动页）
  - Error: 不显示

- **data-testid**: `profile-switcher`, `profile-switcher-item-{n}`, `profile-switcher-item-add`

### 区块C — 档案信息卡片

- **Logic-ID**: `selectedDetail`, `selectedProfile`
- **职责**: 展示当前选中档案的核心信息——用户在此页了解"这个孩子的档案里有什么"

- **内容**:
  - 档案昵称（突出展示，`nickname` 或 "未命名"）
  - 年龄段（`age_range`，如"3-6岁"）
  - 三项核心标签（紧凑展示）：
    1. 出生日期（`birth_date`）
    2. 诊断类型（`diagnosis_type`）
    3. 主要行为类型（`primary_behavior`）
  - 主要行为标签在涉及自伤/攻击等高风险类型时，应有区别于普通标签的标识语义

- **数据来源**: `selectedDetail`（通过 `getProfile(id)` 获取的 `ProfileResponse`）

- **交互**: 无（纯信息展示）——编辑入口在导航栏

- **条件渲染**: `selectedDetail` 存在时才渲染。切换档案过程中，如果新档案详情尚未加载完成，保留上一个档案的卡片内容不闪烁

- **三态**:
  - Loading（详情加载中）: 保留上一个档案卡片或显示占位内容（同形区域）
  - Empty（无详情数据）: 不适用（档案存在则必有详情）
  - Error（详情加载失败）: 卡片区域展示简要错误提示，但不替换整页

- **data-testid**: `profile-info-card`, `profile-info-card-nickname`, `profile-info-card-age`, `profile-info-card-birth`, `profile-info-card-diagnosis`, `profile-info-card-behavior`

### 区块D — 事件时间线标题区

- **Logic-ID**: `events`, `showQuickRecord`
- **职责**: 展示事件记录数量统计 + 提供快速记录入口

- **内容**:
  - 标题："事件记录（共 N 条）"
  - 快速记录按钮（触发打开快速记录弹窗）

- **数据来源**: `events.length`（总条数统计）

- **交互**:
  - 点击快速记录按钮 → `setShowQuickRecord(true)`，打开底部弹窗（区块F）

- **条件渲染**: 仅在 normal 态且有 `selectedProfile` 时显示

- **三态**:
  - Loading: 事件加载中时标题仍显示，数量可为"…"或上次缓存数
  - Empty: 标题显示"事件记录（共 0 条）"
  - Error: 标题仍显示

- **data-testid**: `profile-timeline-header`, `profile-timeline-header-count`, `profile-timeline-header-add-btn`

### 区块E — 事件时间线

- **Logic-ID**: `events`, `eventsLoading`, `eventsHasMore`
- **职责**: 按时间倒序展示当前档案的所有行为/情绪事件记录

#### 事件条目内容

每条事件记录包含：
- **事件时间**：`event_time`，以相对时间格式展示（如"今天 14:30""昨天 09:15""3天前"）
- **行为类型**：`behavior_type`，以标签形式展示（如"自伤行为""攻击行为""情绪崩溃"等）
- **严重程度**：`severity_level`（轻/中/重）
- **专业评估标记**：`has_professional_note === true` 时，显示"已评估"标记

#### 条目视觉区分

不同行为类型在时间线上有不同的警示层级——高风险行为（自伤/攻击）比其他类型有更强的警示语义。

#### 时间线结构

条目按时间顺序排列，最新记录在顶部。

#### 空态

`events.length === 0` 且 `eventsLoading === false` 时：
- 显示"暂无事件记录"文案
- 辅助提示："点击右上角 + 记录孩子的第一次行为事件"
- 一个操作入口："+ 记录事件" → 打开快速记录弹窗

#### 翻页

- 滚动到底部自动加载下一页（每页约 20 条）
- 加载中：时间线底部显示"加载中…"
- 无更多：底部显示"已加载全部记录"
- 加载失败：底部显示"加载失败，点击重试"

#### 三态

| 态 | 行为 |
|----|------|
| Loading（首次加载） | 时间线区域显示占位条目（同形内容区），不显示空态 |
| Empty（无事件） | 显示空态引导（见上方） |
| Normal（有事件） | 正常渲染时间线条目 |
| Error（加载失败） | 显示"加载失败" + "重试"入口，不替换整个页面 |

- **data-testid**: `profile-timeline`, `profile-timeline-event-{n}`, `profile-timeline-event-time-{n}`, `profile-timeline-event-type-{n}`, `profile-timeline-event-severity-{n}`, `profile-timeline-event-eval-badge-{n}`, `profile-timeline-empty`, `profile-timeline-empty-cta-btn`, `profile-timeline-load-more`

### 区块F — 快速记录弹窗

- **Logic-ID**: `showQuickRecord`, `isSubmitting`
- **职责**: 让用户在档案列表页快速记录一条行为/情绪事件，无需跳转编辑页

- **出现方式**: 从页面底部出现，有遮罩层。点击遮罩关闭弹窗

#### 弹窗内容

**弹窗头部**：
- 下滑关闭提示区域
- 标题："记录行为事件"
- 副标题："完整记录有助于AI精准匹配案例"
- 关闭按钮

**Section 1 — 事件分类**：
- 行为类型（必填）：多选一的标签组，含自伤/攻击/逃跑/情绪崩溃/刻板行为/其他等选项
- 严重程度（必填）：三选一分段选择——"轻度"/"中度"/"重度"

**Section 2 — 发生场景**（可选）：
- 场景选择：多选一的标签组，含"家中""学校/机构""公共场所""户外""其他"

**Section 3 — 事件描述**：
- 触发因素（必填）：文本输入，"如：在超市遇到噪音刺激…"
- 具体表现（必填）：多行文本输入，"如：突然捂耳蹲下，持续约3分钟…"

**Section 4 — 干预记录**（可选，默认折叠）：
- 折叠标题"干预记录" + 展开箭头
- 展开后：
  - 尝试的干预措施：文本输入
  - 干预结果：文本输入

**弹窗底部**：
- 保存按钮："保存记录"
- 提交中按钮显示"保存中…"且不可点击
- 必填项未填完时按钮不可用

#### 交互

- 点击遮罩或关闭按钮 → 关闭弹窗（不保存）
- 必填项全部填写后按钮可用
- 点击保存 → 调用提交 → 关闭弹窗 → 重新加载事件列表
- 提交失败 → 弹窗保持打开，显示错误提示（非阻断式 toast）

#### 三态

| 态 | 行为 |
|----|------|
| Hidden | 弹窗不渲染 |
| Showing（填写中） | 弹窗渲染，按钮在必填项完善后可用 |
| Submitting | 按钮显示"保存中…"且禁用，其他表单仍可查看但不可编辑 |

- **data-testid**: `profile-quick-sheet`, `profile-quick-sheet-overlay`, `profile-quick-sheet-submit-btn`, `profile-quick-sheet-close-btn`, `profile-quick-sheet-collapse-intervention`

## 冷启动状态

### 触发条件

`isLoading === false` 且 `profiles.length === 0`

### 内容

- 一个装饰性图形元素（帮助缓和"空"的感受）
- 标题："创建孩子的第一份档案"
- 说明文字："完善的档案能帮助 AI 更精准地匹配与您孩子情况相似的真实干预案例"
- 一个操作入口："创建第一个档案" → 跳转 `edit.tsx?mode=create`

### 行为

- 此状态下不渲染档案切换器、信息卡片、事件时间线
- 导航栏仍显示品牌标识
- 操作入口触控可达

### data-testid

`profile-cold-start`, `profile-cold-start-graphic`, `profile-cold-start-title`, `profile-cold-start-subtitle`, `profile-cold-start-cta-btn`

---

## 加载状态

### 触发条件

`isLoading === true` 且 `profiles.length === 0`（首次加载）

### 内容

- 档案切换器占位区域（同形占位）
- 档案卡片占位区域（同形占位）
- 加载文字提示

### 行为

- 区别于冷启动——有占位形状暗示此处将有内容
- 数据到达后自动切换到 cold-start 或 normal 态

### data-testid

`profile-index-loading`, `profile-index-loading-skeleton`

---

## 错误状态

### 触发条件

`error !== null` 且 `profiles.length === 0`（首次加载失败）

### 内容

- 错误示意元素
- 标题："加载失败"
- 辅助说明：错误信息或默认提示
- 重试操作入口

### 行为

- 部分加载失败（如档案列表加载成功但事件列表加载失败）：不替换整页，在对应区块显示重试入口
- 全部失败（档案列表未加载）：显示全页错误态 + 重试按钮
- 不使用 toast/alert 阻断式弹窗

### data-testid

`profile-index-error`, `profile-index-error-retry-btn`

---

## 下拉刷新

- 用户在页面顶部下拉触发
- 同时刷新：档案列表（`fetchProfiles()`）+ 当前选中档案的详情和事件列表
- 刷新完成后保持在当前选中的档案
- 刷新指示在数据返回后消失
- 刷新期间不阻止页面交互

## 交互行为汇总

| 行为 | 触发方式 | 结果 |
|------|---------|------|
| 切换档案 | 点击切换器条目 | 加载该档案详情 + 事件列表 |
| 编辑档案 | 点击导航栏"编辑" | 跳转编辑页 |
| 创建档案 | 点击切换器"添加" / 冷启动按钮 | 跳转创建页 |
| 快速记录 | 点击时间线标题区 + 按钮 | 打开底部弹窗 |
| 提交事件 | 点击弹窗"保存记录" | 关闭弹窗 → 重新加载事件列表 |
| 关闭弹窗 | 点击遮罩 / 关闭按钮 | 关闭弹窗，不保存 |
| 展开干预 | 点击弹窗"干预记录"折叠区 | 展开/收起干预措施输入区 |
| 事件加载更多 | 滚动到时间线底部 | 自动加载下一页事件 |
| 下拉刷新 | 页面顶部下拉 | 刷新档案列表 + 当前档案详情 + 事件列表 |
| 重试（档案） | 点击错误态"重试" | 重新调用 `fetchProfiles()` |
| 重试（事件） | 点击事件区"点击重试" | 重新加载当前页事件 |

## 文件拆分

```
profiles/pages/
├── index.tsx              ← 接线容器
│   → 导入 useProfile Hook、useQuickRecord Hook
│   → 管理 selectedIdx、冷启动/loading/error/normal 四态路由
│   → 管理事件列表分页状态
│   → 通过 Props 注入数据到子组件
├── index.config.ts        ← 页面配置
└── components/
    ├── ProfileSwitcher.tsx        ← 横向档案切换器（区块B）
    │   → 接收 profiles: ProfileListItem[], selectedIdx, 最大数量
    │   → 通过 onSelect(idx) / onAdd() 回调上报
    ├── ProfileInfoCard.tsx        ← 档案信息卡片（区块C）
    │   → 接收 ProfileResponse
    │   → 纯展示组件
    ├── EventTimeline.tsx          ← 事件时间线（区块D+E）
    │   → 接收 events, isLoading, hasMore, onLoadMore
    │   → 管理空态/正常态/加载更多
    ├── EventTimelineItem.tsx      ← 单条事件条目
    │   → 接收 EventListItem
    │   → 根据 behavior_type 决定视觉语义等级
    ├── QuickRecordSheet.tsx       ← 快速记录弹窗（区块F）
    │   → 接收 visible, form, isSubmitting
    │   → 通过 onFieldChange / onSubmit / onClose 回调上报
    ├── ProfileColdStart.tsx       ← 冷启动引导页
    ├── ProfileLoading.tsx         ← 加载骨架
    └── ProfileError.tsx           ← 错误态
```

**原则**:
- `index.tsx` 是唯一接线点 — 管理四态路由、档案切换、事件分页
- 所有子组件为纯展示型 — 不 import 任何 Hook，数据通过 Props 接收
- 快速记录弹窗的表单状态由 `useQuickRecord` Hook 管理

## 与应急咨询模块的内容关联

| 内容模式 | 应急咨询 | 档案列表 | 说明 |
|---------|---------|---------|------|
