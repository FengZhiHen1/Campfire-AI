## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 咨询详情（consult-detail） |
| 所属页面 | 应急咨询（consult） |
| 页面职责 | 展示一条历史咨询记录的完整内容：用户查询摘要 + AI 生成方案 + 元数据 + 操作入口 |
| 入口来源 | 咨询历史列表（`history.tsx`）通过路由跳转，携带 `id` 参数 |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `consultationId` | `string` | 路由参数 `id` | 咨询记录唯一标识 |
| `isLoading` | `boolean` | 组件内部状态 | 数据加载中 |
| `hasError` | `string \| null` | 组件内部状态 | 加载错误信息 |
| `detailData` | `ConsultationHistoryDetail \| null` | `consultApi.fetchHistoryDetail()` | 详情全部数据 |
| `crisisLevel` | `CrisisLevel` | `detailData.crisis_level` | 危机等级 |
| `behaviorDescription` | `string` | `detailData.behavior_description` | 用户原始行为描述 |
| `consultationTime` | `string` | `detailData.consultation_time` | 咨询发生时间 |
| `generatedPlan` | `string` | `detailData.generated_plan` | AI 生成的方案全文 |
| `planSections` | 派生数据 | `generatedPlan` 解析结果 | 四段式结构化方案（优先使用后端结构化字段，旧数据回退为段落列表） |
| `disclaimer` | `string` | `detailData.disclaimer` | 合规免责声明 |
| `generationTimeMs` | `number` | `detailData.generation_time_ms` | 生成耗时（毫秒） |
| `finishReason` | `string` | `detailData.finish_reason` | 流终止原因（COMPLETE/PARTIAL/BLOCKED/TIMEOUT/ERROR） |
| `isPartial` | `boolean` | `detailData.is_partial` | 是否不完整生成 |
| `referencedSliceIds` | `string[]` | `detailData.referenced_slice_ids` | 引用案例切片 ID 列表 |
| `sourceList` | `string[]` | `detailData.source_list` | 案例来源引用列表 |

**状态路由**：详情页自身管理 loading / error / normal 三态，无外部状态机依赖。

## 内容区块

### 区块A — 导航栏

- **Logic-ID**: 无（原生渲染）
- **职责**: 页面标识 + 返回入口
- **内容**: 标题"咨询详情"
- **交互**: 原生导航栏自带返回按钮，返回历史列表
- **三态**: 不适用（原生组件，静态）

### 区块B — 危机概要头部

- **Logic-ID**: `crisisLevel`, `behaviorDescription`, `consultationTime`
- **职责**: 一瞥即知咨询的基本情况——多严重、什么行为、何时发生
- **内容**:
  - 危机等级标识文字（"轻度"/"中度危机"/"重度危机"），与主咨询页危机等级标识语义一致
  - 咨询发生时间
  - 用户原始行为描述（完整展示，详情页有充分空间）
- **数据来源**: `detailData.crisis_level` → 等级文字；`detailData.consultation_time` → 时间；`detailData.behavior_description` → 描述
- **交互**: 无（纯信息展示）
- **三态**:
  - Loading: 显示占位内容（内容区域有形状提示但无具体文字），让用户知道此处将有信息
  - Empty: 不适用（必要字段）
  - Error: 不渲染（走错误态页面）
- **data-testid**: `detail-header`, `detail-header-crisis-badge`, `detail-header-time`, `detail-header-behavior`

### 区块C — 结构化方案卡片（核心内容区）

- **Logic-ID**: `generatedPlan`, `planSections`（派生）, `isPartial`
- **职责**: 展示 AI 生成的完整应急方案——用户打开此页面的唯一目的

#### 内容解析策略

对方案数据进行尽力解析：
1. 优先使用后端结构化字段 `plan_sections`（Record<string, string[]>），按四段标题匹配
2. 若无结构化数据，尝试从 `generated_plan` 文本中按四段标题关键词（"即时安全干预"/"情绪安抚"/"观察指标"/"就医判断"）拆分
3. 解析成功 → 每段独立展示，各有明确的段标题
4. 解析失败 → 回退为按空行拆分的纯文本段落列表，无段标题

#### 四段式内容（解析成功时）

| 段名 | 内容特征 |
|------|---------|
| 即时安全干预 | 操作步骤列表 |
| 情绪安抚话术 | 首句为可直接使用的安抚引语（区别于后续说明性文字） |
| 后续观察指标 | 观察项列表 |
| 就医判断标准 | 就医触发条件列表（可能为空） |

#### 回退模式（解析失败时）

- 标题 "应急干预方案"
- 按空行拆分为段落
- 无独立段标题
- 无段间特殊区分

#### 部分生成标记

`isPartial === true` 时，在方案区显示 "部分生成" 提示标记，告知用户此方案可能不完整。

#### 底部信息栏

- 案例引用数（来自 `sourceList.length`）
- 生成耗时（来自 `generationTimeMs`，转换为可读格式如 "X.Xs"）

#### 三态

| 态 | 行为 |
|----|------|
| Loading | 显示占位内容——各段落区域有形状提示但无具体文字 |
| Empty | 显示 "暂无方案内容" 占位提示 |
| Error | 不适用（走错误态页面） |

- **data-testid**: `detail-plan-card`, `detail-plan-card-partial-warning`, `detail-plan-section-{n}`, `detail-plan-card-footer`

### 区块D — 参考案例引用

- **Logic-ID**: `referencedSliceIds`, `sourceList`
- **职责**: 展示方案引用的案例来源——提供可溯源性
- **内容**: 每条案例一行，展示案例编号/标识（如 `CASE-2024-0087`）
- **数据来源**: `sourceList` 或 `referencedSliceIds`
- **条件渲染**: `referencedSliceIds.length > 0` 时才展示
- **交互**: 点击案例编号跳转案例卡片详情页
- **三态**:
  - Loading: 不单独处理（Loading 态由方案卡片骨架覆盖）
  - Empty: 无引用时整个区块不渲染
  - Error: 不适用
- **data-testid**: `detail-ref-cases`, `detail-ref-case-{n}`

### 区块E — 免责声明

- **Logic-ID**: `disclaimer`
- **职责**: 合规提示
- **内容**: `disclaimer` 字段全文。若 disclaimer 为空，使用兜底免责文案
- **数据来源**: `detailData.disclaimer`
- **交互**: 无（纯信息展示）
- **三态**:
  - Loading: 显示占位内容（1-2 行文字区域提示）
  - Empty: 显示硬编码的兜底免责文案
  - Error: 不适用
- **data-testid**: `detail-disclaimer`

### 区块F — 底部元信息

- **Logic-ID**: `generationTimeMs`, `finishReason`, `isPartial`
- **职责**: 展示技术元信息，供了解方案生成质量
- **内容**: 多个标签项
  - "生成耗时 X.Xs"
  - 终止原因（COMPLETE / PARTIAL / BLOCKED / TIMEOUT / ERROR）
  - "部分生成"（仅 `isPartial === true` 时显示）
- **数据来源**: `generationTimeMs`, `finishReason`, `isPartial`
- **交互**: 无（纯信息展示）
- **三态**:
  - Loading: 显示标签占位
  - Empty: 不适用（必要字段）
  - Error: 不渲染
- **data-testid**: `detail-meta-footer`, `detail-meta-generation-time`, `detail-meta-finish-reason`

### 区块G — 固定操作栏

- **Logic-ID**: 无（纯导航操作）
- **职责**: 提供返回主咨询页的入口
- **内容**: 操作入口 "开始新咨询"
- **交互**: 点击后跳转到主咨询页（`index.tsx`），替换当前页面栈
- **始终可见**：不随页面内容滚动而消失，加载失败也保留入口
- **三态**:
  - Loading: 操作入口不可用
  - Empty: 始终显示
  - Error: 始终显示
- **data-testid**: `detail-action-bar`, `detail-action-new-btn`

## 错误状态

### 加载失败

- **触发**: `fetchHistoryDetail(id)` 请求失败
- **内容**:
  - 错误图形元素
  - 标题："加载失败"
  - 错误详情文字：`hasError` 的具体信息
  - 两个操作入口：
    - "重试" → 重新调用 `fetchHistoryDetail(id)`
    - "返回列表" → 返回上一页
- **不使用** toast/alert 阻断式弹窗
- **data-testid**: `detail-error`, `detail-error-graphic`, `detail-error-message`, `detail-error-retry-btn`, `detail-error-back-btn`

## 与主咨询页的内容关联

以下内容模式与主咨询页语义一致，在 OD 设计时应保持统一的信息传达方式：

| 内容模式 | 主咨询页 | 详情页 | 说明 |
|------|---------|--------|------|
| 危机等级标识 | 导航栏内 | 概要头部 | 同样的三档等级体系，语义一致 |
| 四段式方案 | PlanCard 内（流式填充） | 独立展示（静态报告） | 内容结构一致，展示方式因场景不同（动态 vs 静态） |
| 安抚话术引用句 | 首句引用突出 | 首句引用突出 | 内容区分规则一致 |
| 免责声明 | 页面底部常驻条 | 内容区内 | 内容语义一致 |

## 平台行为差异

| 行为 | miniprogram |
|------|------------|
| 导航 | 原生导航栏，标题居中，自带返回按钮 |
| 滚动 | 页面内容区可滚动（scroll-view），操作栏固定在底部 |
| 安全区 | 底部预留 safe-area-inset-bottom。内容区滚动底部预留空间，防止被固定操作栏遮挡 |

## 文件拆分

```
consult/pages/
├── detail.tsx           ← 接线容器
│   → 管理 loading / error / normal 三态
│   → 调用 consultApi.fetchHistoryDetail(id)
│   → 解析 generated_plan 为结构化段落
│   → 通过 Props 注入数据到子组件
├── detail.config.ts     ← 页面配置
└── components/
    ├── DetailHeader.tsx       ← 危机概要头部（区块B）
    ├── DetailPlanCard.tsx     ← 结构化方案卡片（区块C）
    │   → 四段式解析逻辑
    │   → 解析成功：段标题 + 内容
    │   → 解析失败：回退为纯文本段落
    ├── DetailRefCases.tsx     ← 参考案例引用（区块D）
    ├── DetailDisclaimer.tsx   ← 免责声明（区块E）
    ├── DetailMetaFooter.tsx   ← 底部元信息（区块F）
    └── DetailActionBar.tsx    ← 固定操作栏（区块G）
```

**原则**:
- `detail.tsx` 是唯一接线点 — 管理数据获取和状态路由
- 所有子组件为纯展示型 — 数据和回调通过 Props 接收
- 与主咨询页共享的内容模式（四段式方案、危机等级标识、安抚话术引用）应在实现层通过共享逻辑复用
