# 应急咨询 — 咨询详情页 页面骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#02-智能应急咨询`、`apps/mini-program/src/views/consult/pages/detail.tsx`（现有实现）
> 目标平台：miniprogram (750rpx 基准)
> 数据来源：`consultApi.fetchHistoryDetail(id)` → `ConsultationHistoryDetail`
> 设计方向：延续主咨询页仪表盘语言 — 静态报告风格，方案内容是唯一主角

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram（750rpx 基准） |
| 视图名称 | 咨询详情（consult-detail） |
| 所属页面 | 应急咨询（consult） |
| 页面职责 | 展示一条历史咨询记录的完整内容：用户查询摘要 + AI 生成方案 + 元数据 + 操作入口 |
| 入口来源 | 咨询历史列表（`history.tsx`）通过 `Taro.navigateTo({ url: '/views/consult/pages/detail?id=xxx' })` |
| 视觉方向 | 报告风格 — 延续主咨询页的 credit bar + pill 徽章语言，方案采用四段式结构化排版 |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `consultationId` | `string` | 路由参数 `id` | 咨询记录唯一标识 |
| `isLoading` | `boolean` | 组件内部 `useState` | 数据加载中 |
| `hasError` | `string \| null` | 组件内部 `useState` | 加载错误信息 |
| `detailData` | `ConsultationHistoryDetail \| null` | `consultApi.fetchHistoryDetail()` | 详情全部数据 |
| `crisisLevel` | `CrisisLevel` | `detailData.crisis_level` | 危机等级 |
| `behaviorDescription` | `string` | `detailData.behavior_description` | 用户原始行为描述 |
| `consultationTime` | `string` | `detailData.consultation_time` | 咨询发生时间 |
| `generatedPlan` | `string` | `detailData.generated_plan` | AI 生成的方案全文 |
| `planSections` | 派生数据 | `generatedPlan` 解析结果 | 四段式结构化方案（努力解析，失败则回退为段落列表） |
| `disclaimer` | `string` | `detailData.disclaimer` | 合规免责声明 |
| `generationTimeMs` | `number` | `detailData.generation_time_ms` | 生成耗时 |
| `finishReason` | `string` | `detailData.finish_reason` | 流终止原因 |
| `isPartial` | `boolean` | `detailData.is_partial` | 是否不完整生成 |
| `referencedSliceIds` | `string[]` | `detailData.referenced_slice_ids` | 引用案例切片 ID 列表 |
| `sourceList` | `string[]` | `detailData.source_list` | 案例来源引用列表 |

**状态路由**：详情页自身管理 loading / error / normal 三态，无外部状态机依赖。

- **导航**: 原生导航栏，居中标题 "咨询详情"

- **参考案例**: `referencedSliceIds.length > 0` 时显示可点击案例列表
- **固定按钮**: 底部固定，不随内容滚动。内容区 scroll-view 底部预留 padding 防止遮挡
- **返回**: 原生导航栏自带返回按钮

- **骨架策略**: 同形骨架 — 骨架矩形与其对应的真实内容区域形状一致（同宽度、同高度），用 `surface-variant` 色 + `cf-pulse-border` 动画
- **不使用**居中 spinner — 与主咨询页 submitting 态一致，用骨架暗示内容即将到达
- **按钮**: 加载期间禁用并显示为 surface-dim 占位，防止用户点击

## 功能块详述

### 区块A — 导航栏

- **Logic-ID**: 无（纯原生渲染）
- **职责**: 页面标识 + 返回入口
- **标题**: "咨询详情"，居中
- **返回**: 原生导航栏自带返回按钮，返回历史列表
- **三态**: 不适用（原生组件，静态）

### 区块B — 危机概要头部

- **Logic-ID**: `crisisLevel`, `behaviorDescription`, `consultationTime`
- **职责**: 一瞥即知咨询的基本情况 — 多严重、什么行为、何时
- **形态**: `surface` 背景卡片, 16rpx 圆角, 32rpx padding, 左右 32rpx margin

**内容结构**:
```
[中度危机 ●]                    2026-06-28 14:30
"孩子在商场突然蹲下尖叫，持续约5分钟…"
```

- **危机等级徽章**: pill 形状，与主咨询页导航栏危机徽章同款，三色：
  | 等级 | crisisLevel | 背景色 token | 文字 |
  |------|-------------|-------------|------|
  | 轻度 | `mild` | `$cf-tertiary` | 轻度 ● |
  | 中度 | `moderate` | `$cf-primary` | 中度危机 ● |
  | 重度 | `severe` | `$cf-error` | 重度危机 ● |
- **时间**: `footnote` 字号，`on-surface-variant` 色，右对齐
- **行为描述**: `body-md`，`on-background` 色，完整展示（不限行数，详情页有充分空间）
- **三态**:
  - Loading: 同形骨架（pill 占位 + 时间占位 + 2 行文字占位）
  - Empty: 不适用（必要字段）
  - Error: 不渲染（走错误态页面）

### 区块C — 结构化方案卡片 (HERO)

- **Logic-ID**: `generatedPlan`, `planSections`（派生）, `isPartial`
- **职责**: 展示 AI 生成的完整应急方案 — 用户打开此页面的唯一目的

#### 结构化解析策略

对 `generated_plan` 文本进行尽力解析：
1. 尝试按四段标题关键词（"即时安全干预" / "情绪安抚" / "观察指标" / "就医判断"）拆分
2. 解析成功 → 每段独立展示，配备对应 accent bar 和标题
3. 解析失败 → 回退为按空行拆分的纯文本段落列表，无 accent bar

#### 四段式规范（解析成功时）

| 段名 | 左侧 accent bar | 标题色 token | 正文规格 |
|------|----------------|-------------|---------|
| 即时安全干预 | `tertiary` 4rpx | `$cf-tertiary` | body-md, 1.8 line-height |
| 情绪安抚话术 | `primary` 4rpx | `$cf-primary` | italic + 引用块样式（首句） |
| 后续观察指标 | `secondary` 4rpx | `$cf-secondary` | body-md, bullet list |
| 就医判断标准 | `error`/`secondary` 4rpx | `$cf-error`/`$cf-secondary` | body-md（仅需就医时红色） |

**安抚话术特殊处理**: 第一句作为引用块（`surface-variant` 背景 + 4rpx `primary` 左竖条, italic），后续句子为普通列表

#### 回退模式（解析失败时）

- 标题 "应急干预方案"
- 按 `\n\n+` 拆分为段落
- 每段间 32rpx 间距
- 无 accent bar
- 无段落标题

#### 部分生成标记

`isPartial === true` 时，在卡片标题右侧显示 warning pill "部分生成"（`error-container` 背景，`on-error-container` 文字）

#### 卡片外壳

- `surface` 背景, 16rpx 圆角, `diffuse shadow`, 32rpx 内边距, 左右 32rpx margin

#### 底部信息栏

- ghost border 分割线
- 案例引用数 pill（如 "基于 4 个案例"，来自 `sourceList.length`）
- 生成耗时（来自 `generationTimeMs`）

#### 三态

| 态 | 行为 |
|----|------|
| Loading | 同形骨架 — 4 段矩形占位（高度与真实段落接近）+ accent bar 占位 |
| Empty | "暂无方案内容" 浅色占位文字（`on-surface-variant` 色, 居中） |
| Error | 不适用（走错误态页面） |

### 区块D — 参考案例引用

- **Logic-ID**: `referencedSliceIds`, `sourceList`
- **职责**: 展示方案引用的案例来源 — 提供可溯源性
- **条件渲染**: `referencedSliceIds.length > 0`
- **形态**: 紧凑列表卡片
  - 每条一行: 案例 ID 标签（如 `CASE-2024-0087`）
  - `surface` 背景, 16rpx 圆角, 左右 32rpx margin
- **交互**: 点击案例跳转 `card-detail?cardId={case_id}`
- **三态**:
  - Loading: 不单独显示（骨架已覆盖）
  - Empty: 无引用时整个区块不渲染
  - Error: 不适用

### 区块E — 免责声明

- **Logic-ID**: `disclaimer`
- **职责**: 合规提示
- **形态**: `primary-container` 背景卡片, 16rpx 圆角, 24rpx padding, 左右 32rpx margin
- **文字**: `disclaimer` 字段全文, `label-md` 字号, `on-primary-container` 色, 前置 ℹ️ 图标（非 emoji，使用文本符号或 CSS 绘制）
- **三态**:
  - Loading: 同形骨架（1-2 行文字占位）
  - Empty: disclaimer 为空时仍显示硬编码的兜底免责文案
  - Error: 不适用

### 区块F — 底部元信息

- **Logic-ID**: `generationTimeMs`, `finishReason`, `isPartial`
- **职责**: 技术元信息展示，供了解方案生成质量
- **形态**: pill 标签行，flex-wrap 布局
- **标签**:
  - "生成耗时 X.Xs" — `surface-variant` 背景 pill
  - "{finishReason}" — `surface-variant` 背景 pill（COMPLETE / PARTIAL / BLOCKED / TIMEOUT / ERROR）
  - "部分生成" — 仅 `isPartial === true` 时显示，`error-container` 背景
- **三态**:
  - Loading: 2-3 个 pill 占位
  - Empty: 不适用（必要字段）
  - Error: 不渲染

### 区块G — 固定操作栏

- **Logic-ID**: 无（纯导航操作）
- **职责**: 提供返回主咨询页的入口
- **形态**: `surface` 背景, 固定底部, 顶部 1px ghost border 分割线
- **按钮**: "开始新咨询", `primary` 背景, 全宽 96rpx 高, 32rpx 圆角, 左右 32rpx margin
- **点击**: `Taro.redirectTo({ url: '/views/consult/pages/index' })` — 用 redirectTo 替换当前页面栈
- **三态**:
  - Loading: 禁用态，surface-dim 背景占位
  - Empty: 始终显示（即使加载失败也保留入口）
  - Error: 始终显示

## 和主咨询页的视觉组件复用

以下组件可直接共享或仅微调 Props：

| 组件 | 主咨询页 | 详情页 | 差异 |
|------|---------|--------|------|
| 危机徽章 (pill) | 导航栏内 | 概要头部 | 尺寸/颜色 token 一致，仅位置不同 |
| 方案段 (accent bar + 标题 + 内容) | PlanCard 内部 | 独立渲染 | streaming 动画移除，其余一致 |
| 安抚话术引用块 | PlanCard 内部 | 独立渲染 | 一致 |
| 免责声明 | DisclaimerBar | 内联渲染 | 背景 token 可能不同（主: surface-dim vs 详情: primary-container）— 待 OD 设计统一 |

**建议**: 将 `ConfidenceBadge` 的 pill 形状样式 + `PlanCard` 的段落渲染逻辑抽取为共享组件，或至少在 SCSS 层通过 `@mixin` 共享样式 token。

## 文件拆分方案

```
consult/pages/
├── detail.tsx                 ← 接线容器（≤120 行）
│   → 管理 loading / error / normal 三态
│   → 调用 consultApi.fetchHistoryDetail(id)
│   → 解析 generated_plan 为结构化段落
│   → 通过 Props 注入数据到子组件
├── detail.config.ts           ← 页面配置（已存在）
└── components/
    ├── DetailHeader.tsx        ← 危机概要头部（区块B, ≤60 行）
    ├── DetailPlanCard.tsx      ← 结构化方案卡片（区块C, ≤150 行）
    │   → 四段式解析逻辑
    │   → 解析成功: accent bar + 段落标题
    │   → 解析失败: 回退为纯文本段落
    ├── DetailRefCases.tsx      ← 参考案例引用（区块D, ≤60 行）
    ├── DetailDisclaimer.tsx    ← 免责声明（区块E, ≤50 行）
    ├── DetailMetaFooter.tsx    ← 底部元信息（区块F, ≤50 行）
    └── DetailActionBar.tsx     ← 固定操作栏（区块G, ≤50 行）
```

**原则**:
- `detail.tsx` 是唯一接线点 — 管理数据获取和状态路由
- 所有子组件为纯展示型 — 数据和回调通过 Props 接收
- 每个组件 ≤ 150 行
- 与主咨询页共享的视觉模式（accent bar、pill 徽章、引用块）通过 SCSS mixin 复用

## 视觉验证输入

### 截图关键点

| 编号 | 状态/场景 | 关键验证点 | 前置条件 |
|------|----------|-----------|---------|
| SS-1 | loading | 同形骨架（非居中 spinner）、骨架形状与真实内容区一致 | 页面加载中 |
| SS-2 | normal-structured | 四段式 accent bar 正确渲染、危机徽章三色正确、安抚话术 italic + 引用块 | generated_plan 可解析为四段 |
| SS-3 | normal-fallback | 回退为纯文本段落展示、无 accent bar | generated_plan 无法解析为四段 |
| SS-4 | normal-partial | "部分生成" warning pill 可见 | is_partial: true |
| SS-5 | no-references | 参考案例区不渲染 | referencedSliceIds: [] |
