# 案例管理 — 叙事提交 内容骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#04-真实案例库管理`（CASE-01/08/09）
> 目标平台：miniprogram（触控）
> 数据来源：`useNarrativeSubmit()` Hook

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 叙事提交（cases-narrative-submit） |
| 所属页面 | 案例管理（cases） |
| 页面职责 | 提交原始干预叙事文本——填写标题、来源类型、正文，提交后触发 AI 提取卡片 |
| 入口来源 | 案例列表 FAB / 叙事编辑页 |

### Logic-ID 清单

| ID | 类型 | 用途 |
|----|------|------|
| `title` | `string` | 叙事标题（≤100 字） |
| `sourceType` | `string` | 来源类型 |
| `narrative` | `string` | 叙事正文（≤5000 字） |
| `submitting` | `boolean` | 提交中 |
| `extracting` | `boolean` | AI 提取卡片进行中 |
| `tipsExpanded` | `boolean` | 写作提示展开 |
| `canSubmit` | `boolean` | 标题非空 + 正文非空 |

---

## 内容区块

### 状态路由

| 状态 | 展示 |
|------|------|
| `extracting` | 提取中动画 + "正在分析叙事，提取干预卡片..." + 预计时间 |
| Normal | 表单页 |

### 区块A — 叙事标题
- **职责**: 填写案例标题
- **内容**: 文本输入框，占位"请输入案例标题，如：ASD 商场感官过载干预案例"，字数计数（/100）
- **data-testid**: `ns-input`

### 区块B — 来源类型
- **职责**: 选择案例来源
- **内容**: 按钮组选择（专家撰写/机构合作等）
- **data-testid**: `ns-source-btn--active`

### 区块C — 写作提示（折叠）
- **职责**: 指导用户如何写好叙事——提高 AI 提取质量
- **内容**: 折叠标题"写作提示" + 展开箭头；展开后显示多条提示（如"包含场景描述""描述干预动作与结果"等）
- **data-testid**: `ns-tips`

### 区块D — 叙事正文
- **职责**: 填写干预叙事的完整文本
- **内容**: 大文本输入区，占位提示引导用户按四段式结构描述，字数计数（/5000）
- **data-testid**: `ns-textarea`

### 区块E — 底部操作栏
- **职责**: 草稿保存 + 提交提取
- **内容**: "保存草稿"按钮 + "提取卡片"按钮（提交后触发 AI 提取）
- **提交中**: "提取卡片"→"提交中..."
- **data-testid**: `ns-footer`, `ns-footer__draft`, `ns-footer__submit`

### data-testid 节点清单

```
ns-input, ns-section__counter
ns-source-btn--active
ns-tips, ns-tips__header, ns-tips__content
ns-textarea, ns-textarea__counter
ns-footer, ns-footer__draft, ns-footer__submit
```
