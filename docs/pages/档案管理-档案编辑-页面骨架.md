# 档案管理 — 档案编辑 内容骨架

> 生成日期：2026-06-28
> 源文档：`docs/功能设计/功能模块全拆解.md#03-个性化档案`（PROF-01/03/04/06）
> 目标平台：miniprogram（触控）
> 数据来源：`useProfile()` Hook（`getProfile` / `createProfile` / `updateProfile` / `deleteProfile`）

---

## 元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | miniprogram |
| 视图名称 | 档案编辑（profiles-edit） |
| 所属页面 | 档案管理（profiles） |
| 页面职责 | 创建新档案或编辑已有档案——填写基础信息、配置标签、管理事件记录、删除档案 |
| 入口来源 | 档案列表页 → `edit.tsx?mode=create` 或 `edit.tsx?mode=edit&profileId={id}` |
| 出口目标 | 保存/取消/删除 → 返回档案列表页 |

### Logic-ID 清单

| ID | 类型 | 数据来源 | 用途 |
|----|------|---------|------|
| `mode` | `'create' \| 'edit'` | 路由参数 | 决定页面是创建还是编辑模式 |
| `profileId` | `string` | 路由参数 | 编辑模式下目标档案 ID |
| `loading` | `boolean` | 组件内部状态 | 编辑模式下的初始数据加载 |
| `saving` | `boolean` | 组件内部状态 | 保存提交中 |
| `saveError` | `string \| null` | 组件内部状态 | 保存失败的错误信息 |
| `nickname` | `string` | 表单状态 | 档案昵称 |
| `birthDate` | `string` | 表单状态 | 出生日期（YYYY-MM-DD） |
| `diagnosisType` | `string` | 表单状态 | 诊断类型 |
| `primaryBehavior` | `string` | 表单状态 | 主要行为类型 |
| `languageLevel` | `string` | 表单状态（新增） | 语言水平（选填） |
| `medicationNotes` | `string` | 表单状态（新增） | 用药备注（选填） |
| `selectedTags` | `string[]` | 表单状态 | 已选感官特征 + 触发因素标签 |
| `customTags` | `string[]` | 表单状态 | 用户自定义触发因素标签 |
| `customTagInput` | `string` | 表单状态 | 自定义标签输入框文本 |
| `errors` | `Record<string, string>` | 表单校验结果 | 字段级错误信息 |
| `isDirty` | `boolean` | 派生状态 | 是否有未保存的修改 |
| `events` | `EventListItem[]` | `listEvents(profileId)` | 事件记录列表（编辑模式） |
| `eventsLoading` | `boolean` | 组件内部状态 | 事件列表加载中 |
| `eventsExpanded` | `boolean` | 组件内部状态 | 事件区域是否展开（默认折叠） |
| `menuVisible` | `boolean` | 组件内部状态 | 更多菜单是否可见 |
| `showDeleteModal` | `boolean` | 组件内部状态 | 删除确认弹窗可见 |
| `deleteConfirmName` | `string` | 组件内部状态 | 删除确认输入框文本 |

**状态路由**：本页管理 loading / normal 两态（normal 内含 form-editing / saving / save-error / delete-modal 子态）。

## 导航关系

```
档案列表页（index.tsx）
    │
    ├── mode=create → 档案编辑（本页，新建模式）
    │      │ 保存成功 / 取消 → 返回列表页
    │      ▼
    │
    ├── mode=edit&profileId={id} → 档案编辑（本页，编辑模式）
    │      │ 保存成功 / 取消 / 删除 → 返回列表页
    │      ▼
    │
    └── 返回列表页 → 列表页自动刷新数据
```

## 内容区块

### 区块A — 导航栏

- **Logic-ID**: `mode`, `saving`, `isDirty`, `menuVisible`
- **职责**: 页面标识 + 取消/保存操作 + 更多菜单（编辑模式）

- **内容**:
  - 左侧："取消"按钮
  - 中间：标题——创建模式"新建档案"，编辑模式"编辑档案"
  - 右侧：
    - 保存按钮（始终可见，`canSave === false` 时不可用）
    - 编辑模式下：更多按钮（"···"），点击展开菜单面板

- **更多菜单**（编辑模式）：
  - 菜单面板包含一个选项："删除此档案"（危险操作，需警示语义）
  - 点击菜单面板外部或再次点击"···"关闭菜单

- **交互**:
  - 点击"取消"：
    - 如果 `isDirty === true` → 弹出未保存确认对话框
    - 如果 `isDirty === false` → 直接返回
  - 点击保存 → 校验表单 → 提交 → 成功自动返回列表页
  - 点击"删除此档案" → 关闭菜单 → 打开删除确认弹窗

- **三态**:
  - Loading（编辑模式初始加载中）: 标题不变，保存按钮不可用
  - Saving: 保存按钮显示"保存中…"且不可点击
  - Error: 不适用（保存错误由区块C通知条承载）

- **data-testid**: `profile-edit-navbar`, `profile-edit-navbar-cancel`, `profile-edit-navbar-title`, `profile-edit-navbar-save`, `profile-edit-navbar-menu`, `profile-edit-navbar-menu-delete`

### 区块B — 新建提示条（仅创建模式）

- **Logic-ID**: `mode`
- **职责**: 告知用户必填字段要求
- **内容**: 固定文案——"请填写孩子的基本信息（*为必填）"
- **条件渲染**: 仅 `mode === 'create'` 时展示
- **三态**: 不适用（纯静态）
- **data-testid**: `profile-edit-tip`

### 区块C — 保存失败通知条

- **Logic-ID**: `saveError`
- **职责**: 告知用户保存失败，不阻断页面操作
- **内容**: 展示 `saveError` 错误信息文字 + 关闭按钮
- **出现方式**: 保存失败时从页面顶部插入（非弹窗）
- **消失方式**: 用户点击关闭按钮；或自动在数秒后消失
- **条件渲染**: `saveError !== null` 时展示
- **data-testid**: `profile-edit-save-error`, `profile-edit-save-error-text`, `profile-edit-save-error-close`

### 区块D — 基础信息区

- **Logic-ID**: `nickname`, `birthDate`, `diagnosisType`, `primaryBehavior`, `languageLevel`, `medicationNotes`, `errors`

- **职责**: 收集档案的核心基础信息

- **内容**:

#### D1 — 头像区域
  - 固定占位图形（MVP 阶段不支持上传）
  - 编辑指示标记（暗示可编辑，为后续版本预留）
  - 交互：无——点击无响应

#### D2 — 档案昵称（必填）
  - 文本输入框
  - 占位提示："如：小明"
  - 最大字符数限制（≤10 字）
  - 校验错误时在输入框下方显示红色错误文案

#### D3 — 出生日期（必填）
  - 原生日期选择器
  - 未选择时显示占位文案"请选择日期"
  - 日期范围：不晚于今天
  - 校验错误时在下方显示错误文案

#### D4 — 诊断类型（必填）
  - 原生单列选择器
  - 未选择时显示占位文案"请选择类型"
  - 选项来自预设诊断类型列表（如"孤独症谱系障碍""阿斯伯格综合征"等）
  - 校验错误时在下方显示错误文案

#### D5 — 主要行为类型（必填）
  - 原生单列选择器
  - 未选择时显示占位文案"请选择类型"
  - 选项来自预设行为类型列表（如"自伤行为""攻击行为""情绪崩溃"等）
  - 校验错误时在下方显示错误文案

#### D6 — 语言水平（选填，新增）
  - 原生单列选择器
  - 未选择时显示占位文案"请选择（选填）"
  - 选项来自预设语言水平列表（如"无口语""单词""短句""流利"等）

#### D7 — 用药备注（选填，新增）
  - 多行文本输入区
  - 占位提示："如：每日需服用…"
  - 与基础信息区分——属于补充信息

- **数据来源**: 编辑模式下从 `getProfile(profileId)` 的返回值填充；创建模式下全部为空

- **三态**:
  - Loading: 编辑模式下显示表单占位（同形区域）
  - Normal: 表单正常可编辑
  - Saving: 表单可见但不可编辑

- **data-testid**: `profile-edit-section-basic`, `profile-edit-avatar`, `profile-edit-field-nickname`, `profile-edit-field-birthdate`, `profile-edit-field-diagnosis`, `profile-edit-field-behavior`, `profile-edit-field-language`, `profile-edit-field-medication`

### 区块E — 标签配置区

- **Logic-ID**: `selectedTags`, `customTags`, `customTagInput`
- **职责**: 配置档案的感官特征标签和触发因素标签，为 RAG 检索提供过滤维度

- **内容**:

#### E1 — 感官特征标签
  - 预设标签列表（如"听觉敏感""视觉敏感""触觉防御""前庭寻求"等）
  - 多选——标签以可选/已选两种状态展示
  - 点击切换选中状态

#### E2 — 触发因素标签
  - 预设标签列表（如"噪音""人群""陌生环境""例行程序改变"等）
  - 多选——与感官特征相同的交互方式
  - 已选中的触发因素标签和感官特征标签混合存储在 `selectedTags` 中

#### E3 — 自定义标签
  - 文本输入框 + 确认添加按钮
  - 已添加的自定义标签以标签形式展示，每个有移除按钮
  - 输入限制：≤10 字
  - 不允许重复标签
  - 添加后清空输入框

- **数据来源**: 编辑模式下从 `ProfileResponse.sensory_features` 和 `triggers` 填充

- **交互**:
  - 点击预设标签 → 切换选中/取消
  - 输入自定义标签文本 → 点击添加 → 追加到 `customTags`
  - 点击自定义标签的移除按钮 → 删除该标签

- **三态**:
  - Loading: 标签占位区域
  - Normal: 标签可交互
  - Saving: 标签可见但不可交互

- **data-testid**: `profile-edit-tag-section`, `profile-edit-tag-sensory-{n}`, `profile-edit-tag-trigger-{n}`, `profile-edit-tag-custom-input`, `profile-edit-tag-custom-add`, `profile-edit-tag-custom-{n}`

### 区块F — 事件历史区（仅编辑模式）

- **Logic-ID**: `events`, `eventsLoading`, `eventsExpanded`
- **职责**: 展示当前档案的事件记录——让用户在编辑档案时了解已记录的行为事件

- **内容**:
  - 折叠标题栏："事件历史（N 条）" + 展开/收起箭头
  - 展开后：事件列表，每条显示
    - 事件时间（相对时间格式）
    - 行为类型 + 严重程度
    - 如有专业评估，显示"已评估"标记
    - 每条事件有删除入口

- **交互**:
  - 点击折叠标题栏 → 切换展开/收起
  - 点击单条事件的删除入口 → 删除该事件（直接删除，无二次确认）→ 本地移除该条目

- **条件渲染**: 仅在 `mode === 'edit'` 时展示

- **三态**:
  - Loading（事件加载中）: 折叠标题栏显示"加载中…"
  - Empty（无事件）: 展开后显示"暂无事件记录"
  - Normal: 事件列表正常展示

- **data-testid**: `profile-edit-event-section`, `profile-edit-event-section-toggle`, `profile-edit-event-item-{n}`, `profile-edit-event-delete-{n}`

### 区块G — 删除确认弹窗

- **Logic-ID**: `showDeleteModal`, `deleteConfirmName`, `nickname`
- **职责**: 确认用户删除档案的意图——不可逆操作需要明确确认

- **内容**:
  - 弹窗标题："删除档案"
  - 警告文字：告知此操作不可撤销，所有事件记录将被一并删除
  - 输入确认区域：提示"请输入档案昵称以确认删除"，文本输入框
  - 底部两个操作：
    - "取消" → 关闭弹窗
    - "确认删除" → 仅在输入框内容与档案昵称完全匹配时可用；不匹配时不可点击

- **交互**:
  - 点击遮罩 → 关闭弹窗（不删除）
  - 点击"取消" → 关闭弹窗
  - 输入昵称匹配 → 按钮可用
  - 点击"确认删除" → 调用 `deleteProfile(profileId)` → 成功后返回列表页

- **出现方式**: 弹窗出现在页面中央，有遮罩层。点击遮罩关闭
- **三态**:
  - Hidden: 弹窗不渲染
  - Showing: 用户输入中
  - Deleting: 按钮显示"删除中…"且不可操作

- **data-testid**: `profile-edit-delete-modal`, `profile-edit-delete-modal-overlay`, `profile-edit-delete-modal-input`, `profile-edit-delete-modal-cancel`, `profile-edit-delete-modal-confirm`

### 区块H — 未保存确认对话框

- **Logic-ID**: `isDirty`
- **职责**: 当用户修改了内容但未保存就尝试离开时，询问用户意图
- **触发条件**: `isDirty === true` 且用户点击"取消"或触发系统返回手势
- **内容**:
  - 标题："放弃修改？"
  - 说明文字："你有未保存的修改，确定要放弃吗？"
  - 两个操作：
    - "继续编辑" → 关闭对话框，留在编辑页
    - "放弃" → 返回列表页
- **三态**: Hidden / Showing
- **data-testid**: `profile-edit-discard-dialog`, `profile-edit-discard-dialog-stay`, `profile-edit-discard-dialog-discard`

## 表单校验规则

| 字段 | 规则 | 错误文案 |
|------|------|---------|
| `nickname` | 非空；≤10 字 | "请输入档案昵称" / "昵称不超过10个字" |
| `birthDate` | 非空；格式 YYYY-MM-DD | "请选择出生日期" |
| `diagnosisType` | 非空 | "请选择诊断类型" |
| `primaryBehavior` | 非空 | "请选择主要行为类型" |
| `languageLevel` | 选填，无校验 | — |
| `medicationNotes` | 选填，无校验 | — |

- 校验时机：点击保存按钮时
- 校验失败后：每个有错误的字段在输入区域下方显示对应错误文案
- 校验通过后：进入保存流程

## 交互行为汇总

| 行为 | 触发方式 | 结果 |
|------|---------|------|
| 修改字段 | 输入/选择器变更 | 更新表单状态；`isDirty` 重新计算 |
| 选择标签 | 点击预设标签 | 切换选中/取消 |
| 添加自定义标签 | 输入 + 点击添加 | 追加到 `customTags` |
| 移除自定义标签 | 点击标签移除按钮 | 从 `customTags` 中删除 |
| 保存 | 点击导航栏保存按钮 | 校验 → 提交 → 成功自动返回 |
| 取消 | 点击导航栏取消 | `isDirty ? 弹出确认 : 直接返回` |
| 打开菜单 | 点击"···" | 显示菜单面板 |
| 关闭菜单 | 点击遮罩/再次点击"···" | 隐藏菜单面板 |
| 删除档案 | 菜单 → "删除此档案" → 弹窗确认 | 输入昵称 → 确认删除 → 返回列表 |
| 关闭删除弹窗 | 点击遮罩/取消 | 弹窗关闭 |
| 展开事件历史 | 点击折叠标题栏 | 切换展开/收起 |
| 删除单条事件 | 点击事件删除入口 | 删除 + 本地移除条目 |
| 关闭保存错误 | 点击通知条关闭按钮 | 隐藏通知条 |
| 放弃修改确认 | 修改后点击取消 | 弹出对话框，选择继续/放弃 |

## 文件拆分

```
profiles/pages/
├── edit.tsx               ← 接线容器
│   → 管理 mode 判断、表单状态、校验、保存/删除流程
│   → 管理事件列表获取与删除
│   → 管理 isDirty 和未保存确认对话框
│   → 通过 Props 注入数据到子组件
├── edit.config.ts         ← 页面配置
└── components/
    ├── ProfileEditNavbar.tsx       ← 导航栏（区块A）
    │   → 接收 mode, canSave, isSaving, onCancel, onSave, onMenuOpen
    ├── ProfileEditBasicInfo.tsx    ← 基础信息区（区块D）
    │   → 接收所有表单字段 + errors + onChange 回调
    │   → 管理日期/单选选择器的交互
    ├── TagSection.tsx              ← 标签配置区（区块E）
    │   → 接收 selectedTags, customTags, customTagInput
    │   → 通过 onToggleTag / onAddCustomTag / onRemoveCustomTag 回调上报
    ├── ProfileEditEventSection.tsx ← 事件历史区（区块F）
    │   → 接收 events, isLoading, expanded, onToggle, onDeleteEvent
    ├── DeleteConfirmModal.tsx      ← 删除确认弹窗（区块G）
    │   → 接收 visible, nickname, confirmName, onConfirmNameChange, onCancel, onDelete
    └── DiscardChangesDialog.tsx    ← 未保存确认对话框（区块H）
        → 接收 visible, onStay, onDiscard
```

**原则**:
- `edit.tsx` 是唯一接线点 — 管理表单状态、校验、提交、删除流程
- 所有子组件为纯展示型 — 不 import 任何 Hook，数据通过 Props 接收
- `TagSection` 与档案列表页共享（但各自独立实例化）
