## 1.8 异常的业务策略

### 1.8.1 异常 1：CI 质量检查失败

- **业务触发条件**：后端 lint 未通过、测试用例失败、前端 TypeScript 类型检查发现错误
- **[确定] 业务处理策略**：
  1. CI 工作流标记为失败
  2. 不触发后续部署流水线
  3. 开发者需修正代码后重新推送
- **业务影响**：代码变更不进入部署流程，部署流水线被阻塞

### 1.8.2 异常 2：CI 测试环境服务不可用

- **业务触发条件**：CI runner 中的 PostgreSQL 或 Redis 服务容器启动失败或健康检查超时
- **[确定] 业务处理策略**：
  1. CI 工作流标记为失败（服务无法就绪）
  2. 重试机制由 GitHub Actions 内置（每个 Job 运行在全新的 runner 环境中，下次执行会重新启动服务容器）
- **业务影响**：CI 失败，需手动重新触发

### 1.8.3 异常 3：Docker 镜像构建失败

- **业务触发条件**：Dockerfile 语法错误、依赖安装失败、网络原因导致基础镜像拉取失败
- **[推断] [待确认] 业务处理策略**：
  1. Deploy 工作流标记为失败
  2. 开发者在 GitHub Actions 日志中查看构建错误详情
  3. 修正 Dockerfile 或依赖配置后重新触发部署
- **业务影响**：无法生成可部署的容器镜像，部署被阻塞

### 1.8.4 异常 4：部署失败（推测）

- **业务触发条件**：部署到目标环境失败（网络、权限、配置等原因）
