# 「篝火智答」标准案例库设计方案

## 一、设计概述

### 1.1 核心定位
标准案例库是 RAG 引擎的**主知识燃料**，为智能应急咨询提供**可检索、可溯源、可执行**的通用干预方案。所有案例均基于真实干预经验，经结构化处理后入库。

### 1.2 学术锚定
- **分类体系**：锚定 NCAEP（National Clearinghouse on Autism Evidence and Practice）2020 确认的 **28 种循证实践（EBP）**，确保干预方法的专业权威性。
- **生产流程**：借鉴 AFIRM（Autism Focused Intervention Resources and Modules）的**步骤化模块设计**，将干预经验转化为可执行的协议卡片。
- **检索架构**：参考医学 RAG 与 CBR（Case-Based Reasoning）研究的**结构化摘要（Structured Summaries）**范式，以结构化卡片作为检索单元，原始叙事作为参考底稿。

### 1.3 双层存储架构（决策：选项 C）

标准案例库采用**双层分离存储**：

| 层级 | 名称 | 作用 | 存储方式 | 面向对象 |
|---|---|---|---|---|
| **L1** | 原始叙事层（Raw Case Layer） | 保留干预的真实场景感与细节 | MySQL 文本存储 | 专家审核、家属延伸阅读 |
| **L2** | 结构化卡片层（Structured Protocol Layer） | 供 RAG 检索与四段式生成 | ChromaDB 向量切片 + MySQL 元数据 | AI 引擎、专家快速查阅 |

---

## 二、双层存储架构详解

### 2.1 L1 原始叙事层

**内容来源**：
- 专家基于多案例提炼的通用化叙事
- 机构合作脱敏后的真实干预记录
- 人工工单经专家改写后的沉淀记录

**内容形态**：
以自然语言撰写的完整干预故事，包含：
- 场景背景（时间、地点、触发因素）
- 患者基本特征（年龄区间、诊断类型、能力水平）
- 行为表现细节
- 干预过程描述
- 结果与反思

**示例**：
> 「7 岁 ASD 男孩小明在商场因吹风机声音突然捂住耳朵蹲下，拒绝移动。妈妈尝试拉他起身无效。老师到场后，先关闭附近吹风机电源，提供降噪耳机和挤压玩具，3 分钟后患者平复，随后带离至安静角落。」

### 2.2 L2 结构化卡片层

**生产流程**（决策：自然语言 → LLM 提取 → 专家微调）：
1. **原始文本录入**：专家以自然语言撰写 L1 叙事
2. **LLM 自动提取**：系统调用 LLM 从 L1 文本中抽取结构化字段，生成 L2 初稿
3. **专家微调确认**：专家在 LLM 提取结果基础上修正、补充，确认后提交审核

**设计意图**：
- 降低专家录入负担（无需一开始就填表格）
- 保证结构化精度（LLM 提取 + 人工校验，比纯 LLM 生成更可控）
- 保留真实叙事（L1 供人阅读，建立信任）

### 2.3 两层关系

- **一对多**：一条 L1 原始叙事可衍生出多条 L2 结构化卡片（如一个复杂案例包含「感官过载应对」和「拒绝移动引导」两个独立干预协议）
- **审核绑定**：L2 卡片审核通过后，其关联的 L1 原始叙事同步标记为「已发布」
- **版本联动**：L2 卡片更新迭代时，可选择是否同步更新关联的 L1 叙事

## 三、字段定义表

### 3.1 L1 原始叙事层字段

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `raw_id` | UUID | 是 | 原始叙事唯一标识 |
| `title` | String | 是 | 叙事标题（通用化描述） |
| `narrative` | Text | 是 | 完整自然语言叙事文本 |
| `source_type` | Enum | 是 | 来源：专家撰写 / 机构脱敏 / 工单沉淀 |
| `source_expert_id` | FK | 是 | 撰写或沉淀该案例的专家 |
| `derived_cards` | JSON | 否 | 关联的 L2 卡片 ID 列表 |
| `created_at` | DateTime | 是 | 创建时间 |
| `updated_at` | DateTime | 是 | 最后更新时间 |

### 3.2 L2 结构化卡片层字段（核心）

#### 基础信息

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `card_id` | UUID | 是 | 卡片唯一标识 |
| `raw_id` | FK | 是 | 关联的 L1 原始叙事 |
| `title` | String | 是 | 卡片标题（通用化干预方案名） |
| `scenario` | Text | 是 | 适用场景描述 |
| `behavior_type` | Enum | 是 | 行为类型：自伤 / 攻击 / 刻板 / 逃跑 / 情绪崩溃 / 其他 |
| `age_range` | JSON | 是 | 适用年龄区间，如 `["3", "6"]` 表示 3-6 岁 |
| `severity_level` | Enum | 是 | 适用严重程度：轻 / 中 / 重 |
| `setting` | Enum | 否 | 发生场景：家庭 / 学校 / 公共场合 / 机构 / 不限 |

#### 干预方法（NCAEP EBP 锚定）

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `ebp_tags` | JSON | 是 | NCAEP 28 种循证实践标签（专家端使用） |
| `parent_category` | Enum | 是 | 家属端展示大类（5-6 类映射，见 3.3 节） |

#### 四段式输出字段（强制对齐）

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `immediate_action` | Text | 是 | **即时安全干预动作**（对应四段式第 1 段） |
| `comforting_phrase` | Text | 是 | **情绪安抚话术**（对应四段式第 2 段） |
| `observation_metrics` | Text | 是 | **后续观察指标**（对应四段式第 3 段） |
| `medical_criteria` | Text | 是 | **就医判断标准**（对应四段式第 4 段） |

#### 循证与质量

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `evidence_level` | Enum | 是 | 循证等级（见 3.4 节） |
| `caution_notes` | Text | 是 | 禁忌、注意事项与常见误用 |
| `contraindications` | Text | 否 | 明确不适用的人群或场景 |
| `vector_id` | String | 是 | ChromaDB 向量切片 ID |
| `audit_status` | Enum | 是 | 审核状态：待审 / AI预审通过 / 专家终审通过 / 已驳回 |
| `reviewed_by` | FK | 否 | 终审专家 ID |
| `published_at` | DateTime | 否 | 发布时间（审核通过后） |
| `next_review_date` | DateTime | 否 | 下次复核日期 |
| `is_template` | Boolean | 是 | 是否为从多案例提炼的标准化模板 |

### 3.3 NCAEP EBP 标签与家属端大类映射（决策：A+B 结合）

**专家端使用**：NCAEP 2020 确认的 28 种循证实践标签，包括但不限于：
- 前因干预（ABI）
- 辅助替代沟通（AAC）
- 行为动力干预（Behavioral Momentum Intervention）
- 认知行为策略（Cognitive Behavioral/Instructional Strategies）
- 差别强化（Differential Reinforcement）
- 直接教学（Direct Instruction）
- 离散试教学（DTT）
- 消退（Extinction）
- 功能行为评估（FBA）
- 功能沟通训练（FCT）
- 自然主义干预（Naturalistic Intervention）
- 家属实施干预（Parent-Implemented Intervention）
- 同伴教学（Peer-Based Instruction）
- 提示（Prompting）
- 强化（Reinforcement）
- 反应中断/重定向（Response Interruption/Redirection）
- 自我管理（Self-Management）
- 社交叙事（Social Narratives）
- 社交技能训练（Social Skills Training）
- 任务分析（Task Analysis）
- 视频示范（Video Modeling）
- 视觉支持（Visual Supports）
- …（其余 6 项依 NCAEP 完整列表补充）

**家属端展示映射**（5-6 大类）：

| 家属端大类 | 涵盖的 NCAEP EBP 标签 | 典型场景 |
|---|---|---|
| **环境调整** | ABI、Visual Supports、感觉统合相关策略 | 减少触发因素、调整物理环境 |
| **沟通替代** | AAC、FCT、Social Narratives | 替代问题行为的功能性表达 |
| **行为塑造** | DTT、Direct Instruction、Task Analysis、Prompting、Reinforcement、Differential Reinforcement | 分步骤教授新技能 |
| **危机安全** | Response Interruption/Redirection、Extinction、FBA | 阻断危险行为、紧急降级 |
| **社交引导** | Peer-Based Instruction、Social Skills Training、Naturalistic Intervention | 同伴互动、社交场景适应 |
| **自我管理** | Self-Management、Cognitive Behavioral Strategies、Video Modeling | 情绪调节、自我监控 |

### 3.4 循证等级标注规范（决策：选项 A）

| 等级 | 标识 | 判定标准 | 案例示例 |
|---|---|---|---|
| **NCAEP 循证实践** | `EBP` | 干预方法属于 NCAEP 2020 确认的 28 种 EBP 之一，且有明确的文献支持 | 使用 DTT 进行任务分解教学 |
| **机构经验总结** | `INSTITUTIONAL` | 基于合作机构（如大连爱纳）多年实战沉淀，尚未达到 NCAEP 循证标准，但经专家共识验证 | 特定感官角布置方案 |
| **个案观察记录** | `OBSERVATIONAL` | 单次或少数个案的成功干预记录，尚未经过系统性验证，仅作参考 | 某患者对特定安抚音乐的独特反应 |

**前端展示**：
- 每条 RAG 输出标注来源案例时，同步显示循证等级徽章（如「循证」「经验」「观察」）
- 家属端hover提示说明等级含义

## 四、案例生产流程（自然语言 → LLM 提取 → 专家微调）

```
专家/机构上传原始叙事（L1）
    ↓
[步骤 1] LLM 自动提取结构化字段
  - 输入：L1 自然语言文本 + 四段式模板约束
  - 输出：L2 初稿（JSON 格式，含 immediate_action / comforting_phrase / observation_metrics / medical_criteria）
  - 模型建议：轻量级模型（如 DeepSeek-V4-Flash）即可，不要求强推理
    ↓
[步骤 2] 专家微调确认
  - 专家在 Web 界面查看 LLM 提取结果
  - 可修改、补充、删除字段
  - 关键校验：四段式字段是否完整、ebp_tags 是否准确、evidence_level 是否合理
    ↓
[步骤 3] 提交审核
  - L1 叙事 + L2 卡片一并进入审核队列
```

---

## 五、审核流程

### 5.1 AI 辅助预审（1-2 个工作日）

| 检查项 | 规则 |
|---|---|
| 格式完整性 | 四段式字段是否全部非空 |
| PII 脱敏 | L1 叙事中是否含真实姓名、地址、学校名 |
| 必填字段 | behavior_type / age_range / severity_level / ebp_tags 是否存在 |
| 循证等级一致性 | ebp_tags 与 evidence_level 是否匹配（如标注 EBP 但 ebp_tags 不在 28 种列表中则报错） |

### 5.2 专家终审（1-2 个工作日）

| 审核项 | 标准 |
|---|---|
| 专业性 | 干预动作是否符合孤独症干预规范，有无危险操作 |
| 四段式质量 | 即时动作是否可执行、安抚话术是否恰当、就医标准是否准确 |
| 循证准确性 | evidence_level 标注是否合理，是否夸大 |
| 通用化程度 | L1 叙事是否已脱敏，L2 卡片是否已剥离个体标识 |

**终审操作**：
- 通过：L2 卡片切片入 ChromaDB，L1 叙事标记发布
- 驳回：附修改意见，退回专家微调
- 标记为模板：若从多案例提炼，勾选 `is_template = true`

## 六、有效期与淘汰机制（决策：A+B 结合）

### 6.1 时间字段

| 字段 | 作用 |
|---|---|
| `published_at` | 首次发布时间 |
| `next_review_date` | 下次计划复核日期（默认发布后 2 年） |
| `last_reviewed_at` | 最后复核时间 |

### 6.2 复核触发条件

- **到期复核**：`next_review_date` 到达时，系统自动通知撰写专家复核
- **争议复核**：专家标记「过时/有误/争议」时，立即触发复核
- **知识更新复核**：NCAEP 发布新版 EBP 列表时，系统自动扫描关联案例

### 6.3 动态权重衰减

- RAG 检索时，案例的向量相似度得分乘以**时效权重系数**：
  - 0-1 年：权重 1.0
  - 1-2 年：权重 0.9
  - 2-3 年：权重 0.7
  - 3 年以上：权重 0.5
- 权重仅影响排序，不直接过滤（避免误杀仍有价值的经典方案）

### 6.4 淘汰操作

| 操作 | 效果 |
|---|---|
| 专家标记「过时」 | 案例从 RAG 检索中排除，但保留记录供追溯 |
| 管理员强制下架 | 案例完全不可见，需紧急修复或法律合规时触发 |
| 版本迭代 | 发布新版案例，旧版标记为「已更新」，RAG 默认使用最新版 |

## 七、与个性化案例库的检索交互协议（决策：独立双库）

### 7.1 架构原则

- **标准案例库**：ChromaDB 向量索引，供语义检索，全员共享
- **个性化案例库**：MySQL 结构化存储，按 `profile_id` 隔离，仅关联用户可见，**不入向量库**

### 7.2 检索流程

```
家属发起应急咨询
    ↓
[步骤 1] 读取个性化档案
  - 从 MySQL 读取该患者最近 3-5 条历史干预事件
  - 筛选与当前查询行为类型相同的事件
  - 格式化为上下文文本，直接注入 Prompt（第一轮注入）
    ↓
[步骤 2] 标准案例库检索
  - 将家属输入 + 档案标签（年龄、诊断类型、行为类型）组装为查询
  - 在 ChromaDB 中检索 Top-K 标准案例（K=3-5）
  - 结果注入 Prompt（第二轮注入）
    ↓
[步骤 3] LLM 生成四段式输出
  - 结合个性化历史 + 通用方案，生成专属建议
```

### 7.3 隔离规则

| 规则 | 说明 |
|---|---|
| 个性化案例绝对不入向量库 | 避免其他患者检索时交叉污染 |
| 个性化案例仅作为结构化标签过滤 | 如档案显示「听觉敏感」，则在标准案例检索时优先返回含「降噪策略」的卡片 |
| 历史事件注入数量上限 | 最多 3-5 条，避免挤占 Prompt 上下文窗口 |

## 八、专家一键沉淀流程（从工单到标准库）

```
工单解决后，专家点击「沉淀为案例」
    ↓
[步骤 1] 系统自动脱敏
  - 剥离：患者姓名、家庭住址、学校、具体日期
  - 保留：年龄区间、行为类型、干预动作、结果
    ↓
[步骤 2] 专家通用化改写
  - 将个体描述改写为通用场景
  - 例：「小明在万达」→「ASD 人士在大型商场」
    ↓
[步骤 3] LLM 提取结构化卡片
  - 从改写后的叙事中提取四段式字段
    ↓
[步骤 4] 专家微调确认
  - 校验四段式完整性、ebp_tags 准确性、循证等级
    ↓
[步骤 5] 提交审核
  - 进入标准案例库审核队列（AI 预审 + 专家终审）
    ↓
[步骤 6] 入库
  - L2 卡片切片入 ChromaDB
  - L1 叙事发布供查阅
```

**关键约束**：
- 沉淀是**复制+改写**，个性化档案中的原始记录**永不删除**
- 专家可选择「沉淀为通用模板」或「沉淀为单条案例」
- 工单沉淀的默认循证等级为 `INSTITUTIONAL`（机构经验），除非该干预方法本身属于 NCAEP EBP

## 九、边界与约束

- **不做多媒体**：MVP 阶段仅限文本，图片/视频纳入二期规划
- **不做激励机制**：专家/老师的案例上传激励由平台运营或合作机构另行管理
- **严格脱敏**：所有 L1 叙事上传时必须完成 PII 脱敏，机构合作案例需签署脱敏使用协议
- **免责声明**：所有案例与模板均需附带「AI 建议仅供参考，不构成医疗诊断或治疗建议」声明
- **权限隔离**：L1 原始叙事向认证用户开放查阅，L2 结构化卡片供 AI 检索与专家快速查阅
