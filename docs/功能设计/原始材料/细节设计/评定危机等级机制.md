# 「篝火智答」危机分级与兜底策略设计方案

## 一、设计原则

1. **安全底线不可突破**：涉及身体伤害、走失、用药等场景，必须人工介入
2. **宁过度拦截，不漏风险**：规则引擎宁可误判为重度，也不允许真实重度漏判为轻度
3. **家属参与关键判定**：将「是否涉及身体安全」的初始判断交给家属，降低模型黑盒风险
4. **并行不阻塞**：LLM 复审与 RAG 生成并行执行，不增加首字延迟

---

## 二、整体架构：三层递进判定

```
家属输入行为描述
    ↓
[第零层] 前置行为类型选择（家属主动勾选）
    ↓
[第一层] 规则引擎保底（硬拦截，0ms）
    ├── 命中重度关键词/类型 → 直接重度，阻断AI深度回答
    └── 未命中 → 继续
        ↓
[第二层] RAG 应急建议生成（正常输出）
        ↓
[第三层] LLM 精调复审（并行执行，3s内返回）
    ├── 复审升级 → 平滑切换应急模式
    └── 复审维持 → 提示条消失，保持原输出
```

---

## 三、第零层：前置行为类型选择

**触发时机**：家属输入文字描述后、系统生成建议前。

**交互设计**：

> 您描述的情况更接近以下哪类？（可多选）
> 
> □ 情绪哭闹/尖叫  
> □ 刻板行为升级（重复动作/语言）  
> □ **自伤行为**（撞头/咬手/抓挠等）  
> □ **攻击他人**（推搡/踢打/咬人等）  
> □ **逃跑/走失**（冲向危险区域/找不到人）  
> □ **用药相关**（药物使用/剂量/副作用）  
> □ 其他日常行为问题

**判定规则**：
- 若勾选 **自伤/攻击/逃跑/用药** 中任意一项 → **跳过全部自动分级，直接判定为重度**
- 若未勾选上述四项 → 进入第一层规则引擎

**设计意图**：
- 将「是否涉及身体安全」这一最关键的判断，从模型手里交还给家属
- 降低规则引擎和 LLM 的双重漏判风险
- 交互极简，危机时刻不增加认知负担（默认单选，多选仅用于复杂场景）

---

## 四、第一层：规则引擎保底（硬拦截）

**执行时机**：前置选择完成后立即执行，0 延迟。

### 4.1 重度硬拦截规则

| 触发来源 | 判定条件 | 动作 |
|---|---|---|
| **前置选择** | 家属勾选自伤/攻击/逃跑/用药 | 直接重度 |
| **关键词匹配** | 输入文本含高危词（撞墙、撞头、咬手、抓破、打别人、跑出去、吃药、吞药、割腕等） | 直接重度 |
| **否定词过滤** | 输入含「没有/不会/以前」+ 高危词 → 排除触发 | 正常进入 LLM 复审 |
| **档案叠加** | 档案中该患者历史有自伤/攻击记录，且输入含「又/再次/还是」 | 至少中度，建议人工复核 |

### 4.2 规则引擎输出

| 输出项 | 说明 |
|---|---|
| `rule_level` | `severe` / `moderate` / `mild` |
| `rule_trigger` | 触发规则 ID（如 `KW_SELF_HARM_001`、`PRE_CHOICE_ATTACK`） |
| `rule_confidence` | `1.0`（规则命中为确定性判定） |

### 4.3 规则命中重度时的系统行为

- **RAG 生成**：仅输出「初步安全提示」（如「请立即移除周围危险物品，确保患者安全」）
- **页面状态**：立即标红，锁定为应急模式
- **人工通道**：自动生成特急工单，群发专家即时通知
- **LLM 复审**：**不执行**，仅记录规则触发日志

---

## 五、第二层：RAG 应急建议生成

**执行时机**：规则引擎未命中重度时，立即启动。

- 检索同年龄段、同行为类型、同情绪等级的真实案例
- 注入个性化档案数据
- 生成四段式结构化输出

**前端表现**：
- 页面顶部显示轻量提示条：「系统正在复核安全风险…」
- 四段式建议正常展示，供家属即时阅读

---

## 六、第三层：LLM 精调复审（并行执行）

**执行时机**：与 RAG 生成并行启动，不阻塞主流程。

### 6.1 输入上下文

```json
{
  "child_profile": {
    "age": "6岁",
    "diagnosis": "ASD",
    "history_tags": ["刻板行为", "听觉敏感"],
    "last_incident": "3天前因噪音崩溃"
  },
  "parent_input": "患者在商场突然捂住耳朵蹲在地上，怎么拉都不走",
  "pre_choice": ["情绪哭闹/尖叫", "刻板行为升级"],
  "rule_engine_result": {
    "level": "mild",
    "trigger": null
  },
  "rag_output_summary": "感官过载应对建议"
}
```

### 6.2 LLM 输出要求（强制 JSON）

```json
{
  "review_level": "moderate",
  "reason": "公共场合感官过载存在走失风险，且家属描述'怎么拉都不走'表明行为难以阻断",
  "confidence": 0.82,
  "risk_factors": ["公共场合", "拒绝移动", "感官过载史"],
  "suggested_action": "AI建议+专家入口"
}
```

### 6.3 LLM 边界约束

| 约束 | 说明 |
|---|---|
| **无权降级规则重度** | 若规则已判重度，LLM 仅记录，不得输出 `mild`/`moderate` |
| **宁升勿降** | 置信度 < 0.7 时，默认升级至更高一级 |
| **超时兜底** | 3 秒内未返回，默认采纳规则初审结果，LLM 结果仅记录日志 |

## 七、最终判定矩阵

| 规则初审 | LLM 复审 | 最终等级 | 说明 |
|---|---|---|---|
| 重度 | — | **重度** | 规则硬拦截，LLM 不执行 |
| 中度 | 重度 | **重度** | LLM 升级覆盖 |
| 中度 | 中度 | **中度** | 一致 |
| 中度 | 轻度 | **中度** | 规则保底，不采纳降级 |
| 轻度 | 重度 | **重度** | LLM 补漏 |
| 轻度 | 中度 | **中度** | LLM 升级覆盖 |
| 轻度 | 轻度 | **轻度** | 一致 |

**核心原则**：
- **任何一方判重度，最终就是重度**
- **规则可以向上覆盖 LLM，LLM 不能向下覆盖规则**
- **冲突时以更高等级为准**

---

## 八、前端状态流转

| 阶段 | 页面状态 | 家属可见内容 |
|---|---|---|
| **输入完成** | 等待前置选择 | 行为类型选择弹窗 |
| **前置选择后** | 规则初审中 | 若勾选高危类型 → 立即进入红色应急模式 |
| **规则未命中重度** | 建议生成中 + 复核中 | 四段式建议展示 + 顶部「复核中」提示条 |
| **LLM 复审维持** | 复核完成 | 提示条消失，保持原建议 |
| **LLM 复审升级** | 应急模式切换 | 页面平滑覆盖为红色应急界面，提示「系统复核发现安全风险，建议立即联系专家」 |

---

## 九、与工单优先级的衔接

| 最终等级 | 人工通道策略 | 工单优先级 | 响应时效 |
|---|---|---|---|
| **重度** | 强制转人工，AI 仅输出安全提示 | 特急 | 即时通知所有在线专家，先抢先得 |
| **中度** | AI 建议 + 浮动专家入口，家属可一键转人工 | 紧急 | 30 分钟内响应 |
| **轻度** | AI 直接输出四段式建议，底部提示「持续未缓解可联系专家」 | 普通（家属手动发起） | 按排队顺序 |

---

## 十、持续优化机制

| 优化项 | 执行方式 |
|---|---|
| **规则词库迭代** | 每周审查人工工单日志，将「规则未触发但专家判定为重度」的案例反向补充进关键词库 |
| **LLM Prompt 迭代** | 每月分析 LLM 复审与专家最终判定的一致性，调整复审 Prompt 中的风险因素权重 |
| **家属反馈闭环** | 若家属在前置选择中勾选「其他」但后续被 LLM/专家升级为重度，触发词库补充流程 |

## 十一、免责声明（所有等级统一附加）

> **AI 建议仅供参考，不构成医疗诊断或治疗建议。涉及身体伤害、走失风险或用药决策时，请立即联系专业医疗机构。**
