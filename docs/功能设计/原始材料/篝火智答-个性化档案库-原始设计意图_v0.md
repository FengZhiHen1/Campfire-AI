## AI 提炼摘要

### 功能定位

「篝火智答」核心 P0 模块。为智能应急咨询提供个性化检索透镜，通过「家长 + 老师/专家」共建的行为-情绪档案，提升 RAG 案例匹配的精准度与建议的个性化程度。家长拥有档案的绝对控制权。

### 目标用户

- **核心操作者**：孤独症儿童家长（创建档案、记录日常行为/情绪事件、维护基础标签）
- **协同共建者**：特教/康复老师、专家（补充专业评估与干预记录，不可修改家长内容）
- **间接受益者**：使用应急咨询模块的家长（获得基于档案的个性化建议）

### 核心场景

1. **快速冷启动**：家长注册后填写少量核心标签（年龄、诊断类型、主要行为类型），即可在应急咨询中获得基础个性化建议，其余信息在使用中逐步完善。
2. **日常记录沉淀**：家长持续记录孩子的行为/情绪事件，系统自动沉淀为档案数据。
3. **专业共建**：家长主动邀请关联的老师/专家，后者补充专业评估与干预记录。
4. **档案驱动咨询**：应急咨询时，系统自动关联当前孩子的档案，作为 RAG 检索过滤条件注入。
5. **权限自主管理**：家长可随时解除某位老师的关联，回收其档案访问权限。

### 关键功能点

1. **多孩子档案管理**：支持一个家庭账号下创建多个孩子的独立档案，应急咨询时可自动关联或手动切换。
2. **档案共建**：
   - 家长：记录日常行为/情绪事件，维护基础标签，拥有绝对控制权。
   - 老师/专家：仅可补充专业评估与干预记录，不可修改家长创建的内容。
3. **标签体系**：平台提供标准化预设标签（年龄、诊断类型、语言水平、刻板行为类型、情绪触发因素、既往干预反应等），同时允许家长在限定范围内添加自定义标签。
4. **档案驱动 RAG**：档案数据作为检索过滤条件，提升案例匹配精准度。
5. **关联管理**：家长主动邀请老师/专家建立档案关联，可随时单方面解除。
6. **隐私控制**：档案仅家属本人及关联老师/专家可见，严格脱敏；无关联者不可查看。
7. **冷启动引导**：采用「简化必填标签 + 后续逐步完善」策略，降低新用户门槛。

### 边界与约束

- **权限边界**：老师/专家不可修改家长创建的任何内容。
- **访问边界**：档案访问严格基于关联关系，无关联者不可查看。
- **控制边界**：家长拥有解除关联的绝对权力，无需审核。
- **标签边界**：标签自定义在平台限定范围内，非完全开放。
- **隐私边界**：所有档案数据严格脱敏，手机号等敏感信息仅管理员可见。

### 依赖项

- 智能应急咨询模块（档案的核心消费方，档案数据为其提供个性化检索过滤条件）
- 真实案例库（档案与案例库共同服务于 RAG 引擎）
- 用户账号与权限体系（支撑多孩子管理、关联关系、RBAC）
