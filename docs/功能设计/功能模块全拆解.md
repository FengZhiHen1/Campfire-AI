## 附录 C：边界决策记录

1. **CSLT-02（RAG语义检索）与 CSLT-03（应急方案生成）**：将 RAG 检索与 LLM 生成拆分为两个独立模块。虽然两者在流程中紧密衔接（检索结果直接注入 Prompt），但检索策略（向量索引、混合查询、标签过滤）与生成策略（Prompt 模板、四段式约束、温度控制）由不同的技术栈和不同的专家知识支撑，可分配给不同工程师独立交付与调优。

2. **CSLT-01（危机分级判定）与 SEC-02（AI输出安全护栏）**：危机分级判定聚焦于输入端的风险等级评估（前置选择 + 关键词匹配 + LLM 复审），而 AI 输出安全护栏聚焦于输出端的内容安全（免责声明注入 + 幻觉检测 + 关键词阻断）。两者在安全链路上串联但处理的数据流向相反，归属不同责任域。

3. **CASE-03（案例审核工作流）与 CSLT-05（置信度后校验）**：案例审核工作流管理的是案例从提交到发布的完整审核生命周期，置信度后校验管理的是应急方案生成后的质量评估与工单触发。两者虽有审核理念相似但面向不同实体且无共享代码，保持独立。

4. **TICK-01（工单自动生成）与 CSLT-05（置信度后校验）**：置信度后校验是工单自动生成的触发来源之一（四种触发来源中的"置信度不足"分支），但工单自动生成还支持另外三种触发来源且包含上下文继承逻辑，因此与置信度后校验保持独立。CSLT-05 调用 TICK-01 的接口但不包含工单创建逻辑。

5. **PROF-02（档案驱动检索过滤）与 CSLT-02（RAG语义检索）**：档案驱动检索过滤负责从档案中提取标签并构建过滤条件，RAG 语义检索负责接收过滤条件并执行向量检索。两者各司其职且通过标准化接口（标签过滤条件 Dict）解耦，使档案标签体系变更不影响检索引擎实现。

6. **CASE-04（案例向量化入库）与 CASE-05（案例版本迭代）**：切片与索引是案例审核通过后的一次性数据流水线（文本拆分→向量化→写入索引），版本迭代管理的是案例内容的持续更新与历史回退。两者触发时机不同（索引在审核通过后触发，版本管理在已发布案例更新时触发）且涉及不同的技术组件，保持独立。

7. **KNOW-05（应急场景引导）与 CSLT-01（危机分级判定）**：应急场景引导在科普问答场景中检测紧急关键词并引导跳转，危机分级判定在应急咨询场景中执行三层递进判定。两者检测的关键词库共享但判定结果的处理方式不同（引导跳转 vs 阻断+分级），且运行于不同的业务模块上下文中。
