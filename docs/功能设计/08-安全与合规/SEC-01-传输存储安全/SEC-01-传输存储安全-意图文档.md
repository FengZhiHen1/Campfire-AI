### 1.1 项目定位

传输存储安全模块是篝火智答平台的安全防护基座。它横跨全平台所有服务端通信链路和数据落地路径，确保用户敏感信息在传输中不被窃听、在存储中不可还原、在访问中不被越权获取。

本模块处于整个业务流程的入口层与数据入口层之间：所有客户端请求必须经本模块的安全策略校验后才可进入业务处理流程，所有持久化数据在落地前必须经本模块的加密或脱敏保护。它不是用户直接感知的功能，而是平台所有功能模块得以安全运行的基础保障。

### 1.4 业务目标

1. 所有客户端与服务器之间的通信必须经过加密保护，确保数据在传输过程中不被第三方窃听或篡改
2. 用户密码在存储层必须不可逆——即使数据库被拖库，也无法还原用户原始密码
3. JWT 凭证（访问令牌与刷新令牌）的签发与校验必须使用足够强度的密钥，过期后自动失效
4. 敏感数据（手机号等个人身份信息）在存储查询和 API 响应中默认脱敏，仅特定角色可查看明文
5. API 层面的请求频率必须在业务可承受范围内被限制，防止恶意滥用影响正常用户使用
6. 用户上传的文件必须经过类型安全校验，防止恶意文件进入存储系统

### 1.5 用户旅程 / 业务流程

本模块为基础设施级安全防护，不面向最终用户。其业务触发点分布在以下关键流程中：

1. **用户登录**：用户提交账号和密码 → 密码经 bcrypt 哈希校验（不可逆比较）→ 校验通过后签发 JWT 访问令牌和刷新令牌
2. **数据请求**：用户发起任意 API 请求 → 请求先经过 HTTPS 加密传输 → 到达服务器后先通过限流检查 → 通过后解析 JWT Token 验证身份和权限 → 通过后进入业务处理
3. **敏感数据查看**：管理员查看用户手机号 → 系统校验其 RBAC 角色为管理员 → 返回明文手机号；非管理员角色查询同一用户时 → 返回脱敏后的手机号（如 "138****1234"）
4. **文件上传**：用户上传案例附件 → 系统校验文件类型是否在允许白名单内 → 校验通过后存储到安全存储区 → 返回安全下载链接（带时效限制的签名 URL）

### 1.6 输入/输出业务定义

#### 1.6.1 输入

| 字段名 | 业务含义 | 是否必填 | 业务约束 | 示例值 |
|--------|---------|---------|---------|--------|
| 用户密码 | 用户登录时输入的密码原文 | 是（登录场景） | 长度 8-64 个字符；必须至少包含字母和数字 | "MySecurePass123" |
| JWT 访问令牌 | 客户端携带的身份凭证 | 是（除登录/注册外的所有请求） | 由服务端签发，有效期 15 分钟；过期后须用刷新令牌换新 | "eyJhbGci...（JWT 字符串）" |
| JWT 刷新令牌 | 用于换取新访问令牌的长期凭证 | 是（刷新场景） | 由服务端签发，有效期 7 天；每个用户仅保留最新一个有效刷新令牌 | "eyJhbGci...（JWT 字符串）" |
| API 请求 | 客户端发往服务器的业务请求 | 是 | 必须通过 HTTPS 传输；请求频率受用户级和 IP 级双重限流约束 | "GET /api/v1/consult/history" |
| 上传文件 | 用户通过客户端上传的附件 | 否 | 文件类型必须在允许白名单内；文件大小不超过单次上传上限 | "case_attachment_001.pdf" |

#### 1.6.2 输出

| 字段名 | 业务含义 | 业务约束 | 示例值 |
|--------|---------|---------|--------|
| 密码哈希 | 用户密码的不可逆保密存储值 | 使用 bcrypt 算法，salt 加固轮次不低于 12；同一密码每次哈希结果不同（随机 salt） | "$2b$12$...（bcrypt 哈希字符串）" |
| 访问令牌 | 客户端身份凭证，用于后续请求鉴权 | 有效期 15 分钟，过期自动失效；签发后不可撤销（通过短有效期控制风险） | "eyJhbGci...（JWT 字符串）" |
| 刷新令牌 | 用于获取新访问令牌的长期凭证 | 有效期 7 天；每个用户仅保留最新一个 | "eyJhbGci...（JWT 字符串）" |
| 安全下载链接 | 文件的临时下载地址 | 含时效签名，过期后自动失效；仅允许在有效期内访问 | "https://storage.example.com/file?signature=...&expires=..." |
| 脱敏手机号 | 非授权角色查询时返回的手机号 | 保留前三位和后四位，中间四位替换为星号；管理员角色查询时返回完整号码 | "138****1234" / "13800001234" |
| 限流拒绝响应 | 超出频率限制时的业务提示 | 明确告知用户"请求过于频繁"及恢复时间；不透露限流阈值内部细节 | "请求过于频繁，请在 60 秒后重试" |

### 1.8 异常的业务策略

#### 1.8.1 异常 1：传输链路不安全

- **业务触发条件**：客户端尝试通过非加密连接（HTTP）访问服务端 API
- **业务处理策略**：
  1. 服务端拒绝处理该请求，不返回业务数据
  2. 若为 HTTP 明文请求，强制重定向至 HTTPS
  3. 记录本次拒绝事件到审计日志
- **业务影响**：请求被拒绝，不产生任何业务结果；用户数据在传输过程中未被泄露

#### 1.8.2 异常 2：身份凭证无效或过期

- **业务触发条件**：客户端携带的 JWT Token 签名校验失败、已过期或已被轮换替代
- **业务处理策略**：
  1. 若访问令牌过期但刷新令牌有效，返回"令牌已过期，请刷新"的提示，引导客户端自动刷新
  2. 若签名校验失败或令牌被篡改，返回"身份验证失败"，要求用户重新登录
  3. 记录凭证异常事件（含用户名和时间）到审计日志，用于后续安全分析
- **业务影响**：用户的本次请求被拒绝；若刷新令牌仍有效，客户端可自动换新访问令牌后重试原请求，用户可能无感知

#### 1.8.3 异常 3：请求频率超出限制

- **业务触发条件**：同一用户在 1 分钟内的 API 调用次数超过 30 次，或同一 IP 地址在 1 分钟内的调用次数超过 100 次
- **业务处理策略**：
  1. 告知用户"请求过于频繁，请在 60 秒后重试"
  2. 不永久封禁用户，限流窗口滑动重置后自动恢复
  3. 连续多次触发限流（5 分钟内累计 3 次）时，在审计日志中标记为潜在异常行为
- **业务影响**：用户需等待恢复后方可正常使用；不影响其他用户的正常使用

#### 1.8.4 异常 4：文件类型不在白名单内

- **业务触发条件**：用户上传的文件扩展名或实际文件类型不在平台允许列表中
- **业务处理策略**：
  1. 禁止该文件进入存储系统
  2. 明确告知用户哪些文件类型被允许（如 PDF、JPG、PNG、DOCX）
  3. 记录该拒绝事件（含用户名、文件名和声称类型）到审计日志
- **业务影响**：本次上传被拒绝，用户需更换文件后重试；已有文件和数据不受影响

### 1.9 验收标准

| 编号 | 验收条件 | 验收方式 | 通过标准 |
|------|---------|---------|---------|
| AC-01 | 传输加密 | 安全扫描 | 所有外部 API 端点仅接受 HTTPS 请求；HTTP 明文请求被强制重定向或直接拒绝 |
| AC-02 | 密码不可逆存储 | 数据库审计 | 数据库用户表中不存在密码明文；所有密码存储值均为 bcrypt 哈希，且每个用户使用独立随机 salt |
| AC-03 | JWT 签名强度 | 配置审查 | JWT 签名密钥长度不低于 256 位；访问令牌有效期为 15 分钟，刷新令牌有效期为 7 天 |
| AC-04 | JWT 过期处理 | 集成测试 | 过期的访问令牌被拒绝并返回"令牌已过期"提示；有效刷新令牌可成功换取新的访问令牌 |
| AC-05 | 敏感数据脱敏 | 接口测试 | 非管理员角色查询用户数据时，手机号字段返回脱敏格式（保留前三位后四位）；管理员角色返回完整号码 |
| AC-06 | 请求限流生效 | 压力测试 | 同一用户在 1 分钟内第 31 次请求被拒绝并返回限流提示；限流窗口到期后第 31 次请求被正常处理 |
| AC-07 | 文件上传白名单 | 测试用例 | 上传白名单外的文件类型（如 .exe、.sh）被拒绝并返回明确错误信息 |
| AC-08 | 安全事件可追溯 | 日志审计 | 所有凭证异常、限流触发和文件上传拒绝事件均记录审计日志，包含事件时间、触发用户和事件类型 |

### 1.11 业务约束清单

1. **传输加密强制**：全平台所有面向客户端的 API 端点必须使用 HTTPS 加密传输，TLS 版本不得低于 1.3，禁止任何面向客户端的 HTTP 明文接口
2. **密码不可逆存储**：用户密码在任何存储介质中（数据库、备份、日志）均不得以明文或可逆加密形式存在；密码哈希的 salt 加固轮次不得低于 12
3. **JWT 密钥安全**：JWT 签名密钥长度不得低于 256 位；生产环境密钥必须通过密钥管理服务或环境变量注入，禁止硬编码在代码或配置文件中
4. **敏感数据最小可见**：手机号等个人身份信息默认对非管理员角色脱敏展示；审计日志中记录的敏感数据必须脱敏处理
5. **请求频率公平使用**：每个用户在 1 分钟内最多调用 30 次 API，每个未登录 IP 在 1 分钟内最多调用 100 次；限流策略不影响不同用户之间的独立性
6. **文件上传类型控制**：仅允许上传业务所需的文件类型（文档类与图片类）；禁止可执行文件、脚本文件和服务端配置文件进入存储系统
7. **安全事件可审计**：所有凭证异常、限流触发和文件安全拒绝事件必须记录审计日志，日志保留期不少于 180 天
8. **设计边界**：本模块不负责业务内容的 PII 检测（该职责归属 SEC-03 PII检测脱敏模块）；本模块不负责具体的 RBAC 权限规则定义（权限层级归属用户管理模块与各业务模块自身的权限声明）；本模块不负责限流策略的具体实现编码（该职责归属 SEC-04 防刷限流模块）

### 1.12 留给规范阶段的技术决策

| 序号 | 技术决策项 | 意图文档中的约束 | 留给规范阶段的理由 |
|------|-----------|-----------------|-------------------|
| 1 | JWT 密钥轮换策略 | 密钥长度 ≥ 256 位，须定期轮换 | 具体轮换周期、轮换期间新旧密钥共存策略和自动化轮换方案由 spec-writer 根据部署运维能力确定 |
| 2 | HTTPS 证书管理 | 所有外网端点必须 HTTPS，TLS ≥ 1.3 | 证书的签发方式（Let's Encrypt / 购买 / 云厂商托管）、自动续期流程和证书有效期由 spec-writer 确定 |
| 3 | bcrypt salt 精确轮次 | salt rounds ≥ 12 | 是否在 12 之上进一步提高（考虑服务器性能与安全性的权衡）由 spec-writer 根据性能基准测试结果确定 |
| 4 | 限流阈值配置化颗粒度 | 用户级 30 req/min，IP 级 100 req/min | 阈值是硬编码还是通过配置中心/环境变量动态调整；限流恢复窗口（当前为 60 秒滑动窗口）的参数可调性由 spec-writer 确定 |
| 5 | Redis 故障时限流降级策略 | 限流触发时返回"请求过于频繁"提示 | Redis 不可用时的降级行为（放行所有请求还是拒绝所有请求），由 spec-writer 根据业务可用性要求权衡 |
| 6 | 文件上传白名单具体条目 | 仅允许业务所需的文件类型 | 白名单的具体文件类型列表（PDF、JPG、PNG、DOCX 等）和各类型的最大允许大小由 spec-writer 根据各业务模块的需求确定 |
| 7 | 敏感数据脱敏的具体算法 | 手机号保留前三位后四位 | 是否扩展脱敏范围到其他字段（如身份证号、家庭住址）；脱敏占位符格式由 spec-writer 确定 |
| 8 | MinIO 预签名 URL 过期时间 | 下载链接含时效签名，过期后自动失效 | 具体有效期时长（秒/分钟/小时级）由 spec-writer 根据文件安全需求和使用体验确定 |
| 9 | 审计日志存储与格式 | 安全事件必须记录审计日志，保留 ≥ 180 天 | 日志的具体存储方式（数据库表/文件/ELK）、结构化字段定义和日志轮转策略由 spec-writer 确定 |
