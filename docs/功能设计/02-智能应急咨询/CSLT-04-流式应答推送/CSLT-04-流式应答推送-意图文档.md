### 1.1 项目定位

流式应答推送是应急咨询流程中的实时传输通道。它负责将应急方案生成模块产出的每一段文本增量，以逐字逐句的方式实时传送到家属的屏幕上，让家属在等待完整方案的过程中就看到内容正在逐句成形，而非面对空白页面焦虑等待。在整体应急咨询链路中，它是连接「方案生成」与「前端呈现」的数据管道——接收生成模块的内容片段，封装为统一的事件流格式，推送至前端逐句渲染。

### 1.4 业务目标

1. 家属提交行为描述后，系统在 3 秒内开始逐句展示应急方案内容（首字可见延迟 ≤ 3 秒），让家属即时感知到响应
2. 整个方案的完整文字在 10 秒内全部送达家属屏幕（全流程生成 ≤ 10 秒，包含上游生成时间）
3. 传输过程中网络的临时中断不影响家属获取完整方案：中断后系统自动重新连接，并从中断位置继续推送，家属无需从零等待
4. 推送的每段文本内容与上游生成模块的产出完全一致，传输层不增删改任何文字

### 1.5 用户旅程 / 业务流程

1. **家属提交行为描述**：家属在应急咨询界面输入儿童的行为描述文字，点击发送。此阶段本模块处于待命状态，等待上游生成模块产出内容

2. **方案生成开始，逐句推送启动**：上游 CSLT-03 开始产出第一个文本片段后，本模块立即将该片段封装为事件并发送至前端。家属在屏幕上看到第一句文字在 3 秒内出现，内容随时间推移逐句增长

3. **持续推送至完整方案**：本模块持续接收上游生成的每个文本增量，逐个封装推送，直到收到「生成完成」信号。家属看到完整方案逐句成形后，屏幕展示四段式应急建议的全部内容

4. **网络中断与恢复**：若推送过程中家属的网络连接临时中断（如手机信号切换），本模块等待前端重新连接。连接恢复后，推送从中断位置继续，家属可看到已接收的部分保持不变，后续内容继续追加

5. **异常终止**：若上游生成过程出现异常而提前中止，本模块将已产出的部分内容发送给家属，并明确标记「内容未完整」，家属可选择查看已有内容或重新发起咨询

### 1.6 输入/输出业务定义

#### 1.6.1 输入

| 字段名 | 业务含义 | 是否必填 | 业务约束 | 示例值 |
|--------|---------|---------|---------|--------|
| 文本增量内容 | 本次推送的一段应急方案文本片段 | 是 | 单次推文长度不做限制，可以是单个字符也可以是一段完整句子 | "请保持冷静，先将孩子带离当前环境" |
| 是否为最后片段 | 标记本次推送是否是生成的最后一段内容 | 是 | 仅两种状态，用于前端判断是否完成接收 | 否 / 是 |
| 生成完成原因 | 描述生成过程是如何结束的 | 否 | 仅当「是否为最后片段」为「是」时提供；可能的状态包括正常完成、内容被安全阻断、生成超时、生成出错 | "正常完成" |

#### 1.6.2 输出

| 字段名 | 业务含义 | 业务约束 | 示例值 |
|--------|---------|---------|--------|
| 事件类型标记 | 区分推送内容的不同用途 | 至少包含「文本内容」和「流结束」两种事件类型；「流结束」事件必须附带完成原因说明 | "文本内容" |
| 文本内容 | 本次推送的应急方案文本片段 | 原样传递上游文本，不做任何改写、截断或过滤 | "请保持冷静，先将孩子带离当前环境" |
| 流结束原因 | 流的终止状态说明 | 仅当事件类型为「流结束」时存在；取值与输入的「生成完成原因」一致 | "正常完成" |
| 流标识符 | 唯一标识本次推送会话 | 每条推送链路有唯一标识，用于前端重连时定位同一推送会话 | "stream-abc123" |

### 1.7 状态的业务定义

本功能点作为纯数据推送通道，不涉及业务状态流转。推送连接的生命周期由前端和服务器之间的连接管理机制处理（连接 → 推送中 → 断开/完成），该连接管理不属于本模块的业务职责范围，故无需状态机。

### 1.8 异常的业务策略

#### 1.8.1 异常 1：推送连接中断

- **业务触发条件**：在方案推送过程中，家属的网络连接意外中断（如手机信号切换、小程序切后台、网络波动）
- **业务处理策略**：
  1. 中断后系统为该推送会话保留已推送的进度位置
  2. 家属前端检测到断开后自动尝试重新建立连接
  3. 连接恢复后，推送从上次中断的位置继续，已接收的内容无需重传
  4. 家属在屏幕上看到已接收的内容保持不动，新内容在重连后继续追加
- **业务影响**：家属的等待时间因中断而延长，但不会丢失已推送的内容，也无需从零重新等待

#### 1.8.2 异常 2：上游生成异常中止

- **业务触发条件**：在方案推送过程中，上游 CSLT-03 生成模块因故无法继续产出内容（如生成超时、内容被安全阻断、大模型服务异常）
- **业务处理策略**：
  1. 若已生成并推送了部分内容，将已推送的片段标记为「部分结果」，并告知家属生成未完成的明确原因
  2. 若未生成任何内容，明确告知家属「当前无法生成方案」，提供重新发起咨询的指引
  3. 生成被安全阻断时，仅展示预设的安全提示文字，不展示任何被判定为不安全的内容
- **业务影响**：家属未获得完整方案，但获得明确的失败说明和后续操作引导

#### 1.8.3 异常 3：推送响应超时

- **业务触发条件**：上游生成模块在超出预期时间内未产出任何内容，家属长时间面对空白屏幕
- **业务处理策略**：
  1. 超过合理等待时间后，主动告知家属「正在生成中，请耐心等待」
  2. 若等待时间进一步延长仍未产生任何内容，告知家属当前异常并建议稍后重试
- **业务影响**：家属等待体验受损，但获得进度提示而非面对无反馈的空白屏

### 1.9 验收标准

| 编号 | 验收条件 | 验收方式 | 通过标准 |
|------|---------|---------|---------|
| AC-01 | 首字延迟达标 | 性能压测 | 在家属网络正常条件下，从提交行为描述到首段文本出现在屏幕上的时间 ≤ 3 秒 |
| AC-02 | 内容完整传输 | 集成测试 | 上游 CSLT-03 生成的完整方案文本与家属屏幕最终展示的文本完全一致，逐字比对无增删改 |
| AC-03 | 连接中断恢复 | 故障注入测试 | 模拟推送中途网络断开 10 秒后恢复，家属屏幕展示的完整内容与无中断场景一致，中断后推送从中断位置继续而非从零开始 |
| AC-04 | 流结束信号可靠 | 集成测试 | 每次推送终止时家属端必然收到明确的流结束事件，且完成原因与上游实际终止原因一致 |
| AC-05 | 异常中止内容保护 | 故障注入测试 | 上游异常中止时，家属端可看到已推送的部分内容，且收到明确的中止原因说明 |
| AC-06 | 并发推送互不干扰 | 并压测试 | 多个家属同时发起应急咨询时，各条推送链路的内容互不串扰，每条链路输出的文本仅属于该链路的输入文本 |

### 1.11 业务约束清单

1. **内容完整性约束**：推送的内容必须与上游 CSLT-03 生成的文本完全一致，传输层不得对文本做任何增删改、截断或格式转换
2. **首字延迟底线**：家属提交行为描述后，首段文本不超过 3 秒必须在屏幕上可见。此指标直接影响用户对系统响应能力的感知
3. **完整方案交付时限**：从提交到完整方案全部送达，总时长不超过 10 秒（包含上游生成耗时）
4. **连接中断不丢数据**：网络连接中断且恢复后，已推送的内容不可丢失、不可重复推送，家属看到的最终内容必须与未中断场景一致
5. **安全阻断内容隔离**：上游因安全原因阻断生成时，本模块只可推送预设的安全提示文本，不可推送任何被阻断前的生成片段
6. **设计边界**：本模块不负责判断推送内容的安全性（该职责归属 SEC-02 AI输出安全护栏）、不负责生成的文本内容质量（归属 CSLT-03）、不负责前端界面的渲染逻辑（归属 CSLT-08）、不负责推送连接的建立与认证（由 CSLT-08 和基础设施层处理）
