## 设计原则与约束

1. **views/logics 硬隔离**：所有页面（`pages/`）和组件（`components/`）禁止直接调用 `Taro.request()`、`fetch()` 或任何 API Service；禁止直接 import 或读写 Zustand Store。数据获取必须通过 `logics/` 层的 Hooks 桥接。

2. **单向数据流**：用户输入 → views 触发 action → logics 更新 store → views 重渲染。

3. **Props 驱动组件**：功能专有 UI 组件（如 `ChatBubble.tsx`、`CaseCard.tsx`）必须为纯展示组件，数据全部通过 Props 接收，回调通过 Props 传出。

4. **错误反馈时效**：所有提交失败场景下，错误提示必须在 500 毫秒内呈现在界面中（意图文档约束）。

5. **状态不持久化**：页面级状态均为内存态（Zustand store），不写入 Taro Storage。页面切换时通过 `resetFormState()` 重置。
