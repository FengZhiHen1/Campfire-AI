## 1. 页面概述

案例提交页是「篝火智答」案例库的唯一表单入口，同时承担**新建案例**（`mode=create`）和**编辑已有案例**（`mode=edit`）两种模式。所有案例数据的录入、修改、保存草稿、提交审核均在此页面完成。采用纵向长表单布局，按信息逻辑分组排列，确保作者能完整、清晰地记录干预案例的四段式核心内容。

**设计目标**：
- 表单逻辑唯一：新建和编辑共用同一套界面，减少代码重复和用户认知成本
- 降低录入门槛：必填字段明确标记，草稿随时保存，不强迫一次填完
- 四段式引导：每段提供填写提示，帮助作者按标准格式输出
- 审核前置感知：提交前预览 AI 预审结果，提前发现格式问题

## 2. 模式差异总览

| 维度 | 新建案例（`mode=create`） | 编辑案例（`mode=edit`） |
|:---|:---|:---|
| **页面标题** | "新建案例" | "编辑案例" |
| **初始数据** | 全部空白，必填字段高亮 | 加载已有案例全部数据，预填充表单 |
| **底部按钮** | "保存草稿" + "提交审核" 双按钮 | "保存修改" 单按钮 |
| **保存后状态** | `draft`（草稿）或 `pending_review`（提交审核） | 保持原状态（编辑已审核案例变为 `draft`，需重新提交） |
| **版本历史** | 无 | 提交后自动生成新版本（v{N+1}） |
| **删除入口** | 无 | 页面底部显示 "删除此案例" 危险操作区 |

## 4. 区域详细设计

### 4.1 顶部导航栏（NavBar）

| 属性 | 规格 |
|:---|:---|
| **高度** | `56px`（含状态栏安全区） |
| **背景** | `surface-dim` (`#EBE5DE`) |
| **取消按钮** | 左侧 `24px`，文字 "取消"，`14px`，`on-background`，`44×44` 触摸区；点击后弹出确认："放弃本次编辑？"（若表单有修改）或直接返回 |
| **标题** | 居中，`18px`，`font-weight: 600`，`on-background`；`mode=create` 时 "新建案例"，`mode=edit` 时 "编辑案例" |
| **保存按钮** | 最右侧，`✓` 勾选图标或 "保存" 文字，`primary` 色，`44×44` 触摸区；点击执行快速保存（不提交审核，仅保存当前表单数据） |
| **保存按钮状态** | 未修改时禁用（`on-surface-variant` 40% opacity）；已修改且校验通过时激活；保存中显示 spinner |
| **底部过渡** | 无分割线，`surface-dim` → `background` 自然分界 |

> **快速保存**：导航栏 "保存" 按钮是**兜底操作**，作者可随时点击保存当前进度（生成/更新 `draft`），无需滚动到底部。

| 循证等级 | 必填，必须选 1 项 | "请选择循证等级" |
| 场景描述 | 必填，`10-500` 字符 | "场景描述不能为空" / "描述过短/过长" |
| 行为表现 | 必填，`10-500` 字符 | 同上 |
| 干预动作 | 必填，`10-500` 字符 | 同上 |
| 结果反馈 | 必填，`10-500` 字符 | 同上 |

### 5.2 提交审核前的 AI 预审预览（可选增强）

点击 "提交审核" 后、正式提交前，可弹出一个轻量级预览面板：

| 属性 | 规格 |
|:---|:---|
| **触发** | 点击 "提交审核" 且前端校验通过后 |
| **面板** | 底部滑出 `surface` 面板，`16px` 顶部圆角 |
| **内容** | "AI 预审预览" + 4 项检查结果（格式/PII/必填/循证）+ "确认提交审核" 按钮 |
| **目的** | 让作者在提交前预知可能的驳回原因，减少审核往返次数 |
| **跳过** | 若 AI 预审全部通过，可自动跳过此面板直接提交 |

> **注**：此为 MVP 后的增强功能，文档中预留位置，首期可实现纯前端校验。

### 5.3 错误呈现方式

- 提交时若校验失败，页面**自动滚动**到第一个错误字段
- 错误字段显示 `error` ghost border + 下方 `error` 色提示文字 `12px`
- 顶部导航栏 "保存" 按钮变为 `error` 色，点击后同样触发滚动到错误字段

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 所有触摸目标 ≥ `44px`：输入框 `44px`、按钮 `44px` 高、循证等级卡片 `44px` 高
- 必填字段必须有明确的 `*` 标记 + 错误提示，屏幕阅读器可识别
- 四段内容卡片的段标题必须使用 heading 语义
- 底部操作栏按钮必须有明确的 `aria-label`（"保存为草稿" 而非 "保存"）
- 附件上传进度必须有 `aria-live` 区域通知屏幕阅读器

## 9. 与关联页面的协作规范

| 协作点 | 案例提交页行为 | 关联页面响应 |
|:---|:---|:---|
| **从列表新建** | 接收 `mode=create` | 列表页 FAB 点击 `navigateTo` 跳转 |
| **从详情编辑** | 接收 `mode=edit&case_id=xxx` | 详情页点击 "编辑案例" `navigateTo` 跳转，预填充数据 |
| **保存草稿** | 提交 `draft` 状态到后端 | 列表页和详情页均显示最新草稿数据 |
| **提交审核** | 提交 `pending_review` 状态 | 列表页状态变为 "审核中"；详情页审核面板显示 AI 预审结果 |
| **保存修改（编辑）** | 更新数据，若原 `approved` 则降级为 `draft` | 详情页自动刷新；列表页状态更新 |
| **删除案例** | 确认删除后 `navigateBack()` | 返回列表页，该案例移除；若从详情页进入编辑后删除，返回列表页 |
