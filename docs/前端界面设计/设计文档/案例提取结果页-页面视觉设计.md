## 1. 页面概述

提取结果页是「篝火智答」案例库叙事提取流程的**确认与微调环节**。用户从叙事提交页输入自然语言故事后，AI 自动提取出 N 张结构化干预卡片，用户在此页面逐张确认 AI 提取结果、修正推断字段、最终提交审核。页面承载"AI 初稿 → 人工确认 → 提交审核"的核心闭环。

**设计目标**：
- 概览先行：顶部概览浮层让用户一眼看到全部卡片的摘要和状态
- 对照编辑：可随时展开叙事原文，编辑卡片时参照原始故事
- 逐张确认，统一提交：编辑过程中自动保存，最终提交前弹出确认清单
- 推断透明：AI 推断的字段明确标注理由，降低人工判断成本

## 2. 页面状态总览

| 状态 | 触发条件 | 展示内容 |
|:---|:---|:---|
| **加载中** | 前端正在等待提取 API 返回 | 骨架屏 + "AI 正在分析叙事，提取干预卡片…" + 预计耗时提示 |
| **有结果**（正常态） | AI 成功提取 ≥ 1 张卡片 | 概览浮层 + 横滑 Tab 编辑区 + 底部操作栏 |
| **空结果** | AI 返回 0 张卡片 | 空状态插画 + 提示文字 + 双按钮兜底 |
| **提取失败** | API 异常或超时 | 错误图标 + 错误信息 + 重试按钮 |
| **已提交**（终态） | 用户提交审核后短暂展示 | 成功标记 + Toast，随即跳转案例列表 |

## 4. 区域详细设计

### 4.1 顶部导航栏（NavBar）

| 属性 | 规格 |
|:---|:---|
| **高度** | `56px`（含状态栏安全区） |
| **背景** | `surface-dim` (`#EBE5DE`) |
| **返回按钮** | 左侧 `24px`，`←` 箭头，`on-background`，`44×44` 触摸区；点击返回叙事提交页 |
| **标题** | 居中，`18px`，`font-weight: 600`，`on-background`；格式 "提取结果 (N 张卡片)" |
| **原文按钮** | 右侧 `16px`，文字 "原文"，`14px`，`primary`，`44×44` 触摸区；点击展开/收起原文折叠区；原文展开时按钮变为 `primary-container` 背景的激活态 |
| **底部过渡** | 无分割线，`surface-dim` → `background` 自然分界 |

---

### 4.2 叙事原文折叠区

| 属性 | 规格 |
|:---|:---|
| **默认状态** | 折叠，高度 `0`，不可见 |
| **展开状态** | 高度自适应（最大 `120px`，超出纵向滚动），`background` 背景 |
| **展开动画** | `max-height 0 → 120px`，`250ms` `ease-in-out` |
| **内边距** | 展开时 `12px 16px` |
| **原文文字** | `14px`，`on-background`，line-height `1.6`；首行缩进 `2em` |
| **标签** | 原文上方 "叙事原文" 标签，`11px`，`on-surface-variant`，`font-weight: 500` |
| **分隔线** | 原文区域下方 `1px` `outline` 15% opacity 分割线 |
| **收起方式** | 再次点击导航栏「原文」按钮、或向下滑动手势 |

> **触发逻辑**：导航栏「原文」按钮是 toggle 开关。页面初次进入时原文区折叠。用户展开后可对照原文编辑卡片。切换 Tab 时原文区保持当前展开/折叠状态。

### 4.3 概览浮层

| 属性 | 规格 |
|:---|:---|
| **高度** | `80px` |
| **背景** | `surface-variant` (`#F5F0EB`) |
| **内边距** | `12px 16px` |
| **布局** | 横向滚动（`ScrollView scrollX`），`12px` 间距 |
| **底部指示器** | 可选：若卡片数 > 3，底部显示分页圆点 |

**概览卡片（单个）**：

| 属性 | 规格 |
|:---|:---|
| **尺寸** | `140px` 宽 × `56px` 高 |
| **背景** | `surface` (`#FFFFFF`) |
| **圆角** | `8px` |
| **阴影** | `diffuse shadow` |
| **内边距** | `8px 10px` |
| **选中态** | 当前 Tab 对应的概览卡片显示 `primary` 色 `3px` 左侧强调条 |
| **点击行为** | 点击后横滑 Tab 切换到对应卡片，编辑区随之切换 |

**概览卡片内容**：

| 行 | 内容 | 样式 |
|:---|:---|:---|
| **第 1 行** | 卡片标题（如 "感官过载环境调整"） | `13px`，`font-weight: 600`，`on-background`；单行，溢出省略号 |
| **第 2 行** | 分类标签 pill（行为类型 + 严重程度） | `10px`，`primary-container` 背景 pill，`on-primary-container`，`4px 6px` padding |
| **第 3 行** | EBP 标签（取前 2 个，用 `/` 分隔） | `10px`，`on-surface-variant`，单行溢出省略 |
| **第 4 行** | 推断与保存状态指示 | 见下方状态指示规格 |

**状态指示**：

| 状态组合 | 展示 | 样式 |
|:---|:---|:---|
| 无推断、已保存 | `✓ 已确认` | `10px`，`tertiary` 色 |
| 有推断、已保存 | `⚠ N 字段推断` | `10px`，`warning` 色（`#D97706`） |
| 有推断、有未保存修改 | `⚠ N 推断 · ● 未保存` | `10px`，`warning` 色 + 红点 `●` `error` 色 `6px` |

> **自动保存时机**：用户切换 Tab、点击概览卡片、点击提交按钮时触发自动保存当前卡片。保存时为静默操作（无 Toast），概览卡片状态文字短暂变为 "保存中…"。

| **提示文字** | "提交后卡片将进入审核队列，审核通过后自动索引至案例库"，`12px`，`on-surface-variant`，line-height `1.5` |

| **确认按钮** | 右侧，`primary` 背景，`on-primary`，`16px`，`font-weight: 600`，`40px` 高，`8px` radius；若存在未保存卡片则按钮文字变为 "保存并提交" |

**确认后行为**：
- 弹窗关闭
- 若存在未保存卡片，先批量保存
- 逐张调用 `POST /api/v1/cards/{card_id}/submit`
- Toast："N 张卡片已提交审核"，`surface-dim` 背景，底部居中，`2s`
- `1s` 延迟后 `navigateBack()` 跳转案例列表页

## 5. 空状态与异常

### 5.1 空结果（AI 返回 0 张卡片）

- 页面中央显示空状态
- 图标：`📋` 或抽象文档图形（`on-surface-variant` 30% opacity，`80px`）
- 标题："AI 未能识别到干预场景"，`18px`，`font-weight: 600`，`on-background`
- 副标题："请检查叙事内容是否包含完整的干预事件描述（场景、行为、干预动作、结果）"，`14px`，`on-surface-variant`，line-height `1.5`，最大宽度 `280px` 居中
- **双按钮兜底**（横向排列，`12px` 间距）：
  - "返回修改原文"：`primary` 背景，`on-primary`，`14px`；点击 `navigateBack()` 返回叙事提交页，保留已填写内容
  - "手动录入"：`surface-variant` 背景，`on-background`，`14px`；点击跳转到 `submit.tsx`（结构化表单录入）

### 5.2 提取失败

- 图标：`⚠` 或 `error` 色感叹号图形，`32px`
- 标题："提取失败"，`16px`，`font-weight: 600`，`on-background`
- 错误详情：`error` 色文字，`14px`，展示后端返回的错误信息
- 操作按钮：
  - "重试"：`primary` 背景，`16px`，`48px` 高；重新调用提取 API
  - "返回修改"：`secondary` 文字按钮，`14px`；返回叙事提交页

### 5.3 加载中

- 页面中央显示加载动画
- 旋转 spinner：`primary` 色，`48px`
- 主文字："AI 正在分析叙事，提取干预卡片…"，`16px`，`on-background`
- 副文字："预计需要 10-30 秒"，`14px`，`on-surface-variant`
- 底部骨骼屏：模拟卡片表单的灰色骨架，`surface-variant` 背景，圆角，脉冲动画

## 6. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 从加载态进入有结果态 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **原文区展开** | 点击「原文」按钮 | `max-height 0→120px`，`250ms` `ease-in-out` | 折叠同理 |
| **概览卡片点击** | 点击概览浮层中的卡片 | `scale 0.96`，`80ms`；Tab 切换条 `translateX`，`200ms` `ease-out` | 联动切换 |
| **Tab 切换** | 点击横滑 Tab | 选中指示条 `translateX` 滑动，`200ms` `ease-out` | 自动保存后切换 |
| **推断标记消失** | 用户修改推断字段的值 | badge `opacity 1→0` + `scale 0.8→0`，`150ms` `ease-in` | 修改即消失 |
| **提交弹窗** | 点击提交按钮 | `opacity 0→1` + `scale 0.95→1`，`200ms` `ease-out` | 居中模态 |
| **提交成功** | 确认提交后 | 弹窗关闭 → Toast 滑入（`translateY→0`，`200ms`）→ `1s` 后 `navigateBack` | 页面退出 |

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 概览卡片和 Tab 触摸区 ≥ `44px`
- 推断标记不仅靠颜色区分（`⚠` 图标 + "推断" 文字），色盲友好
- 屏幕阅读器：概览卡片应朗读为"{标题}，{行为类型}，{推断数量}个字段需确认"
- 提交确认弹窗内卡片清单应朗读每张卡片的标题和状态
- 原文折叠区展开/收起状态应有 `aria-expanded` 标记

## 9. 与关联页面的协作规范

| 协作点 | 提取结果页行为 | 关联页面响应 |
|:---|:---|:---|
| **从叙事提交页进入** | 接收 `narrativeId` 参数，调用提取 API | 叙事提交页显示 "AI 正在提取" 加载态后重定向 |
| **返回修改原文** | 空结果时点击"返回修改原文" → `navigateBack()` | 叙事提交页保留用户之前填写的内容 |
| **手动录入** | 空结果时点击"手动录入" → `navigateTo('submit.tsx')` | 跳转结构化表单页 |
| **提交审核** | 逐张提交卡片 → Toast → `navigateBack()` | 案例列表页刷新，卡片状态变为"待审核" |
| **导航栏返回** | 点击返回 → `navigateBack()` | 叙事提交页恢复；已提取的卡片保留在数据库中（草稿状态） |
