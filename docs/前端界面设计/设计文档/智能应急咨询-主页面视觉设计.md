## 1. 页面概述

主咨询页是「篝火智答」最核心的用户触点。家属在孤独症人士出现行为危机时，通过此页面描述当前状况，接收 AI 基于真实案例生成的结构化应急方案。页面必须在高度焦虑的情境下传递**即时安全感**与**行动清晰度**。

**设计目标**：
- 零认知摩擦：任何元素都不得让用户思考"这是什么"
- 危机可视化：危机等级必须在 200ms 内被感知
- 行动导向：应急方案必须让人"立刻知道该做什么"
- 单手友好：全部主操作位于屏幕底部 40%

### 3.4 流式输出状态（SSE Streaming）

当 AI 正在生成回答时，不显示完整卡片外壳，而是以**流式气泡**形式逐字呈现。

**流式气泡外观**：
- 与普通 AI 气泡相同背景（`surface` + `diffuse shadow`）
- 内部文字以 `opacity 0 → 1` + `translateY(2px) → 0` 逐字浮现，`150ms`，`ease-out`
- 每字间隔 `40ms`
- 段落呼吸：每完成一段（即时干预 → 情绪安抚 → 观察指标 → 就医判断），自动插入 `12px` 垂直间距过渡（`height 0 → 12px`，`200ms` `ease-in-out`）

**打字机光标**：
- `2px` 宽垂直竖条，`primary` (`#B45309`) 色
- 脉动动画：`opacity 1 → 0.4`，`1.5s`，`ease-in-out`，`infinite`
- 位于 SSE token 边界精确位置
- **绝对禁止**使用方块光标或下划线光标

**流式期间输入区行为**：
- 底部输入框**保持可用**
- 用户可以边阅读边输入下一条追问
- 发送按钮在流式期间仍然可点击，新消息作为独立用户气泡插入
- AI 继续完成当前流式输出，完成后按顺序处理下一条

**加载骨架（首字到达前）**：
- 不使用灰色闪烁骨架
- 显示一个 `surface-variant` 静态块（宽 `120px`，高 `24px`，`8px` radius），带微弱 `primary` 脉冲边框（`opacity 0.3 → 0.6`，`2s`）
- 文字提示："正在分析案例库…"，`14px`，`on-surface-variant`，位于骨架下方 `8px`

### 3.6 置信度徽章（Confidence Badge）

位于 AI 方案卡片底部信息栏，标识当前回答的可信程度。

| 档位 | 条件 | 背景 | 文字色 | 文字内容 | 动画 |
|:---|:---|:---|:---|:---|:---|
| **高可信** | ≥ 0.85 | `tertiary` | `on-tertiary` | "高可信 ●" | 流式结束后 `opacity 0→1` + `translateY(4px)→0`，`300ms` `ease-out` |
| **中可信** | 0.70–0.84 | `primary-container` | `on-primary-container` | "中可信 ●" | 同上 |
| **低可信** | < 0.70 | `surface-variant` | `error` | "需人工复核 ●" | 同上，**同时触发人工兜底 sticky 条展开** |

- Badge 形状：`pill`，`8px` radius，`8px 12px` padding
- `●` 圆点直径 `8px`，与文字同色，用于快速色觉识别

### 3.7 人工兜底 Sticky 条（Human Escalation Bar）

**默认状态**：不显示（或显示为卡片底部 subtle 文字链接 "Need a human expert?"，`secondary` 色，`14px`）

**触发条件**（满足任一即触发）：
- 置信度 < 0.70
- 高风险关键词命中（自伤、自杀、药物、走失）
- 用户连续追问未解决

**触发后状态**：

| 属性 | 规格 |
|:---|:---|
| **位置** | 屏幕底部，输入区上方（或覆盖输入区上方边缘） |
| **背景** | `error` (`#DC2626`) |
| **文字** | "🚨 立即联系人工专家"，`16px`，`font-weight: 600`，`on-error` (`#FFFFFF`) |
| **高度** | `56px` |
| **圆角** | 顶部 `16px`（`border-radius: 16px 16px 0 0`） |
| **触摸区** | 全宽，无内边距死角 |
| **交互** | **点击立即生成工单，无确认弹窗**（Speed is safety） |
| **动画** | 从底部滑入（`translateY(100%) → 0`），`300ms` `ease-out` |

> **安全说明**：此条出现时，输入区仍然可用，但视觉上被挤压上移 `56px`。用户可继续与 AI 对话，同时知晓人工兜底通道已激活。

## 4. 状态与交互矩阵

### 4.1 页面状态流转

```
[首次进入] ──▶ 行为类型弹窗 ──选择+确认──▶ [空聊天态]
                                              │
                                              ▼
[已有咨询中] ◀── 返回/后台恢复 ───────── [用户输入中]
                                              │
                                              ▼
                                        [SSE 流式生成中]
                                              │
                            ┌───────────────┼───────────────┐
                            ▼               ▼               ▼
                      [生成完成-高可信] [生成完成-中可信] [生成完成-低可信]
                            │               │               │
                            └───────────────┴───────┬───────┘
                                                    ▼
                                          [人工兜底条展开（低可信时）]
                                                    │
                                                    ▼
                                              [等待用户追问]
```

### 4.2 交互规则表

| 场景 | 用户操作 | 系统响应 | 视觉变化 |
|:---|:---|:---|:---|
| 首次进入 | 打开页面 | 弹出行为类型选择弹窗 | 底部滑入模态层，背景变暗 |
| 行为选择 | 点击某个行为卡片 | 卡片变为 `primary-container` 背景，其他恢复默认；确认按钮激活 | 缩放 `0.97` 反馈，勾选标记出现 |
| 发送消息 | 点击发送/按回车 | 用户消息气泡右对齐插入；输入框清空；触发 SSE 请求 | 消息以 `opacity 0→1` + `translateY(8px)→0` 出现，`200ms` |
| SSE 首字到达 | 收到第一个 token | 骨架消失，流式气泡出现，打字机光标开始脉动 | 骨架 `opacity→0`，气泡 `opacity→1` |
| SSE 进行中 | 逐字渲染 | 文字逐字浮现，段落呼吸自动插入 | `opacity 0→1` + `translateY(2px)→0`，每字 `40ms` |
| SSE 完成 | 收到结束标记 | 流式气泡固化为完整 AI 气泡；打字机光标消失；置信度徽章淡入；若低可信则展开兜底条 | 徽章 `opacity 0→1` + `translateY(4px)→0`，`300ms` |
| 危机等级变化 | 后端推送等级更新 | 顶部徽章颜色切换 | 背景色过渡 `300ms`，重度时增加脉动 |
| 点击人工兜底 | 点击 sticky 条 | 立即生成工单；按钮变为加载态；成功后跳转工单详情 | 按钮文字变为 "正在连接专家…"，加载 spinner |
| 滚动历史 | 用户上滑查看旧消息 | 自动加载更早消息（分页，每页 20 条） | 顶部出现加载指示器 |

## 5. 空状态与异常

### 5.1 空聊天态（首次进入后、行为选择完成、尚未发送第一条消息）

- 聊天区中央显示一个 `Empty` 状态插图占位
- 插图：抽象的篝火/暖光几何图形（`primary` 色，线条风格），**禁止卡通人物**
- 标题："描述当前情况，获取应急建议"，`18px`，`on-background`，居中
- 副标题："基于真实干预案例，AI 将在几秒内生成个性化方案"，`14px`，`on-surface-variant`，居中
- 底部快捷入口：一排 `primary-container` pill 按钮（常见场景快捷触发词）

### 5.2 网络异常（发送失败）

- 用户消息气泡右侧显示红色感叹号 icon，`error` 色，`16px`
- 点击 icon 弹出 bottom sheet："消息发送失败" + "重试" 按钮
- **不**使用 Toast 或 Alert 打断当前流

### 5.3 后端服务不可用

- 输入框上方出现 `surface-variant` 背景的通知条（非阻断）
- 文字："当前咨询人数较多，响应可能延迟，请耐心等待"，`14px`，`on-surface-variant`
- 右侧显示关闭按钮 `×`
- `3s` 后自动消失

## 6. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 页面加载 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **弹窗滑入** | 行为类型选择/确认 | `translateY(100%)→0`，`300ms` `ease-out`；遮罩 `opacity 0→1` `200ms` | 底部 sheet 风格 |
| **消息出现** | 新消息插入 | `opacity 0→1` + `translateY(8px)→0`，`200ms` `ease-out` | 用户/AI 消息相同 |
| **Token 浮现** | SSE 逐字渲染 | `opacity 0→1` + `translateY(2px)→0`，`150ms` `ease-out` | 每字间隔 `40ms` |
| **打字机光标** | SSE 活跃期间 | `opacity 1→0.4`，`1.5s` `ease-in-out` `infinite` | `2px` 宽竖条，`primary` 色 |
| **段落呼吸** | 每段内容完成 | `height 0→12px`，`200ms` `ease-in-out` | 自动插入间距过渡 |
| **徽章浮现** | SSE 结束 | `opacity 0→1` + `translateY(4px)→0`，`300ms` `ease-out` | 置信度 badge |
| **危机脉动** | 重度等级激活 | `opacity 1→0.85`，`2s` `ease-in-out` `infinite` | 仅重度徽章 |
| **兜底条滑入** | 低可信/高风险触发 | `translateY(100%)→0`，`300ms` `ease-out` | 底部 sticky |
| **卡片缩放** | 点击行为选项 | `scale 1→0.97`，`100ms` `ease-out` | 按下反馈 |
| **手风琴展开** | 点击段标题 | `height` + `opacity`，`200ms` `ease-in-out` | 四段式卡片内部 |

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 全部触摸目标 ≥ `44×44px`（语音按钮 `48px`）
- 单手操作区：主要操作（发送、语音、兜底）位于屏幕底部 40%
- 色盲友好：危机等级除颜色外，文字标签必须同时变化（"轻度/中度/重度"）
- 屏幕阅读器：方案卡片四段式标题必须作为 `heading` 朗读；置信度徽章必须在方案生成完成后朗读
