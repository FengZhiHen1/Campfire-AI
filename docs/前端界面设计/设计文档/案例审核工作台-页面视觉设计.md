## 1. 页面概述

案例审核工作台是**专家角色**的专属工作页面，用于批量处理待审核的干预案例。工作台以紧凑列表形式展示所有 `pending_review` 状态的案例，专家可逐条进入底部审核面板深度审阅，也可通过多选批量处理 AI 预审全部通过的简单案例。页面追求**审核效率**与**决策准确性**的平衡。

**设计目标**：
- 效率优先：一屏展示 8-10 条待审案例，减少滚动
- AI 辅助决策：AI 预审进度条帮助专家快速识别问题案例
- 批量提速：简单案例批量通过，复杂案例逐条深度审核
- 质量兜底：批量驳回必须附统一意见，逐条驳回必须写具体意见

## 2. 角色与权限

| 角色 | 可访问 | 可见内容 | 操作权限 |
|:---|:---:|:---|:---|
| **专家** | ✓ | 全部 `pending_review` 案例 | 单条审核（通过/驳回）、批量通过、批量驳回（需填意见） |
| **管理员** | ✓ | 全部 `pending_review` 案例 + 审核统计 | 同专家 + 强制通过/驳回 |
| **老师/家属** | ✗ | — | 无权限访问，页面自动跳转案例列表 |

**第一行：复选框 + 案例标题**

| 元素 | 规格 |
|:---|:---|

| **标题** | 复选框右侧 `12px`，`15px`，`font-weight: 500`，`on-background`；单行，溢出省略号；最大宽度占行宽 `60%` |

**第二行：标签 + 提交者 + AI预审进度条 + 操作按钮**

| 元素 | 规格 |
|:---|:---|
| **行为类型标签** | `primary-container` pill，`on-primary-container`，`10px`，`4px 8px`，`4px` radius |
| **提交者** | 标签右侧 `8px`，`12px`，`on-surface-variant`；格式 "{昵称}" |
| **AI预审进度条** | 提交者右侧 `12px`，宽 `60px`，高 `6px`，`4px` radius；`surface-variant` 背景底；通过项填充 `tertiary`，未通过项填充 `error` 或留空 |
| **进度文字** | 进度条右侧 `4px`，`11px`，`on-surface-variant`；格式 "{通过数}/4" |
| **去审核按钮** | 最右侧，`primary` 文字按钮，`12px`，`44×44` 触摸区；文字 "审核" 或 "去审核"；点击后底部滑出审核面板 |

**进度条颜色逻辑**：

| 通过项数 | 进度条视觉 | 背景提示 |
|:---:|:---|:---|
| **4/4** | 全部 `tertiary` 绿色 | 行背景轻微 `tertiary` 3% tint（暗示可快速通过） |
| **3/4** | `tertiary` 75% + `error` 25% | 正常 `surface` 背景 |
| **2/4** | `tertiary` 50% + `error` 50% | 正常 `surface` 背景 |
| **1/4** | `tertiary` 25% + `error` 75% | 行左侧 `2px` `error` 色细条（暗示问题较大） |
| **0/4** | 全部 `error` 红色 | 行左侧 `2px` `error` 色细条 |

> **设计意图**：AI预审通过率直接影响行的视觉权重，4/4 的案例看起来"更健康"，专家可凭直觉快速识别简单案例。

### 4.5 批量操作栏

当用户勾选 **≥1** 个复选框时，底部滑出批量操作栏。

| 属性 | 规格 |
|:---|:---|
| **位置** | 页面底部，固定在安全区上方 |
| **背景** | `surface` (`#FFFFFF`) |
| **高度** | `64px`（含安全区） |
| **圆角** | 顶部 `16px` |
| **阴影** | `0 -4px 20px rgba(28,25,23,0.08)` |
| **内边距** | `12px 16px`，底部加 `env(safe-area-inset-bottom)` |
| **动画** | `translateY(100%)→0`，`250ms` `ease-out` |

**内部结构**：

```
┌─────────────────────────────────────────┐
│  已选择 3 条                            │
│  ┌──────────────┐  ┌──────────────┐    │
│  │   批量通过   │  │   批量驳回   │    │
│  │  (tertiary)  │  │   (error)    │    │
│  └──────────────┘  └──────────────┘    │
└─────────────────────────────────────────┘
```

| 元素 | 规格 |
|:---|:---|
| **选择计数** | 左侧，`14px`，`on-background`，`font-weight: 500`；"已选择 N 条" |
| **全选/取消** | 计数右侧，`12px`，`primary` 文字按钮；"全选" / "取消全选" |
| **批量通过按钮** | `tertiary` 背景，`on-tertiary` 文字，`14px`，`font-weight: 600`；`48%` 宽度，`40px` 高，`8px` radius |
| **批量驳回按钮** | `error` 背景，`on-error` 文字，`14px`，`font-weight: 600`；`48%` 宽度，`40px` 高，`8px` radius |
| **按钮间距** | `4%` |

**批量通过行为**：
- 点击后弹出确认："确定批量通过 N 条案例？" → 确认后全部标记为 `approved`
- 列表中被选中的行以 `opacity 1→0` + `translateX` 向左滑出消失，`200ms`
- Toast："N 条案例已审核通过"，`surface-dim` 背景，`on-background`，底部居中，`2s`

**批量驳回行为**：
- 点击后**不立即执行**，而是弹出驳回意见输入弹窗
- 弹窗规格见 §4.7

**批量操作栏消失**：
- 取消全部选择后，操作栏向下滑出（`translateY→100%`，`200ms`）

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 列表行高度 `72px` 远大于最小 `44px` 触摸目标
- 复选框 `20px` 方形 + `44×44` 触摸区满足要求
- 进度条不仅靠颜色区分，同时显示 "{通过数}/4" 文字，色盲友好
- 屏幕阅读器：列表行应朗读为"{标题}，{行为类型}，提交者{昵称}，AI预审{通过数}项通过"
- 批量操作必须朗读选中数量和操作按钮含义
- 审核面板内的通过/驳回按钮必须有明确的 `aria-label`（"审核通过此案例" 而非 "通过"）

## 9. 与关联页面的协作规范

| 协作点 | 审核工作台行为 | 关联页面响应 |
|:---|:---|:---|
| **从列表进入** | 案例列表页点击"审核台"按钮 → `navigateTo` 跳转 | 携带专家角色信息 |
| **查看完整案例** | 审核面板内点击 "查看完整案例" | 跳转 `detail.tsx`，加载该案例完整内容 |
| **审核通过** | 标记为 `approved`，面板关闭 | 列表页该案例状态变为"已审核"；详情页审核面板更新 |
| **审核驳回** | 标记为 `rejected` + 填写驳回意见，面板关闭 | 列表页该案例状态变为"已驳回"；详情页显示驳回意见；作者收到通知 |
| **批量通过** | 批量标记为 `approved` | 列表页批量更新；详情页同步 |
| **批量驳回** | 批量标记为 `rejected` + 统一驳回意见 | 列表页批量更新；所有被驳回案例的作者收到相同意见通知 |
