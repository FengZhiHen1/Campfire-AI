## 1. 页面概述

真实案例列表是「篝火智答」案例库的核心浏览入口。不同角色在此页面看到的内容和操作权限各不相同：家属浏览已审核通过的干预案例用于学习参考；老师/专家管理自己提交的案例并查看状态；专家还可快捷进入独立审核工作台处理待审案例。页面通过角色自适应在同一套 UI 骨架下呈现差异化的信息流与操作入口。

**设计目标**：
- 角色零感知切换：同一路由下，不同角色自然看到属于自己的内容，无突兀的权限提示
- 信息快速扫描：循证等级、审核状态、行为类型三个核心维度一眼可辨
- 审核效率优先：专家的待审入口直接、明显，不绕路
- 录入触手可及：老师/专家的 FAB 按钮随时可见，降低贡献门槛

## 2. 角色自适应总览

| 角色 | 可见案例范围 | 可见筛选项 | 可见操作入口 | 点击卡片去向 |
|:---|:---|:---|:---|:---|
| **家属** | 仅 `approved`（已通过审核） | 行为类型、循证等级、排序 | 无 | 案例详情页（纯浏览） |
| **老师** | 自己的全部案例（所有状态） | 行为类型、审核状态、循证等级、排序 | FAB（录入新案例） | 案例详情页（自己的可编辑） |
| **专家** | 全部案例（所有状态） | 行为类型、审核状态、循证等级、排序 | FAB + 审核台按钮 | 案例详情页（自己的可编辑） |
| **管理员** | 全部案例（所有状态） | 行为类型、审核状态、循证等级、排序 | FAB + 审核台按钮 | 案例详情页（可编辑任何案例） |

> **实现原则**：前端不根据角色隐藏整个页面区块（避免布局跳动），而是根据角色**调整下拉选项**、**隐藏/显示按钮**、**过滤列表数据**。所有角色共享同一套布局骨架。

## 5. 空状态与异常

### 5.1 无案例记录（系统级空状态）

整个案例库为空时（系统尚未录入任何案例）：

- 页面中央显示空状态插图
- 插图：抽象篝火暖光几何图形，`primary` 色，`120px`，`opacity 0→1` + `scale 0.9→1`，`400ms`
- 标题："案例库正在建设中"，`18px`，`font-weight: 600`，`on-background`
- 副标题："真实干预案例需要专业人士贡献"，`14px`，`on-surface-variant`
- **老师/专家额外显示**："成为第一个贡献者" 按钮，`primary` 背景，`on-primary`，`16px`，`48px` 高，`16px` radius，点击跳转 `submit.tsx`
- **家属显示**："案例库开放后将为您展示真实干预经验"，`14px`，`on-surface-variant`

### 5.2 当前角色无可见案例（角色级空状态）

- 家属视角下没有 `approved` 案例时：
  - 标题："暂无已发布案例"，`18px`
  - 副标题："专家团队正在审核案例中，请稍后再来"，`14px`
- 老师视角下自己没有案例时：
  - 标题："您还没有提交过案例"，`18px`
  - 副标题："分享您的干预经验，帮助更多家庭"，`14px`
  - 按钮："录入第一个案例"，`primary` 背景按钮

### 5.3 筛选无结果

- 搜索图标，`32px`，`on-surface-variant` 40% opacity
- 标题："未找到符合条件的案例"，`16px`，`on-background`
- 副标题："尝试更换筛选条件或关键词"，`14px`，`on-surface-variant`
- 按钮："清除全部筛选"，`primary` 文字按钮，`14px`，点击后重置所有筛选条件

### 5.4 网络异常

- 网络断开图标，`32px`，`on-surface-variant` 40% opacity
- 标题："无法加载案例"，`16px`
- 副标题："请检查网络连接后重试"，`14px`
- 按钮："重新加载"，`primary` 背景，`16px`，`48px` 高

## 6. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 打开案例列表 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **FAB 入场** | 页面加载完成 | `opacity 0→1` + `scale 0.8→1`，`300ms` `ease-out`，延迟 `200ms` | 悬浮按钮温和出现 |
| **FAB 滚动缩小** | 向下滚动 >100px | `scale 1→0.86`（`56px→48px`），`200ms` `ease-out` | 减少视觉干扰 |
| **卡片点击反馈** | 点击案例卡片 | `scale 0.98`，`100ms` `ease-out` | 按压感 |
| **筛选下拉展开** | 点击筛选按钮 | `opacity 0→1` + `translateY(-4px)→0`，`150ms` `ease-out` | 下拉面板 |
| **筛选切换** | 选择新筛选条件 | 列表 `opacity 1→0.5`（`100ms`）→ 刷新数据 → `opacity 0.5→1`（`200ms`） | 内容切换过渡 |
| **菜单展开** | 点击 `···` | 同筛选下拉动画 | 导航栏菜单 |
| **搜索过滤** | 输入关键词 | 列表即时重排，无过渡动画 | 响应优先 |
| **无限滚动加载** | 滚动至底部 | `primary` spinner 旋转，`1s` `linear` `infinite` | 标准加载动画 |
| **骨架脉冲** | 首次加载 | `primary` 边框 `opacity 0.3→0.6`，`2s` `ease-in-out` `infinite` | 参照 DESIGN.md |

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 案例卡片整行为触摸目标，高度 `120px` 远大于最小 `44px`
- FAB `56px` 圆形触摸区满足要求
- 色盲友好：循证等级除颜色外，字母标识（A/B/C/D）同时显示；审核状态除颜色圆点外，文字标签同时显示
- 屏幕阅读器：卡片应朗读为"{标题}，{行为类型}，{适用人群}，{循证等级}级，{审核状态}，发布于{日期}"
- 角色自适应不应影响屏幕阅读器的正常朗读顺序
