## 1. 页面概述

档案编辑页是档案数据的核心维护入口，同时承担**新建档案**（`mode=create`）和**编辑档案**（`mode=edit`）两种场景。页面上半部分为固定可编辑的基础信息与标签体系，下半部分为事件记录的管理区（查看、编辑、删除历史事件）。

**设计目标**：
- 新建时降低门槛：必填字段仅 3 项，其余可后续补充
- 编辑时信息完整：所有档案维度（基础信息、标签、事件）在一个页面内闭环管理
- 快速录入不阻塞：首页 Bottom Sheet 2 字段快速记录，后续在编辑页补全
- 删除操作防误触：隐藏在菜单中，需二次确认

点击事件行删除按钮后：
- 事件行左右抖动（`translateX: 0 → -4px → 4px → -4px → 0`，`300ms`）
- 同时弹出确认对话框（非阻断式 bottom sheet）：

## 4. 保存与校验

### 4.1 表单校验规则

| 字段 | 校验规则 | 错误提示 |
|:---|:---|:---|
| 档案昵称 | 必填，`2-20` 字符 | "昵称不能为空" / "昵称长度为 2-20 个字符" |
| 出生日期 | 必填，合法日期，不能为未来日期 | "请选择出生日期" / "日期不能晚于今天" |
| 诊断类型 | 必填，必须从下拉选项中选择 | "请选择诊断类型" |
| 主要行为 | 必填，必须从下拉选项中选择 | "请选择主要行为类型" |

### 4.2 保存按钮状态

| 状态 | 视觉表现 | 说明 |
|:---|:---|:---|
| **未修改** | `✓` 图标，`on-surface-variant` 40% opacity，不可点击 | 用户未做任何修改 |
| **已修改且校验通过** | `✓` 图标，`primary` 色，可点击 | 正常保存态 |
| **已修改但校验未通过** | `✓` 图标，`error` 色，可点击但点击后滚动到第一个错误字段并显示提示 | 提示用户修正 |
| **保存中** | `✓` 图标变为 loading spinner，`primary` 色，不可点击 | 网络请求中 |

### 4.3 保存成功/失败反馈

- **成功**：`Taro.navigateBack()` 返回首页，首页自动刷新显示最新数据；不显示 Toast（减少打扰）
- **失败**：顶部出现非阻断通知条，`surface-dim` 背景，"保存失败，请检查网络后重试"，`14px`，`error` 色文字，右侧 `×` 关闭按钮，`3s` 后自动消失

## 6. 与首页快速录入 Bottom Sheet 的协作

### 6.1 Bottom Sheet 规格

当用户在**档案首页**点击时间线标题区的 **"+ 记录"** 时，从底部滑出快速录入面板。

| 属性 | 规格 |
|:---|:---|
| **高度** | 屏幕高度的 `55%`（最小 `400px`） |
| **背景** | `surface` (`#FFFFFF`) |
| **圆角** | 顶部 `16px`（`border-radius: 16px 16px 0 0`） |
| **阴影** | `deep shadow` |
| **动画** | `translateY(100%)→0`，`300ms` `ease-out`；遮罩 `opacity 0→0.5`，`200ms` |
| **关闭方式** | 点击遮罩区域、向下滑动手势、点击顶部横条均可关闭 |

**顶部拖动手柄**：
- 面板顶部居中，`36px` 宽、`4px` 高、`2px` radius 的横条，`surface-dim` 色
- 提示用户可向下拖动关闭

### 6.2 快速录入表单（Step 1：2 字段）

| 属性 | 规格 |
|:---|:---|
| **标题** | "快速记录事件"，`18px`，`font-weight: 600`，`on-background`；下方副标题 "先记录关键信息，稍后可在档案中补全"，`12px`，`on-surface-variant` |
| **字段 1：触发因素** | 标签 "发生了什么？（触发因素）"，`14px`，`font-weight: 500`，`on-background`，`*` 必填标记；下方多行文本域，`surface-variant`，`8px` radius，`12px` padding，Placeholder "如：在超市遇到噪音刺激…" |

| **文本域高度** | 初始 `64px`（3 行），可扩展至 `96px` |
| **提交按钮** | 底部固定，`primary` 背景，`on-primary` 文字，"保存记录"，`16px`，`48px` 高，`16px` radius（顶部两角） |

- 面板关闭，首页时间线顶部自动插入新事件

### 6.3 补全提醒机制

- 在**档案首页**，待补全事件的卡片上显示 `error` 色 `⚠` 标记
- 在**档案编辑页**，待补全事件摘要行显示 "待补全" 状态
