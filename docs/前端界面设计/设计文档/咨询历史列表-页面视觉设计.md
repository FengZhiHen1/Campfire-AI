## 1. 页面概述

咨询历史列表是用户回顾过往应急咨询记录的入口。家属可在此查看每一次咨询的危机等级、AI 置信度、行为类型等关键元数据，通过搜索快速定位特定事件，并点击任意记录回到对应会话继续追问。

**设计目标**：
- 快速扫描：一屏内让用户感知"最近发生了几次危机、严重程度如何"
- 精准回溯：搜索 + 结构化元数据帮助用户快速找到某次特定事件
- 无缝续接：点击即回到主咨询页加载上下文，延续咨询不中断
- 焦虑友好：宽松卡片、充足留白，不制造信息压迫感

#### 3.3.1 历史记录卡片

**外壳**：

| 属性 | 规格 |
|:---|:---|
| **背景** | `surface` (`#FFFFFF`) |
| **阴影** | `diffuse shadow` (`0 4px 24px rgba(28,25,23,0.06)`) |
| **圆角** | `16px` |
| **高度** | 约 `100px`（内容自适应，最小 `100px`） |
| **内边距** | `16px` |
| **触摸反馈** | 点击时卡片缩放至 `0.98`，`100ms` `ease-out` |

**卡片内部结构**（从上到下的信息层级）：

```
┌─────────────────────────────────────────┐
│  ● 轻度              高可信  ●  今天 14:32 │  ◄── 第一行：等级+置信度+时间
│                                         │
│  自伤行为                               │  ◄── 第二行：行为类型标签
│                                         │
│  孩子在商场突然情绪崩溃，开始咬自己的手…  │  ◄── 第三行：用户提问摘要
└─────────────────────────────────────────┘
```

**第一行：危机等级 + 置信度 + 时间**

| 元素 | 规格 |
|:---|:---|
| **危机等级圆点** | `8px` 圆形，左侧 `0px`（与卡片左内边距对齐），颜色跟随等级：`tertiary`（轻度）、`primary`（中度）、`error`（重度） |
| **等级文字** | 紧跟圆点右侧，`4px` 间距，`14px`，`font-weight: 600`，颜色与圆点相同 |
| **置信度徽章** | 右侧对齐，pill 形状，`8px` radius，`6px 10px` padding，文字 `11px` |
| | · 高可信（≥85%）：`tertiary` 背景，`on-tertiary` 文字，文字 "高可信 ●" |
| | · 中可信（70-84%）：`primary-container` 背景，`on-primary-container` 文字，文字 "中可信 ●" |
| | · 低可信（<70%）：`surface-variant` 背景，`error` 文字，文字 "需复核 ●" |
| **时间戳** | 最右侧，`11px`，`on-surface-variant`，格式："今天 HH:mm" / "昨天 HH:mm" / "M月D日 HH:mm" |

**第二行：行为类型标签**

| 属性 | 规格 |
|:---|:---|
| **内容** | 用户在首次进入时选择的行为类型（自伤 / 攻击 / 逃跑 / 用药） |
| **样式** | `primary-container` 背景 pill，`on-primary-container` 文字，`11px`，`6px 10px` padding，`8px` radius |
| **位置** | 第一行下方 `8px` |

**第三行：用户提问摘要**

| 属性 | 规格 |
|:---|:---|
| **内容** | 用户该次咨询的第一条（或最主要一条）提问文本的前 `24` 个字符 + "…" |
| **样式** | `14px`，`on-surface-variant`，单行，溢出显示省略号 |
| **位置** | 第二行下方 `8px` |

> **摘要规则**：若用户提问不足 24 字则显示全文；若超过则截断并加省略号。摘要内容应脱敏处理（不显示真实人名、地点）。

### 3.4 左滑删除手势

**手势定义**：
- 触发：手指在卡片上从右向左水平滑动超过 `80px`
- 取消：滑动不足 `80px` 或手指抬起时自动回弹

**露出层（滑动后卡片下方露出的操作区）**：

| 属性 | 规格 |
|:---|:---|
| **背景** | `error` (`#DC2626`) |
| **宽度** | `80px`（固定，滑动超过此距离后卡片不再左移） |
| **图标** | 白色垃圾桶/删除图标，`24px`，居中 |
| **文字** | "删除"，`12px`，`on-error`，图标下方 `4px` |

**删除确认**：
- 点击删除按钮后，**不弹出二次确认弹窗**（DESIGN.md "Speed is safety" 原则，但此处为数据删除，需谨慎）
- 改为：卡片以 `translateX` 继续向左滑出屏幕（`300ms` `ease-in`），同时显示全局 `Toast`："咨询记录已删除"，`surface-dim` 背景，`on-background` 文字，`14px`，底部居中，`2s` 后自动消失
- 误删恢复：Toast 旁附带 "撤销" 文字按钮，`primary` 色，点击后恢复该条记录

**回弹动画**：
- 未触发删除时，手指松开后卡片回弹至原位：`translateX` 归零，`200ms` `spring`（带轻微弹性）

### 3.5 无限滚动加载

**触发条件**：用户滚动至距离列表底部 `100px` 内时，自动请求下一页数据。

**加载指示器**（列表底部）：

| 状态 | 视觉表现 |
|:---|:---|
| **加载中** | 居中的加载 spinner（`primary` 色，`20px`，旋转动画）+ 文字 "加载中…"，`14px`，`on-surface-variant`，上下 `16px` 间距 |
| **无更多数据** | 文字 "—— 没有更多记录了 ——"，`12px`，`on-surface-variant` 50% opacity，居中，上下 `24px` |
| **加载失败** | 文字 "加载失败，点击重试"，`14px`，`error` 色，可点击重新请求 |

**加载骨架**（首次进入页面数据尚未返回时）：
- 不使用灰色闪烁骨架
- 显示 `3` 个静态 `surface-variant` 块（宽 `100%`，高 `100px`，`16px` radius），带微弱 `primary` 脉冲边框
- 文字提示："正在整理您的咨询记录…"，`14px`，`on-surface-variant`，居中

## 4. 点击行为与页面跳转

### 4.1 点击历史记录卡片

**操作**：点击任意卡片 → 跳转至主咨询页 (`consult/pages/index.tsx`)。

**状态传递**：
- 通过路由参数或全局 Store 传递该次咨询的 `consultation_id`
- 主咨询页接收到 `consultation_id` 后：
  1. 加载该次历史咨询的完整对话上下文（用户消息列表 + AI 回复列表）
  2. 按时间顺序渲染到聊天区
  3. 顶部危机徽章显示该次咨询的最终危机等级
  4. 底部输入区保持可用，用户可继续追问
  5. **不**再次弹出行为类型选择弹窗（已有历史上下文）

**跳转动画**：
- `Taro.navigateTo()` 路由压栈
- 页面过渡：`opacity 0→1`，`200ms`（DESIGN.md 规范）

### 4.2 主咨询页的历史态标识

当主咨询页加载的是历史会话时，需在界面上做**轻微区分**，让用户意识到自己正在查看/延续一次历史咨询而非新建咨询：

- 顶部导航栏标题从 "应急咨询" 变为 "应急咨询 · 历史会话"，`on-surface-variant` 副标题样式
- 或：在聊天区域顶部显示一条 subtle 分隔提示："—— 以下为历史记录，您可继续追问 ——"，`12px`，`on-surface-variant` 50% opacity
- **禁止**使用阻断式弹窗或 Alert 提示

## 5. 空状态与异常

### 5.1 无历史记录（首次使用或所有记录已删除）

- 列表区域中央显示 `Empty` 状态
- 插图：抽象的篝火暖光几何图形（`primary` 色，线条风格），`80px` 高
- 标题："暂无咨询记录"，`18px`，`on-background`，居中
- 副标题："当需要应急建议时，前往应急咨询页面发起对话"，`14px`，`on-surface-variant`，居中
- 操作按钮："前往咨询"，`primary` 背景，`on-primary` 文字，`16px`，`48px` 高，`16px` radius，点击后 `Taro.redirectTo()` 跳转主咨询页
- 上下间距：插图距上方搜索栏 `80px`，按钮距插图 `24px`

### 5.2 搜索无结果

- 列表区域中央显示搜索结果为空提示
- 图标：`🔍` 搜索图标，`32px`，`on-surface-variant` 40% opacity
- 标题："未找到相关记录"，`16px`，`on-background`
- 副标题："尝试更换关键词，或检查拼写"，`14px`，`on-surface-variant`
- 操作按钮："清除搜索"，`primary` 文字按钮（无背景），`14px`，点击后清空搜索框并恢复全部列表

### 5.3 网络异常（列表加载失败）

- 列表区域显示错误状态
- 图标：网络断开抽象图标（`on-surface-variant` 40% opacity）
- 标题："无法加载记录"，`16px`，`on-background`
- 副标题："请检查网络连接后重试"，`14px`，`on-surface-variant`
- 操作按钮："重新加载"，`primary` 背景按钮，`16px`，`48px` 高

## 6. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 打开历史列表页 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **卡片点击反馈** | 点击卡片 | `scale 1→0.98`，`100ms` `ease-out` | 按压感 |
| **卡片释放回弹** | 手指抬起（未触发删除） | `translateX → 0`，`200ms` `spring` | 带轻微弹性 |
| **卡片滑出删除** | 点击删除按钮 | `translateX` 向左滑出屏幕，`300ms` `ease-in` | 卡片消失 |
| **Toast 浮现** | 删除成功 | `opacity 0→1` + `translateY(8px)→0`，`200ms` `ease-out` | 底部居中 |
| **Toast 消失** | `2s` 后 | `opacity 1→0`，`200ms` `ease-in` | 自然淡出 |
| **搜索过滤** | 输入关键词 | 列表即时重排，无过渡动画 | 响应优先 |
| **无限滚动加载** | 滚动至底部 | Spinner 旋转，`primary` 色，`1s` `linear` `infinite` | 标准加载动画 |
| **骨架脉冲** | 首次加载 | `primary` 边框 `opacity 0.3→0.6`，`2s` `ease-in-out` `infinite` | 参照 DESIGN.md |
| **空状态进入** | 无记录/搜索为空 | `opacity 0→1` + `translateY(12px)→0`，`300ms` `ease-out` | 温和出现 |

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 卡片整体为触摸目标，高度 `100px` 远大于最小 `44px`
- 左滑删除：滑动距离阈值 `80px`，防止误触；删除后提供 `2s` 内撤销机制
- 色盲友好：危机等级除颜色圆点外，文字标签同时显示（"轻度/中度/重度"）
- 屏幕阅读器：每条卡片应朗读为"{行为类型}，{危机等级}，{置信度}，{时间}，{摘要}"
