## 1. 页面概述

档案首页是家属管理孤独症人士个性化信息的核心页面。一个家属账号最多可维护 5 份独立档案，每份档案包含基础标签（出生日期、诊断类型、主要行为类型）和事件记录时间线。这些数据直接驱动应急咨询时的 RAG 检索精准度——档案越完整，AI 给出的建议越贴合个体。

**设计目标**：
- 档案切换一目了然：横向滑动快速在不同孩子/个案间切换
- 信息密度适中：关键标签一眼可见，不淹没在细节中
- 事件时间线清晰：按时间回溯行为与干预历史，帮助家属发现规律
- 冷启动温和引导：新用户无档案时不制造焦虑，用温暖插图引导创建

## 3. 区域详细设计

### 3.1 顶部导航栏（NavBar）

| 属性 | 规格 |
|:---|:---|
| **高度** | `56px`（含状态栏安全区） |
| **背景** | `surface-dim` (`#EBE5DE`) |

> **编辑按钮逻辑**：点击后跳转至 `edit.tsx`（档案编辑页），携带当前选中档案的 `profile_id`。

### 3.2 横向档案切换条

| 属性 | 规格 |
|:---|:---|
| **背景** | `background` (`#FDF8F3`) |
| **高度** | `88px` |
| **内边距** | 水平 `16px`，垂直 `12px` |
| **滚动方向** | 横向滚动 (`overflow-x: auto`，隐藏滚动条) |
| **卡片间距** | `12px` |

**单个档案切换卡片**：

| 属性 | 规格 |
|:---|:---|
| **尺寸** | 宽 `64px`，高 `64px`（正方形） |
| **背景** | 未选中：`surface-variant` (`#F5F0EB`)；选中：`primary-container` (`#FEF3C7`) |
| **圆角** | `16px` |
| **头像占位** | 居中，`32px` 圆形，`surface` 背景，内部显示孩子抽象头像图标（`on-surface-variant`） |
| **档案昵称** | 头像下方 `4px`，`11px`，最多显示 4 个汉字，溢出省略号 |
| | 未选中：`on-surface-variant`；选中：`on-primary-container` |
| **选中指示器** | 选中卡片底部中心有一个 `4px` 高、`16px` 宽的 `primary` 色小横条，圆角 `2px` |

**新建档案卡片（"+" 卡片）**：

| 属性 | 规格 |
|:---|:---|
| **尺寸** | 与档案卡片相同，`64×64px` |
| **背景** | `surface` (`#FFFFFF`) |
| **边框** | `1px` dashed `outline` 30% opacity（Ghost Border 规则的唯一例外——虚线用于暗示"可添加"） |
| **圆角** | `16px` |
| **图标** | 居中，`24px`，`+` 加号，`primary` 色 |
| **文字** | "新建"，`11px`，`primary`，头像下方 |
| **触摸反馈** | `scale 0.95`，`100ms`；点击后跳转新建档案流程 |

**切换动画**：
- 横向滑动档案条时无特殊动画（原生滚动）
- 点击某个档案卡片后，下方内容区切换：`opacity 0→1` + `translateY(4px)→0`，`200ms` `ease-out`
- 选中状态横条切换：`scaleX(0)→scaleX(1)`，`150ms` `ease-out`，从卡片底部中心展开


**头像区域**：

| 属性 | 规格 |
|:---|:---|
| **尺寸** | `56px` 圆形 |
| **背景** | `surface-variant` (`#F5F0EB`) |
| **图标** | 孩子/人物抽象轮廓图标，`28px`，`on-surface-variant` |
| **位置** | 卡片左上角 |

**档案昵称**：

| 属性 | 规格 |
|:---|:---|
| **文字** | 档案昵称（如"小明"），`20px`，`font-weight: 600`，`on-background` |
| **位置** | 头像右侧，`16px` 间距，垂直居中于头像 |
| **最大宽度** | 卡片宽度的 `60%`，溢出省略号 |

**分割线**：

- 头像昵称行下方 `16px` 处，一条水平分割线
- **不使用 1px solid border**（No-Line Rule）
- 改用 `height: 1px`，`background: outline` 15% opacity，贯穿卡片内部宽度

**必填标签行**：

三个标签横向排列，左对齐，`8px` 间距：

| 标签 | 图标 | 示例值 | 背景色 | 文字色 |
|:---|:---|:---|:---|:---|
| **出生日期** | 🎂 / cake | "2018-03-12" | `surface-variant` | `on-background` |
| **诊断类型** | 🏷️ / tag | "ASD" / "ADHD" / "其他" | `primary-container` | `on-primary-container` |
| **主要行为** | ⚡ / zap | "自伤行为" / "攻击行为" / … | `tertiary` 背景 (若行为含高风险) 或 `surface-variant` | `on-tertiary` 或 `on-background` |

- 每个标签为 `pill` 形状，`8px` radius，`8px 12px` padding
- 标签内文字 `12px`，`font-weight: 500`
- 标签内图标 `12px`，文字左侧 `4px`
- 若标签文字过长（如诊断类型全称），显示缩写 + 省略号，点击可展开 tooltip

**创建时间**：

| 属性 | 规格 |
|:---|:---|
| **文字** | "创建于 YYYY-MM-DD"，`12px`，`on-surface-variant` 70% opacity |
| **位置** | 标签行下方 `16px`，左对齐 |
| **图标** | 可选，一个小时钟图标，`12px`，`on-surface-variant` 50% opacity |

> **注意**：首页卡片**不展示自定义标签**（自定义标签在编辑页展示），保持首页精简。

| **位置** | 事件类型标签下方 `6px` |

#### 3.5.3 时间线分组（自然时间分组，无 sticky header）

虽然不做筛选，但时间线按自然时间形成视觉分组：

- **今天** 的事件：时间轴圆点为 `primary` 色（最新）或 `surface` 带边框（其他）
- **昨天** 的事件：圆点为 `surface` 带边框
- **更早** 的事件：圆点逐渐变小至 `8px`，颜色变浅（`outline` 20% opacity 边框）
- 同一天内的多个事件：时间戳显示相同日期，视觉上自然形成一组

> **设计意图**：不添加显式的"今天/昨天/本周" sticky 分组标题，保持时间线的纯粹和简洁。时间信息本身已足够形成心理分组。

## 4. 状态与交互矩阵

### 4.1 页面状态流转

```
[首次进入档案页]
       │
       ├──────────── 有档案 ────────▶ [正常态：显示档案信息+时间线]
       │                                    │
       │                                    ├── 横向切换档案 ──▶ 内容区刷新
       │                                    │
       │                                    ├── 点击编辑 ──────▶ 跳转 edit.tsx
       │                                    │
       │                                    ├── 点击新建事件 ──▶ 弹出快速录入 Bottom Sheet
       │                                    │
       │                                    └── 点击事件行 ────▶ 跳转事件详情页
       │
       └──────────── 无档案 ────────▶ [冷启动态：引导创建]
                                            │
                                            └── 点击创建按钮 ──▶ 跳转档案创建流程
```

| 场景 | 用户操作 | 系统响应 | 视觉变化 |
|:---|:---|:---|:---|
| 横向切换档案 | 点击/滑动档案条 | 加载对应档案数据 | 选中卡片背景变 `primary-container`，横条展开；下方内容区 `opacity` + `translateY` 过渡 |
| 点击编辑 | 点击导航栏"编辑" | 跳转 `edit.tsx` | `Taro.navigateTo()`，`opacity` 页面过渡 |
| 点击新建档案 | 点击 "+" 卡片 | 跳转档案创建流程 | 卡片 `scale 0.95` 反馈 |
| 点击新建事件 | 点击时间线标题区 "+ 记录" | 弹出快速录入 Bottom Sheet | 按钮 `scale 0.95` 反馈；提交后时间线顶部插入带 `[待补全 ⚠]` 标记的新事件 |
| 点击事件行 | 点击某条事件 | 跳转事件详情页 | 卡片 `scale 0.98` 反馈 |
| 滚动时间线 | 上滑浏览更早事件 | 自然滚动 | 无特殊动效 |
| 冷启动进入 | 打开页面且无档案 | 展示引导插图+按钮 | 插图 `opacity` + `scale` 入场动画 |

## 5. 空状态与异常

### 5.1 有档案但无事件记录

- 档案信息卡片正常显示
- 时间线区域显示空状态：
  - 时间轴轴线只显示一个孤零零的圆点（`surface` 背景，`outline` 边框）
  - 圆点右侧文字："暂无事件记录"，`14px`，`on-surface-variant`
  - 副文字："点击右上角 + 记录孩子的第一次行为事件"，`12px`，`on-surface-variant` 60% opacity
  - 下方显示一个 subtle 的 "+ 记录事件" 按钮，`primary` 文字按钮（无背景），`14px`；点击同样弹出快速录入 Bottom Sheet

### 5.2 档案数量达到上限（5 个）

- "+ 新建"卡片仍然显示，但变为禁用态
- 背景 `surface-dim`，`+` 图标和 "新建" 文字变为 `on-surface-variant` 40% opacity
- 点击时无响应，并显示 Toast："最多可创建 5 份档案"，`surface-dim` 背景，`on-background`，底部居中，`2s`

### 5.3 网络异常（档案数据加载失败）

- 页面中央显示错误状态
- 图标：网络断开抽象图标，`32px`，`on-surface-variant` 40% opacity
- 标题："无法加载档案"，`16px`，`on-background`
- 副标题："请检查网络连接后重试"，`14px`，`on-surface-variant`
- 按钮："重新加载"，`primary` 背景，`16px`，`48px` 高

## 6. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 打开档案首页 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **档案切换** | 点击横向档案卡片 | 内容区 `opacity 0→1` + `translateY(4px)→0`，`200ms` `ease-out` | 内容刷新过渡 |
| **选中指示器** | 档案切换 | `scaleX(0)→scaleX(1)`，`150ms` `ease-out` | 底部横条从中心展开 |
| **卡片点击反馈** | 点击事件行/新建按钮 | `scale 0.98`（事件行）或 `0.95`（新建卡片），`100ms` `ease-out` | 按压感 |
| **冷启动插图** | 无档案时页面进入 | `opacity 0→1` + `scale 0.9→1`，`400ms` `ease-out` | 温和入场 |
| **最新事件圆点** | 时间线渲染 | `box-shadow` `primary` 20% opacity 脉动，`2s` `ease-in-out` `infinite` | 强调最新事件 |
| **Toast 浮现** | 档案上限提示 | `opacity 0→1` + `translateY(8px)→0`，`200ms` `ease-out` | 底部居中 |

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 档案切换卡片 `64×64px` 大于最小 `44px` 触摸目标；事件行整行可点击，高度 `60px` 满足要求
- 色盲友好：危机/评估标记除颜色外，文字标签同时显示（"已评估"）
- 屏幕阅读器：横向档案切换条应朗读为"档案切换，当前选中：{昵称}"；时间线应朗读为"{日期}，{事件类型}，{摘要}"
- 冷启动引导：按钮必须大于 `44×48px`，当前 `240×48px` 远超标准
