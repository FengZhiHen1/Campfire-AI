## 1. 页面概述

首页是用户登录「篝火智答」后看到的第一个页面，也是底部 TabBar 的**第一项**。它不承担具体的业务功能，而是作为一个**综合仪表盘（Dashboard）**，将用户最关心的信息聚合在一屏内：应急咨询快捷入口、最近咨询记录、个人档案快照。所有角色（家属/老师/专家/管理员）共享同一套布局骨架，仅数据内容根据角色动态变化。

**设计目标**：
- 零秒启动：用户打开小程序，一眼就知道最核心的功能在哪里
- 上下文感知：根据时间、角色、数据状态动态调整展示内容
- 温暖不压迫：避免信息过载，只展示最重要的 3 个模块摘要
- 行动导向：每个区域都有明确的点击去向，不制造"这是什么"的困惑

| **背景** | `background` (`#FDF8F3`) |

**品牌名**：

| 属性 | 规格 |
|:---|:---|
| **文字** | "篝火智答"，`14px`，`font-weight: 500`，`primary` (`#B45309`) |
| **图标** | 右侧可选一个 `🔥` 小火焰图标，`14px`，`primary` 色 |
| **位置** | 左对齐 |

**问候语**：

| 属性 | 规格 |
|:---|:---|
| **文字** | 根据时间和用户身份动态生成 |
| **格式** | "{时段}，{昵称}" |

> **设计意图**：问候语是用户打开小程序后看到的第一个文字，必须传递温暖和关怀，而非冷冰冰的功能列表。

| **圆角** | `20px` |

| **外边距** | 左右 `16px`，上下 `16px` |
| **内边距** | `24px` |
| **阴影** | `0 8px 32px rgba(180,83,9,0.25)`（`primary` 色扩散投影，营造温暖光晕） |

```

│                                         │
│    🚨                                   │
│                                         │
```

**图标**：

| 属性 | 规格 |

| 属性 | 规格 |

| **指示器** | 图标上方 `2px` 处一个 `4px` 宽、`4px` 高的 `primary` 色圆点（或底部 `2px` 高横条） |

**未选中态**：

| 属性 | 规格 |
|:---|:---|
| **图标** | `on-surface-variant` (`#78716C`) 色，`24px` |
| **标签** | `on-surface-variant` 色，`10px` |

## 5. 空状态与异常

### 5.1 新用户冷启动（无档案 + 无咨询记录）

- 应急咨询大卡片：**正常显示**（始终可点击开始新咨询）
- 最近咨询区：显示空状态卡片（"暂无咨询记录"）
- 档案快捷区：显示创建引导卡片（"创建孩子的第一份档案"）
- 整体页面不阻断、不弹窗，用引导卡片温和地推动用户完成首次操作

### 5.2 网络异常

- 页面顶部出现非阻断通知条：`surface-dim` 背景，"网络异常，部分数据可能未及时更新"，`12px`，`error` 色
- 右侧 `×` 关闭按钮，`3s` 后自动消失
- 页面已缓存的数据正常显示，新数据加载失败时保留旧数据

## 8. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 应急咨询大卡片内 `primary` 背景 + `on-primary` 文字对比度为 `5.2:1`，满足要求
- 所有卡片触摸区 ≥ `80px` 高度，远大于最小 `44px`
- TabBar 每个 Tab `56px` 高度满足要求
- 屏幕阅读器：首页应朗读为"篝火智答，{时段}，{昵称}，应急咨询按钮，最近咨询{摘要}，个人档案{昵称}"
- 空状态必须有明确的操作引导，不能只显示"无数据"
