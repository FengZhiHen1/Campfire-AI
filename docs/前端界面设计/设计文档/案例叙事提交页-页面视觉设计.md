## 1. 页面概述

案例叙事提交页是「篝火智答」案例库的 **L1 叙事唯一入口**，同时承担**新建叙事**（`mode=create`）和**编辑草稿叙事**（`mode=edit`）两种模式。用户以自然语言描述完整的干预故事，保存为草稿后可手动触发 AI 提取为结构化干预卡片。

与 L2「案例提交页」（`submit.tsx`，结构化表单录入）形成互补——叙事提交面向"讲故事"的用户，结构化提交面向"填表"的专家。

**设计目标**：
- 降低写作门槛：通过结构化引导帮助用户写出高质量叙事，不要求用户了解四段式格式
- 灵活的工作流：保存草稿与提取卡片解耦，用户可先写后提取、随时修改
- 表单逻辑唯一：新建和编辑共用同一套界面

## 2. 模式差异总览

| 维度 | 新建叙事（`mode=create`） | 编辑草稿（`mode=edit`） |
|:---|:---|:---|
| **页面标题** | "录入案例叙事" | "编辑叙事" |
| **初始数据** | 全部空白 | 加载已有叙事数据，预填充表单 |
| **底部按钮** | "保存草稿" + "提取卡片" 双按钮 | "保存修改" + "提取卡片" 双按钮 |
| **保存后行为** | 保存为 `draft`，返回列表页 | 更新叙事内容，返回列表页 |
| **提取后行为** | 保存 + 触发 LLM 提取 → 跳转提取结果页 | 更新 + 触发 LLM 提取 → 跳转提取结果页 |
| **删除入口** | 无 | 页面底部显示 "删除此叙事" 文字按钮 |

| 属性 | 规格 |

#### 4.4.2 Placeholder 水印引导

输入框为空时，以浅灰色水印文字展示结构化引导清单：

| 属性 | 规格 |
|:---|:---|
| **样式** | `on-surface-variant` 40% opacity，`13px`，line-height `2.0` |
| **内容** | 多行文本：<br>"请描述以下内容：<br>① 孩子的基本情况（年龄、诊断等）<br>② 行为表现与发生场景<br>③ 采取的干预动作与步骤<br>④ 干预后的结果与效果" |
| **行为** | 用户开始输入时水印消失；清空全部内容后水印重新出现 |

> **设计意图**：水印引导在用户未输入时提供结构参照，输入过程中不干扰写作。四要素与 AI 提取 Prompt 的四步流程（识别场景→通用化→四段式提取→循证标注）对应，引导用户写出易于提取的叙事。

## 5. 表单校验

### 5.1 校验规则

| 字段 | 规则 | 错误提示 |
|:---|:---|:---|
| 叙事标题 | 必填，`1-100` 字符 | "请输入叙事标题" / "标题不超过 100 字" |
| 叙事正文 | 必填，`10-5000` 字符 | "请输入至少 10 个字符的叙事内容" / "正文不超过 5000 字" |
| 来源类型 | 必选，默认 "专家撰写" | —（始终有默认值） |

### 5.2 错误呈现

- 点击按钮时若校验失败，页面滚动到第一个错误字段
- 错误字段显示 `error` ghost border + 字段标签变为 `error` 色 + 下方 `error` 色提示文字 `12px`
- 标题输入框错误态：border `error` 色 + 字数统计 `error` 色
- 正文输入框错误态：border `error` 色 + 字数统计 `error` 色

## 6. 空状态与异常

### 6.1 编辑模式加载失败

- 页面中央显示错误信息
- 标题："加载叙事失败"，`16px`，`font-weight: 600`，`on-background`
- 副标题：展示后端错误信息，`14px`，`on-surface-variant`
- 按钮："返回列表"，`primary` 背景

### 6.2 保存失败

- Toast 通知："保存失败，请稍后重试"，`error` 色背景
- 用户当前填写内容不丢失（保留在表单中）

### 6.3 提取失败

- 保存成功但提取失败时，Toast："叙事已保存，但 AI 提取失败，请稍后重试"
- 页面停留在当前页（不跳转提取结果页）
- 叙事已保存为草稿，用户可稍后从列表页重新触发提取

## 9. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 所有触摸目标 ≥ `44px`
- 必填字段必须有 `*` 标记，屏幕阅读器可识别
- 来源类型选项必须可通过屏幕阅读器识别为 radio group
- 正文输入框 Placeholder 水印应通过 `aria-describedby` 与引导清单关联
- 校验错误必须有 `aria-live="polite"` 区域通知

| **从列表新建** | 接收 `mode=create`，FAB 跳入 | 列表页点击 FAB `navigateTo` |
| **从列表编辑** | 接收 `mode=edit&narrativeId=xxx`，预填充数据 | 列表页点击草稿叙事跳入 |
| **保存草稿（新建）** | 写 `draft` → Toast "草稿已保存" → `navigateBack()` | 列表页刷新，"我的提交" Tab 出现新条目 |
| **保存修改（编辑）** | 更新叙事 → Toast "修改已保存" → `navigateBack()` | 列表页和详情页数据同步 |
| **提取卡片** | 保存 → 调用提取 API → 跳转 `extraction-result.tsx` | 提取结果页接收 narrativeId，展示提取结果 |
| **取消（新建）** | 表单有修改时弹窗确认，无修改直接返回 | 列表页恢复 |
| **取消（编辑）** | 同新建 | 叙事详情页恢复（若从详情页进入编辑） |
