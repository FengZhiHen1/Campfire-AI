## 1. 页面概述

案例详情页是用户深入阅读单个真实干预案例的页面。家属在此学习他人的干预经验；老师/专家在此查看、编辑自己提交的案例，并追踪审核进度。页面以卡片分段式布局呈现案例的四段式核心内容，顶部封面图轮播提供情境代入，底部附件列表补充支撑材料。

**设计目标**：
- 阅读沉浸感：长内容分段呈现，降低阅读疲劳
- 情境还原：封面图和四段叙事帮助读者快速进入案例场景
- 审核透明：作者可随时查看 AI 预审结果和专家审核意见
- 编辑触手可及：作者本人可随时唤起编辑模式

## 2. 角色差异化总览

| 角色 | 可见内容 | 可见操作 | 备注 |
|:---|:---|:---|:---|
| **家属** | 封面图、概览信息、四段内容、附件列表 | 无 | 纯浏览态，无编辑按钮，无审核面板 |
| **老师** | 全部内容 + 审核面板（仅自己的案例） | 编辑（跳转 submit.tsx，仅自己的案例）、提交审核、撤回 | 对他人案例同家属纯浏览态 |
| **专家** | 全部内容 + 审核面板（任何案例） | 编辑（跳转 submit.tsx，仅自己的案例）、审核通过/驳回 | 可查看任何案例的审核信息，但只能编辑自己的 |
| **管理员** | 全部内容 + 审核面板 | 编辑（跳转 submit.tsx，任何案例）、强制下架 | 可编辑任何案例 |

### 4.2 封面图轮播区

| 属性 | 规格 |
|:---|:---|
| **高度** | `200px` |
| **背景** | `surface-variant` (`#F5F0EB`) |
| **位置** | 导航栏下方 |

**有封面图时**：

| 属性 | 规格 |
|:---|:---|
| **图片填充** | `object-fit: cover`，填满整个 `200px` 区域 |
| **圆角** | 底部 `16px`（`border-radius: 0 0 16px 16px`） |
| **轮播指示器** | 底部居中，`8px` 间距；当前页 `primary` (`#B45309`) 圆点，`8px`；其他页 `surface` 圆点带 `outline` 边框，`6px` |
| **切换方式** | 左右滑动手势切换；自动轮播 `5s` 间隔（用户开始滑动后暂停自动轮播） |
| **图片计数** | 右下角 `surface` 背景 pill，`on-background` 文字，`11px`，"1/3" 格式 |
| **触摸反馈** | 点击图片进入全屏预览模式（微信原生图片预览） |

**无封面图时**：

- 显示占位状态：
  - 中央显示案例类型抽象图标（`48px`，`on-surface-variant` 40% opacity）
  - 下方文字 "本案例未上传封面图"，`14px`，`on-surface-variant`
  - 若当前用户是作者，额外显示 "+ 添加封面图" 按钮，`primary` 文字按钮，`14px`
- 高度压缩至 `120px`

### 4.5 四段式内容卡片

四段内容各自放在独立的卡片中，每张卡片有左侧彩色强调条。视觉风格与应急咨询的 Stream Output Card 保持一致，但这里是**四个独立卡片**（非单卡片内手风琴）。

**单张卡片外壳**：

| 属性 | 规格 |
|:---|:---|
| **背景** | `surface` (`#FFFFFF`) |
| **阴影** | `diffuse shadow` (`0 4px 24px rgba(28,25,23,0.06)`) |
| **圆角** | `16px` |
| **内边距** | `16px` |
| **外边距** | 左右 `16px`，上下 `12px` |

**左侧强调条**：

| 段落 | 强调条颜色 | 段标题文字色 |
|:---|:---|:---|
| **场景描述** | `secondary` (`#64748B`)，宽 `4px` | `secondary`，`14px`，`font-weight: 600` |
| **行为表现** | `primary` (`#B45309`)，宽 `4px` | `primary`，`14px`，`font-weight: 600` |
| **干预动作** | `tertiary` (`#059669`)，宽 `4px` | `tertiary`，`14px`，`font-weight: 600` |
| **结果反馈** | `tertiary` (`#059669`)，宽 `4px` | `tertiary`，`14px`，`font-weight: 600` |

- 强调条从卡片顶部延伸至底部，圆角与卡片一致
- 段标题位于卡片内部左上角，距左侧强调条 `12px`

**段落正文**：

| 属性 | 规格 |
|:---|:---|
| **文字** | `16px`，`on-background`，line-height `1.75` |
| **位置** | 段标题下方 `12px` |
| **最大行数** | 无限制，允许长文本自然换行 |
| **段落间距** | 段内若有多段，段落之间 `12px` |

**段落间呼吸间距**：

- 相邻两张卡片之间 `12px`
- 卡片进入视口时 `opacity 0→1` + `translateY(8px)→0`，`200ms` `ease-out`（滚动触发，逐段浮现）

| **文件大小** | 右侧，`12px`，`on-surface-variant` |
| **下载/预览箭头** | 最右侧，`▶` 或 `⬇` 图标，`16px`，`on-surface-variant` |
| **点击行为** | 图片直接全屏预览；PDF/文档调用微信原生预览或下载 |

**无附件时**：

- 标题下方显示文字 "本案例未上传附件"，`14px`，`on-surface-variant`，居中
- 若当前用户是作者，显示 "+ 添加附件" 按钮，`primary` 文字按钮，`14px`

## 5. 与 submit.tsx 的协作（方案 A）

> **核心原则**：详情页**纯浏览**，所有数据修改统一在 `submit.tsx` 完成。

### 5.1 跳转行为

- 点击底部操作栏 "编辑案例" 按钮 → `Taro.navigateTo()` 跳转 `submit.tsx`
- 路由参数：`mode=edit&case_id={id}`
- `submit.tsx` 加载该案例全部数据，预填充表单
- 用户在 `submit.tsx` 完成修改后保存 → `navigateBack()` 返回详情页 → 详情页自动刷新数据

### 5.2 新建案例的入口

- 案例列表页 (`index.tsx`) 点击 FAB → 跳转 `submit.tsx?mode=create`
- `submit.tsx` 显示空白表单，必填字段高亮提示

### 5.3 submit.tsx 的表单字段

`submit.tsx` 统一处理新建和编辑，表单包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| 封面图 | 图片上传 | ✗ | 最多 1 张封面图 |
| 案例标题 | 文本输入 | ✓ | 限制 `2-50` 字符 |
| 行为类型 | 下拉选择 | ✓ | 自伤/攻击/逃跑/拒绝服药/情绪爆发/其他 |
| 适用人群 | 多选标签 | ✓ | 年龄段 + 场景标签 |
| 循证等级 | 单选 | ✓ | A/B/C/D |
| 场景描述 | 多行文本 | ✓ | 四段式第一段 |
| 行为表现 | 多行文本 | ✓ | 四段式第二段 |
| 干预动作 | 多行文本 | ✓ | 四段式第三段 |
| 结果反馈 | 多行文本 | ✓ | 四段式第四段 |
| 自定义标签 | 标签输入 | ✗ | 预设标签勾选 + 自定义输入 |
| 附件 | 文件上传 | ✗ | 最多 5 个文件，单文件 ≤10MB |

> submit.tsx 的完整视觉设计见同目录下「案例提交页-页面视觉设计.md」。

## 7. 动效规范汇总

| 动效名称 | 触发条件 | 动画参数 | 备注 |
|:---|:---|:---|:---|
| **页面进入** | 打开详情页 | `opacity 0→1`，`200ms` | 禁止 lateral slide |
| **封面图轮播** | 自动/手动切换 | `opacity 0→1` + `translateX`，`300ms` `ease-out` | 图片切换过渡 |
| **内容卡片浮现** | 滚动进入视口 | `opacity 0→1` + `translateY(8px)→0`，`200ms` `ease-out` | 逐段浮现，使用 IntersectionObserver |
| **审核面板展开** | 点击面板头部 | `height` 过渡，`300ms` `ease-in-out`；内部 `opacity` 同步 | 平滑展开 |
| **FAB 入场** | 页面加载 | `opacity 0→1` + `scale 0.8→1`，`300ms` `ease-out`，延迟 `200ms` | 悬浮按钮温和出现 |
| **FAB 滚动缩小** | 向下滚动 >100px | `scale 1→0.86`，`200ms` `ease-out` | 减少视觉干扰 |
| **底部栏滑入** | 页面加载 | `translateY(100%)→0`，`300ms` `ease-out`，延迟 `200ms` | 操作栏温和出现 |
| **附件列表项点击** | 点击附件 | `scale 0.98`，`100ms` | 按压反馈 |
| **删除确认弹窗** | 点击删除 | `opacity 0→1` + `scale 0.95→1`，`200ms` `ease-out` | 居中模态 |

## 9. 可访问性要求

- 全部文字/背景对比度 ≥ WCAG AA (`4.5:1`)
- 封面图轮播支持屏幕阅读器朗读图片计数（"第 1 张，共 3 张"）
- 四段内容卡片的段标题必须使用 heading 语义，方便屏幕阅读器导航
- 审核面板内的操作按钮必须有明确的 `aria-label`（如"提交案例审核"而非模糊的"提交"）
- 编辑 FAB `56px` 圆形触摸区满足要求；附件列表项 `56px` 高度满足要求
- 删除确认要求输入标题，增加防误触层级
