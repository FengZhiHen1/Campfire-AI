# 「篝火智答」需求分析（转型后）

## 1. 项目概述

| 项 | 内容 |
|---|---|
| **平台名称** | 篝火智答 |
| **定位** | 基于大模型的孤独症垂直领域全天候 AI 咨询平台 |
| **核心场景** | 孤独症人士突发行为/情绪危机时，为家属提供即时、可执行的应急干预建议 |
| **技术底座** | RAG（检索增强生成）+ 真实案例库 + 个性化行为-情绪档案 |
| **转型关键** | 从「泛心智障碍科普」收缩为「孤独症行为+情绪应急干预工具」 |

---

## 2. 目标用户与角色

| 角色 | 核心诉求 | 平台价值 |
|---|---|---|
| **患者家属（核心用户）** | 患者突发异常行为/情绪时，第一时间获得可参考的应急方案 | 7×24 即时响应，基于真实案例的个性化建议 |
| **特教/康复老师** | 沉淀干预经验，辅助家属进行家庭泛化训练 | 共建个性化档案，上传/审核真实案例 |
| **专家** | 把关案例与方案质量，处理 AI 无法覆盖的疑难场景 | 审核案例库、接管疑难工单 |
| **管理员** | 平台运营与人员准入管理 | 案例/人员/工单调度 |
| **维护人员** | 系统稳定与数据安全 | 运维监控、日志审计 |

> **角色调整说明**：原「志愿者」弱化或合并为「老师/专家」体系，突出**专业干预经验**的沉淀与流转。

---

## 3. 核心功能需求

### 3.1 智能应急咨询（RAG 引擎，P0）

| 需求项 | 说明 |
|---|---|
| **应急问答** | 家属输入患者当前行为/情绪表现，RAG 基于案例库生成结构化应急方案（即时动作 → 安抚话术 → 观察指标 → 就医判断） |
| **案例检索** | 优先匹配「同年龄段 + 同行为类型 + 同情绪等级」的真实干预案例 |
| **个性化增强** | 检索时注入该患者的历史行为-情绪档案，生成「专属版」建议 |
| **来源溯源** | 每条建议必须标注来源案例/专家文章，附带免责声明 |
| **兜底机制** | 置信度低于阈值或涉及高风险（用药/自伤）时，强制提示「建议立即联系专家/就医」，并一键转人工 |

### 3.2 个性化档案库（P0，新增核心模块）

| 需求项 | 说明 |
|---|---|
| **档案共建** | 家属记录日常行为/情绪事件；老师/专家补充专业评估与干预记录 |
| **标签体系** | 年龄、诊断类型、语言水平、刻板行为类型、情绪触发因素、既往干预反应等 |
| **档案驱动 RAG** | 档案数据作为检索过滤条件，提升案例匹配精准度 |
| **隐私控制** | 档案仅家属本人及关联老师/专家可见，严格脱敏 |

### 3.3 真实案例库管理（P0，知识库重构）

| 需求项 | 说明 |
|---|---|
| **案例录入** | 老师/专家上传真实干预案例（场景描述 → 行为表现 → 干预动作 → 结果反馈 → 适用人群） |
| **案例审核** | 专家审核案例的完整性与合规性，驳回需附修改意见 |
| **应急方案模板** | 按行为类型（自伤/攻击/刻板/逃跑等）与情绪等级分级沉淀标准化模板 |
| **双写机制** | 案例元数据存 MySQL（供人查阅），切片内容入向量库（供 AI 检索） |
| **切片策略** | 案例类采用「场景-行为-干预-结果」强绑定切片，禁止拆分核心动作与适用前提 |

### 3.4 人工兜底通道（P1）

| 需求项 | 说明 |
|---|---|
| **工单流转** | AI 置信度不足或用户主动要求时，生成人工咨询工单 |
| **专家接单** | 专家/老师直接接取工单，无需管理员二次审核 |
| **紧急标注** | 涉及自伤、攻击等高风险行为的工单自动标红优先处理 |

### 3.5 科普查阅（P2，附带功能）

| 需求项 | 说明 |
|---|---|
| **轻量科普** | 保留基础科普内容（术语解释、干预方法介绍），但不做为主入口 |
| **案例关联** | 科普内容关联相关真实案例，从「知道」延伸到「看到怎么做」 |

---

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| **响应速度** | 应急场景 TTFT（首字延迟）≤ 3 秒，全流程生成 ≤ 10 秒 |
| **可用性** | 7×24 可用，SLA ≥ 99.5% |
| **准确性** | 基于测试集的应急方案，专业人员认可度 ≥ 85%，零幻觉（严禁编造医疗建议） |
| **安全性** | 案例与档案严格脱敏；手机号仅管理员可见；所有建议带免责声明 |
| **合规性** | 明确标注「AI 建议仅供参考，不构成医疗诊断或治疗建议」 |
| **可扩展性** | 档案标签体系、案例分类体系预留扩展字段，适配后续多维度分析 |

---

## 5. 风险与约束

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| **案例真实性不足** | RAG 输出缺乏说服力 | 建立「老师/专家实名认证 + 案例审核」双保险；初期以专家撰写案例为主 |
| **个性化档案冷启动** | 新用户无档案，体验降级 | 允许无档案模式运行（纯案例检索），引导用户逐步完善档案 |
| **应急建议误用风险** | 家属按错误建议操作造成伤害 | 强制免责声明 + 高风险场景自动转人工 + 所有建议标注「请在专业人员指导下使用」 |
| **算力与成本** | 全天候运行 + 向量检索成本 | 轻量模型（DeepSeek 等）+ ChromaDB 本地/容器化部署，控制 Token 消耗 |
| **隐私泄露** | 案例与档案涉及敏感家庭信息 | 上传时自动 PII 检测与脱敏；档案访问严格基于 RBAC |

---

## 6. 待确认事项

1. **案例来源**：现有可录入的真实案例数量？是否需与大连爱纳等机构签署案例脱敏使用协议？
2. **档案共建流程**：老师/专家参与档案共建的意愿与频次如何？是否需要设计激励机制？
3. **应急分级标准**：行为/情绪危机的等级划分（如轻度/中度/重度）是否有统一标准参考？

---

**总结**：「篝火智答」的核心需求可浓缩为一句话——**以真实干预案例为知识燃料，以个性化行为-情绪档案为检索透镜，在孤独症家庭最紧急的时刻，用 AI 点燃一团即时、可信、有温度的应急篝火。**
